# 7. คำสั่งซื้อ — การสร้าง Work Order

## ภาพรวม

นี่คือขั้นตอนการส่งต่อที่สำคัญที่ทีมขายส่งคำสั่งซื้อที่ยืนยันแล้วไปยังพื้นที่การผลิต การสร้าง Work Order จากคำสั่งซื้อจะสร้าง WO หนึ่งรายการต่อรายการสินค้า แต่ละรายการพร้อมขั้นตอนกระบวนการที่เติมข้อมูลอัตโนมัติ

---

## ข้อกำหนดเบื้องต้น

ก่อนสร้าง Work Order คำสั่งซื้อต้องอยู่ในสถานะ **Confirmed** ถ้าคำสั่งซื้อยังอยู่ใน Draft ให้เปลี่ยนสถานะเป็น Confirmed ก่อน (ดู [Sales Orders — List & Manage](./05-sales-orders-list.md))

---

## ขั้นตอน: การสร้าง Work Order

### ขั้นตอนที่ 1: ค้นหาคำสั่งซื้อที่ยืนยันแล้ว

1. ไปที่ **Sales → Sales Order**
2. ค้นหาคำสั่งซื้อที่มีสถานะ **Confirmed**
3. ปุ่ม **+ Work Orders** ในคอลัมน์ Operate จะใช้งานได้ (ข้อความสีน้ำเงิน)

[image: sales-order-wo-button-active — ภาพใกล้ของแถวคำสั่งซื้อ Confirmed พร้อมปุ่ม "+ Work Orders" ที่ไฮไลต์สีน้ำเงิน]

> **หมายเหตุ:** สำหรับคำสั่งซื้อในสถานะอื่น ปุ่ม "+ Work Orders" จะถูกปิดใช้งาน (สีเทา) การวางเมาส์จะแสดง Tooltip: "Order must be Confirmed to create Work Orders."

[image: sales-order-wo-button-disabled — ภาพใกล้ของคำสั่งซื้อที่ไม่ใช่ Confirmed แสดงปุ่มที่ปิดใช้งานพร้อม Tooltip]

### ขั้นตอนที่ 2: เปิดกล่องโต้ตอบสร้าง Work Order

1. คลิก **+ Work Orders** บนแถวคำสั่งซื้อ Confirmed
2. กล่องโต้ตอบจะเปิดแสดงสรุปสิ่งที่จะถูกสร้าง

[image: sales-order-wo-dialog — ภาพหน้าจอของกล่องโต้ตอบ "Create Work Orders from Sales Order"]

### ขั้นตอนที่ 3: ตรวจสอบกล่องโต้ตอบ

กล่องโต้ตอบแสดง:

| ส่วน | เนื้อหา |
|---------|---------|
| Sales Order | รหัสคำสั่งซื้อ (เช่น `SO-2026-001`) |
| Customer | ชื่อลูกค้า |
| ตารางรายการสินค้า | ตารางแสดงแต่ละรายการสินค้า: #, ผลิตภัณฑ์, จำนวน, วัสดุ |
| ข้อความข้อมูล | "One work order per line item will be created with auto-generated codes. The Sales Order status will move to 'In Production'." |

[image: sales-order-wo-dialog-details — ภาพใกล้ของกล่องโต้ตอบแสดงตารางรายการสินค้าและข้อความข้อมูล]

### ขั้นตอนที่ 4: ยืนยันการสร้าง

1. ตรวจสอบรายการสินค้าเพื่อให้แน่ใจว่าถูกต้อง
2. คลิก **Create X Work Order(s)** (โดยที่ X คือจำนวนรายการสินค้า)
3. ปุ่มจะแสดง "Creating…" ขณะประมวลผล

[image: sales-order-wo-creating — ภาพหน้าจอของกล่องโต้ตอบพร้อมปุ่มที่แสดงสถานะ "Creating…"]

### ขั้นตอนที่ 5: เสร็จสมบูรณ์

หลังจากสร้างสำเร็จ:

1. กล่องโต้ตอบปิดอัตโนมัติ
2. Toast สำเร็จจะปรากฏ
3. สถานะ Sales Order เปลี่ยนเป็น **In Production** (ป้ายสีเหลือง)
4. คอลัมน์ Production Progress เริ่มแสดงความคืบหน้า

[image: sales-order-wo-success — ภาพหน้าจอของรายการ Sales Order แสดงคำสั่งซื้อที่อยู่ในสถานะ "In Production" พร้อมตัวบ่งชี้ความคืบหน้า]

---

## สิ่งที่ถูกสร้าง

สำหรับแต่ละรายการสินค้าในคำสั่งซื้อ ระบบจะสร้าง:

| Entity ที่สร้าง | รายละเอียด |
|----------------|---------|
| Work Order | รูปแบบรหัส: `WO-2026-00001`, `WO-2026-00002` ฯลฯ |
| WO Steps | เติมข้อมูลอัตโนมัติจากกระบวนการผลิตของรายการสินค้า |
| Material link | จับคู่จากฟิลด์ `material_spec` ของรายการสินค้า |
| Customer link | สืบทอดจากคำสั่งซื้อ |
| Quantity & Unit | คัดลอกจากรายการสินค้า |

### ลำดับความสำคัญในการกำหนดขั้นตอน

ระบบกำหนดขั้นตอน WO โดยใช้ลำดับความสำคัญนี้:

1. **กระบวนการรายการใบเสนอราคา** — ถ้ารายการสินค้า SO ถูกแปลงจากใบเสนอราคา กระบวนการของรายการใบเสนอราคาจะถูกใช้ (ผ่าน `source_quote_item_id`)
2. **กระบวนการรายการ SO** — ถ้ากระบวนการถูกกำหนดโดยตรงบนรายการสินค้า SO
3. **การจับคู่ชื่อผลิตภัณฑ์** — ย้อนกลับไปจับคู่ตามชื่อผลิตภัณฑ์
4. **ว่างเปล่า** — ถ้าไม่พบกระบวนการ WO จะถูกสร้างโดยไม่มีขั้นตอน (ต้องป้อนด้วยตนเอง)

---

## หลังจากสร้าง Work Order

เมื่อสร้าง Work Order แล้ว งานหลักของทีมขายเสร็จสิ้น คำสั่งซื้อจะมองเห็นได้ใน:

- **Production → Work Orders** — ทีมผลิตจัดการการดำเนินงาน WO
- **Production → Scheduling** — ผู้วางแผนจัดตารางการดำเนินงานบนเครื่องจักร

ทีมขายสามารถติดตามความคืบหน้าต่อไปจากหน้ารายการ Sales Order:

- คอลัมน์ **Production Progress** แสดงเปอร์เซ็นต์ความสำเร็จแบบเรียลไทม์
- เมื่อ WO ทั้งหมดเสร็จสมบูรณ์ SO จะเปลี่ยนสถานะเป็น **Completed** อัตโนมัติ
- ป้าย **"Delivered"** จะปรากฏเมื่อการผลิตทั้งหมดเสร็จสิ้น

[image: sales-order-monitoring — ภาพหน้าจอของรายการ Sales Order แสดงคำสั่งซื้อในขั้นตอนต่างๆ: หนึ่งรายการ "In Production" ที่ 45% และหนึ่งรายการแสดงป้าย "Delivered"]

## การแก้ไขปัญหา

| ปัญหา | สาเหตุ | วิธีแก้ไข |
|-------|-------|----------|
| ปุ่ม "+ Work Orders" ถูกปิดใช้งาน | คำสั่งซื้อไม่ได้อยู่ในสถานะ Confirmed | เปลี่ยนสถานะเป็น Confirmed ก่อน |
| ข้อผิดพลาด "Sales order has no line items" | คำสั่งซื้อไม่มีรายการสินค้า | แก้ไขคำสั่งซื้อและเพิ่มรายการสินค้า |
| WO ถูกสร้างโดยไม่มีขั้นตอน | ไม่มีกระบวนการที่กำหนดบนรายการสินค้าหรือใบเสนอราคาต้นทาง | แก้ไข WO ด้วยตนเองเพื่อเพิ่มขั้นตอน หรือแก้ไข SO เพื่อเพิ่มกระบวนการก่อนสร้าง WO |
| ไม่สามารถลบ SO หลังจากสร้าง WO | SO ถูกอ้างอิงโดยระเบียนโปรเจกต์/WO | ยกเลิก WO ก่อนถ้าจำเป็น หรือเก็บ SO ไว้ตามเดิม |
