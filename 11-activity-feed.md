# Activity Feed (ฟีดกิจกรรม)

## คืออะไร?

Activity Feed คือศูนย์กลางการสื่อสารและบันทึกการตรวจสอบที่แนบกับ Work Order ทุกรายการ ประกอบด้วยสองชั้นในไทม์ไลน์เดียว: เหตุการณ์ระบบที่สร้างอัตโนมัติ (การเปลี่ยนสถานะ, การย้ายตาราง, เหตุการณ์ NCR, การอัปเดต OPO) และความคิดเห็นที่ผู้ใช้ป้อนพร้อมรูปภาพและไฟล์แนบ

คิดว่ามันเป็นบันทึก "เกิดอะไรขึ้นและทำไม" สำหรับทุกงาน เมื่องานด่วนเข้ามาแทรกตาราง เมื่อผู้รับเหมาช่วงส่งงานช้า เมื่อ QC ตรวจพบปัญหา — บริบททั้งหมดถูกบันทึกไว้ที่นี่พร้อมกับเหตุการณ์ระบบ

## วิธีการทำงาน

ทุกการดำเนินการสำคัญบน Work Order จะสร้างเหตุการณ์ในฟีดโดยอัตโนมัติ ผู้ใช้ยังสามารถโพสต์ความคิดเห็นพร้อมรูปภาพ ไฟล์ และแท็กหมวดหมู่เพื่อเพิ่มบริบทที่ระบบไม่สามารถบันทึกได้เอง

```
Work Order Activity Feed
├── เหตุการณ์ระบบ (สร้างอัตโนมัติ, ไม่สามารถแก้ไข)
│   ├── สถานะเปลี่ยน: draft → released (กฤษ, 08:15)
│   ├── จัดตาราง: Op10 → CNC Mill #3, จันทร์ 08:00 (กฤษ, 08:20)
│   ├── ขั้นตอนเริ่ม: CNC Milling Op10 (สมชาย, 08:30)
│   └── ...
├── ความคิดเห็น (ผู้ใช้ป้อน)
│   ├── [QC] "ชิ้นแรกวัดแล้ว — ขนาดรูเกิน 0.02mm" + 📷 รูปภาพ (มาลี, 10:45)
│   ├── [งานด่วน] "ลูกค้าโทรมา — ต้องการ 10 ชิ้นภายในวันศุกร์" (ณัฐพงษ์, 14:00)
│   └── ...
└── แสดงทั้งหมดตามลำดับเวลาย้อนกลับ (ใหม่สุดก่อน)
```

## เข้าถึงได้จากที่ไหน

Activity Feed สามารถเข้าถึงได้จาก 6 ตำแหน่งในแอป มองหาไอคอน 💬 (ข้อความ) พร้อมป้ายจำนวน:

| ตำแหน่ง | วิธีเปิด |
|----------|-------------|
| หน้า Work Orders | คลิกไอคอนกิจกรรมบนแถว WO หรือในแผ่นรายละเอียด WO |
| Scheduling Board | คลิกไอคอนกิจกรรมในป๊อปอัพรายละเอียดการดำเนินงาน |
| Production Kanban | คลิกไอคอนกิจกรรมบนการ์ด WO |
| Operator Panel | กดปุ่มกิจกรรมในส่วนหัวรายละเอียดขั้นตอน |
| Order Traceability | คลิกไอคอนกิจกรรมบนโหนด WO |
| Outside Processing | คลิกไอคอนกิจกรรมในแผ่นรายละเอียด OPO |

ฟีดจะเปิดเป็นแผงเลื่อนด้านขวา (เดสก์ท็อป) หรือแผ่นล่างเต็มหน้าจอ (มือถือ/ผู้ปฏิบัติงาน)

## เหตุการณ์ระบบ (บันทึกอัตโนมัติ)

สร้างโดยอัตโนมัติ — ไม่ต้องดำเนินการใดๆ:

| เหตุการณ์ | สิ่งที่ทริกเกอร์ |
|-------|-----------------|
| สถานะ WO เปลี่ยน | การเปลี่ยนสถานะใดๆ (draft → released, in_progress → on_hold ฯลฯ) |
| สถานะขั้นตอนเปลี่ยน | ผู้ปฏิบัติงานเริ่ม เสร็จสิ้น หรือข้ามขั้นตอน |
| จัดตารางแล้ว | ขั้นตอนถูกวางบนเครื่องจักรบน Scheduling Board |
| ย้ายตาราง | การดำเนินงานที่จัดตารางถูกลากไปยังเวลา/เครื่องจักรอื่น |
| ลบตาราง | การดำเนินงานที่จัดตารางถูกยกเลิก |
| สร้าง NCR | รายงานความไม่สอดคล้อง (NCR) ถูกสร้างสำหรับ WO นี้ |
| แก้ไข NCR | NCR ถูกแก้ไขด้วยการตัดสินใจ (rework, remake, scrap ฯลฯ) |
| สถานะ OPO เปลี่ยน | คำสั่งจ้างผลิตภายนอกส่งออก กลับมา หรือเกินกำหนด |
| เปลี่ยนความสำคัญ | ระดับความสำคัญของ WO ถูกเปลี่ยน |
| รีเซ็ต Rework | ขั้นตอนถูกรีเซ็ตกลับเป็น pending หลังจากการตัดสินใจ rework |
| สร้าง Remake | WO ลูกถูกสร้างจากการตัดสินใจ remake |

เหตุการณ์ระบบไม่สามารถแก้ไขหรือลบได้

## ความคิดเห็น (Comments)

ผู้ใช้ที่ผ่านการยืนยันตัวตนทุกคนสามารถโพสต์ความคิดเห็นบน WO ที่ตนเองดูได้ ความคิดเห็นรองรับ:

- ข้อความอิสระ (สูงสุด 2,000 ตัวอักษร)
- ไฟล์แนบรูปภาพ (JPEG, PNG, WebP — สูงสุด 10MB ต่อรูป, สูงสุด 5 รูปต่อความคิดเห็น)
- ไฟล์แนบเอกสาร (PDF, DOCX, XLSX, DXF, STEP — สูงสุด 25MB ต่อไฟล์, สูงสุด 3 ไฟล์ต่อความคิดเห็น)
- แท็กหมวดหมู่: QC, โลจิสติกส์, วิศวกรรม, ลูกค้า, งานด่วน, เหตุผลหยุด, ทั่วไป

ความคิดเห็นสามารถลบได้โดยผู้เขียนภายใน 15 นาทีหลังโพสต์ หลังจากนั้นจะกลายเป็นถาวร (เพื่อความสมบูรณ์ของบันทึกการตรวจสอบ)

## การกรอง

ฟีดรองรับการกรองเพื่อค้นหาสิ่งที่ต้องการ:

- **ทั้งหมด** — ทุกอย่าง (ค่าเริ่มต้น)
- **เหตุการณ์เท่านั้น** — เหตุการณ์ระบบโดยไม่มีความคิดเห็น
- **ความคิดเห็นเท่านั้น** — ความคิดเห็นของผู้ใช้โดยไม่มีเหตุการณ์ระบบ
- **ตามหมวดหมู่** — กรองความคิดเห็นตามแท็ก (QC, งานด่วน, วิศวกรรม ฯลฯ)

## แนวคิดสำคัญ

| คำศัพท์ | ความหมาย |
|------|---------|
| Activity Event (เหตุการณ์กิจกรรม) | รายการบันทึกระบบที่สร้างอัตโนมัติ (ไม่สามารถแก้ไข) |
| Comment (ความคิดเห็น) | บันทึกที่ผู้ใช้ป้อนพร้อมไฟล์แนบและแท็กหมวดหมู่ (ไม่บังคับ) |
| Category Tag (แท็กหมวดหมู่) | ป้ายจำแนกประเภทบนความคิดเห็น (QC, โลจิสติกส์, วิศวกรรม ฯลฯ) |
| Activity Sheet (แผ่นกิจกรรม) | แผงเลื่อนที่แสดงฟีด |
| Count Badge (ป้ายจำนวน) | ตัวเลขบนปุ่มกิจกรรมแสดงจำนวนรายการทั้งหมดสำหรับ WO นั้น |
