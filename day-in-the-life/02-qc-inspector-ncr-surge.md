## 08:40 — NCR-2026-031: Motor Housing ครบกำหนดวันนี้

เอิร์ธเปิด NCR-2026-031 สร้างโดยช่างกะกลางคืนที่สังเกตเห็นเส้นผ่านศูนย์กลาง Bore ไม่ได้ค่าความเผื่อบน 3 ใน 5 ชิ้น

**รายละเอียด:**
- WO: WO-2026-00018, Motor Housing × 5
- ประเภทข้อบกพร่อง: Dimensional
- คำอธิบาย: "เส้นผ่านศูนย์กลาง Bore 42.05mm, ค่าความเผื่อ 42.00 ±0.01mm ชิ้นงาน 3 ชิ้นไม่ได้ Spec"
- ความรุนแรง: Major
- สถานะ WO ปัจจุบัน: `in_progress`, ที่ขั้นตอน 50 (CMM Inspection)

**คำถาม:** สามารถส่ง 2 ชิ้นที่ดีวันนี้ในขณะที่ 3 ชิ้นที่เสียถูก Rework ได้หรือไม่?

เอิร์ธตรวจสอบ Sales Order ลูกค้า (XPeng Motors) สั่ง 5 ชิ้น SO ไม่มีการจัดการส่งมอบบางส่วน — พวกเขาต้องการทั้ง 5 ชิ้น

เธอตรวจสอบข้อบกพร่อง: เส้นผ่านศูนย์กลาง Bore 42.05mm กับเป้าหมาย 42.00mm นั่นคือ Oversize 0.04mm สามารถแก้ไขด้วยการ Re-boring ได้หรือไม่? เธอโทรหาวิศวกรกระบวนการ คำตอบ: ได้ แต่ต้องการการ Re-fixture บน CNC Mill

**การตัดสินใจ: Rework บน 3 ชิ้น**

### การมอบหมายและ Resolve NCR

เอิร์ธเปิด NCR-2026-031 → คลิก **Edit** → ตั้งค่า:
- มอบหมายให้: กฤต (ผู้วางแผน) — เขาต้องจัดตาราง Rework
- สาเหตุรากฐาน: "CNC Offset Drift — เครื่องมือ Boring Bar สึกหรอ"
- Disposition: **Rework**

เธอคลิก **Move to In Review** (สถานะ NCR: open → in_review) ระบบต้องการให้ตั้ง `assigned_to` ก่อนการเปลี่ยนสถานะนี้ — ตั้งค่าแล้ว

เธอโทรหากฤตเพื่อแจ้งสถานการณ์ เขาต้องรู้ว่าการส่งมอบ Motor Housing มีความเสี่ยง

กฤตเปิด NCR → ตรวจสอบ → คลิก **Resolve** ระบบดำเนินการ Rework Engine:
- ขั้นตอน 30 (CNC Milling Op30 — การดำเนินการ Boring) และขั้นตอนถัดไปทั้งหมด Reset เป็น `pending`
- สถานะ WO ยังคงเป็น `in_progress`
- `rework_count` บน WO เพิ่มเป็น 1

กฤตจัดตาราง Rework Steps บน Scheduling Board — 3 ชิ้นต้องกลับไปที่ CNC Mill เช้านี้

เอิร์ธปิด NCR: **Move to Closed**

**เวลาที่ใช้กับ NCR-2026-031: 25 นาที**

## 09:05 — NCR-2026-033: Bracket Assembly — ความรุนแรง Critical

เอิร์ธเปิด NCR-2026-033 รายการนี้สร้างโดยกะ QC ก่อนหน้า

**รายละเอียด:**
- WO: WO-2026-00031, Bracket Assembly × 3
- ประเภทข้อบกพร่อง: Dimensional
- คำอธิบาย: "รูปแบบรูยึดเลื่อน 0.8mm ในแกน X ทั้ง 3 ชิ้นได้รับผลกระทบ"
- ความรุนแรง: Critical
- สถานะ WO ปัจจุบัน: `in_progress`, ที่ขั้นตอน 50 (CMM Inspection)
- WO ครบกำหนด: วันพุธ

รูปแบบรูเลื่อน 0.8mm บนทั้ง 3 ชิ้น นี่ไม่ใช่ปัญหาการสึกหรอของเครื่องมือ — มันคือข้อผิดพลาด Fixture หรือ Datum ชิ้นงานไม่สามารถ Rework ได้ (ไม่สามารถย้ายรูที่เจาะแล้ว)

**การตัดสินใจ: Remake ทั้ง 3 ชิ้น**

แต่วันพุธคือพรุ่งนี้ สามารถกลึงและตรวจสอบ Bracket Assembly ใหม่ 3 ชิ้นได้ในหนึ่งวันหรือไม่?

เอิร์ธตรวจสอบ Process Route: CNC Milling (2.5h) → Deburring (0.5h) → CMM Inspection (0.3h) → Packing (0.2h) รวม: 3.5h ไม่มี Outside Processing เป็นไปได้ในหนึ่งวันหากจัดตารางเป็นงานแรกของพรุ่งนี้

เธอยังตรวจสอบ: ลูกค้า (Bosch) รู้หรือยัง? เธอดู SO — Delivery Promise Signal เป็นสีแดงแล้ว (WO มีความเสี่ยง) เธอแจ้งฝ่ายขายเพื่อให้ติดต่อ Bosch เชิงรุก

### Resolve เป็น Remake

เอิร์ธเปิด NCR-2026-033 → ตั้งค่า:
- มอบหมายให้: กฤต
- สาเหตุรากฐาน: "ข้อผิดพลาด Fixture Datum — แผ่น Fixture สึกหรอ การอ้างอิงแกน X เลื่อน"
- Disposition: **Remake**

กฤตตรวจสอบและ Resolve Remake Engine:
- ปิด WO-2026-00031 (ต้นฉบับ)
- สร้าง WO-2026-00052 (Child WO, 3 ชิ้น, Route เดิม)
- สร้าง Scrap Inventory Transaction สำหรับ 3 ชิ้นที่มีข้อบกพร่อง
- เชื่อมโยง Child WO กับ NCR-2026-033

กฤต Release WO-2026-00052 ทันทีและจัดตารางบน Scheduling Board สำหรับวันพุธ 08:00 — งานแรกของวันบน CNC Mill

เอิร์ธบันทึกใน NCR: "แผ่น Fixture EQP-0005-FIX-003 ต้องเปลี่ยนก่อนรัน Remake WO ยื่น Maintenance Request แล้ว"

เธอยังยื่น Maintenance Request สำหรับการเปลี่ยนแผ่น Fixture

**เวลาที่ใช้กับ NCR-2026-033: 30 นาที**

- หมายเหตุ: "รอยขีดข่วนบนหน้าที่ไม่ใช่พื้นผิวประกอบ B Spec ของ Continental อนุญาตข้อบกพร่องด้านสวยงามบนพื้นผิวที่ไม่ใช่การทำงาน อ้างอิงแบบ: DWG-2026-0027 Rev B, Note 3"

## สิ่งที่ทำให้วันนี้ทำงานได้

**คัดกรองก่อนดำเนินการ** เอิร์ธไม่ได้เปิด NCR แรกและเริ่มทำงาน — เธอดูทั้งสามรายการก่อนและจัดลำดับตามความเร่งด่วน Motor Housing (ครบกำหนดวันนี้) ถูกจัดการก่อน Bracket Assembly (ครบกำหนดวันพุธ)

**Disposition ตรงกับข้อบกพร่อง** แต่ละ NCR ได้รับ Disposition ที่ถูกต้อง:
- ข้อบกพร่องที่ Re-boring ได้ → Rework (WO เดิม, Reset ขั้นตอน)
- รูปแบบรูที่แก้ไขไม่ได้ → Remake (WO ใหม่, Scrap ต้นฉบับ)
- ด้านสวยงามบนพื้นผิวที่ไม่ใช่พื้นผิวประกอบ → Use As-Is (ไม่กระทบการผลิต)

**สื่อสาร Upstream** เมื่อ Remake ของ Bracket Assembly ทำให้การส่งมอบวันพุธมีความเสี่ยง เอิร์ธแจ้งฝ่ายขายทันที — ไม่ใช่หลังจากนั้น Delivery Promise Signal เปลี่ยนเป็นสีแดงเป็นตัวกระตุ้น

**ปิดวงจร** แผ่น Fixture ที่ทำให้เกิดข้อบกพร่อง Bracket Assembly ได้รับ Maintenance Request ปัญหาบรรจุภัณฑ์บน Manifold Block ได้รับหมายเหตุบน OPO ข้อบกพร่องที่ไม่ได้วิเคราะห์สาเหตุรากฐานมักเกิดซ้ำ

## การตัดสินใจที่เอิร์ธทำ (และเหตุผล)

| การตัดสินใจ | เหตุผล |
|------------|-------|
| Rework บน Motor Housing (ไม่ใช่ Remake) | Bore Oversize แก้ไขได้ด้วยการ Re-boring Remake จะใช้เวลานานกว่า Rework และพลาดการส่งมอบวันนี้ |
| Remake บน Bracket Assembly (ไม่ใช่ Rework) | รูปแบบรูเลื่อนส่งผลต่อทั้ง 3 ชิ้นและไม่สามารถแก้ไขได้ — รูเจาะอยู่ในตำแหน่งผิดแล้ว |
| Use As-Is บน Manifold Block | Spec ลูกค้าอนุญาตข้อบกพร่องด้านสวยงามบนพื้นผิวที่ไม่ใช่พื้นผิวประกอบอย่างชัดเจน ไม่กระทบการทำงาน |
| ตรวจสอบ Shaft Component 100% | Shaft ค่าความเผื่อตึงครบกำหนดพรุ่งนี้ — ตัวอย่าง 10% ไม่เพียงพอสำหรับการส่งมอบวันถัดไป |
| แจ้งฝ่ายขายเกี่ยวกับ Bracket Assembly ทันที | Delivery Promise Signal เปลี่ยนเป็นสีแดง ฝ่ายขายต้องติดต่อลูกค้าก่อนที่ลูกค้าจะติดต่อพวกเขา |
