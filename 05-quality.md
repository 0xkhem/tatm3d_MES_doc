# Quality (คุณภาพ)

## คืออะไร?

โมดูล Quality บันทึกผลการตรวจสอบคุณภาพของชิ้นงานในทุกจุดของกระบวนการผลิต และจัดการสิ่งที่เกิดขึ้นเมื่อชิ้นงานไม่ผ่าน เชื่อมต่อโดยตรงกับ Work Orders — การตรวจสอบที่ไม่ผ่านบนขั้นตอน WO จะสร้าง NCR ที่ต้องแก้ไขก่อนการผลิตจะดำเนินต่อได้

## Inspections (การตรวจสอบ)

Inspection คือการตรวจสอบคุณภาพอย่างเป็นทางการของชิ้นงานชุดหนึ่ง บันทึก:

- WO และขั้นตอนที่ตรวจสอบ
- จำนวนที่ตรวจสอบ ผ่าน และไม่ผ่าน
- วิธีการตรวจสอบและผลลัพธ์ (Pass / Fail / Conditional)
- ผู้ตรวจสอบและเวลา

Inspection สามารถสร้างได้ที่ขั้นตอนใดก็ได้ แต่พบบ่อยที่สุดที่ขั้นตอน QC ที่กำหนดในเส้นทางกระบวนการ

## NCR — Non-Conformance Report (รายงานความไม่สอดคล้อง)

เมื่อการตรวจสอบไม่ผ่าน จะสร้าง NCR ขึ้น บันทึก:

- ข้อบกพร่องที่พบ
- หมวดหมู่สาเหตุรากฐาน
- การตัดสินใจ: จะทำอะไรกับชิ้นส่วนที่ไม่สอดคล้อง

รูปแบบรหัส NCR: `NCR-2026-001`

## การตัดสินใจ NCR

| การตัดสินใจ | สิ่งที่เกิดขึ้น |
|------------|---------------|
| Rework | ขั้นตอนที่ไม่ผ่านถูก Reset เป็น Pending และต้องทำใหม่ WO ดำเนินต่อในเส้นทางเดิม |
| Remake | สร้าง Work Order ลูกใหม่เพื่อผลิตจำนวนที่ไม่ผ่านใหม่ WO เดิมไม่ได้รับผลกระทบ |
| Scrap | ชิ้นส่วนถูกตัดออก ไม่มีการดำเนินการผลิตเพิ่มเติม |
| Use As Is | ยอมรับชิ้นส่วนแม้จะมีความไม่สอดคล้อง บันทึกไว้เพื่อการตรวจสอบย้อนกลับ |
| Return to Vendor | ส่งชิ้นส่วนกลับ (เกี่ยวข้องกับชิ้นส่วนที่ซื้อมา) |

## Rework vs. Remake

**Rework** สำหรับข้อบกพร่องที่แก้ไขได้ — ชิ้นส่วนมีอยู่แต่ต้องการงานเพิ่มเติม ระบบ Reset ขั้นตอนที่ได้รับผลกระทบ (และขั้นตอนปลายน้ำที่ขึ้นอยู่กับมัน) กลับเป็น Pending

**Remake** สำหรับชิ้นส่วนที่ไม่สามารถกู้คืนได้ ระบบสร้าง WO ลูกใหม่ที่มีเส้นทางเดิม โดยกำหนดเป้าหมายเฉพาะจำนวนที่ไม่ผ่าน

## Quality Dashboard

แสดงตัวชี้วัดคุณภาพรวมของ WO ทั้งหมด:

- อัตราผ่านโดยรวม
- การกระจายข้อบกพร่องตามหมวดหมู่
- การแบ่งสถานะ NCR (เปิด, กำลัง Rework, ปิด)
- ปริมาณการตรวจสอบตามเวลา

## แนวคิดหลัก

| คำศัพท์ | ความหมาย |
|---------|----------|
| Inspection | การตรวจสอบคุณภาพอย่างเป็นทางการพร้อมผลผ่าน/ไม่ผ่าน |
| NCR | Non-Conformance Report — สร้างเมื่อการตรวจสอบไม่ผ่าน |
| Rework | Reset และทำขั้นตอนที่ไม่ผ่านใหม่บน WO เดิม |
| Remake | สร้าง WO ใหม่เพื่อผลิตจำนวนที่ไม่ผ่านใหม่ |
| Disposition | การตัดสินใจที่ทำกับชิ้นส่วนที่ไม่สอดคล้อง |
