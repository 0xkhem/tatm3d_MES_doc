## การ์ดข้อมูล (Information Cards)

### การ์ดข้อมูล NCR (NCR Info Card)

| ฟิลด์ | คำอธิบาย |
|-------|-------------|
| Work Order (ใบสั่งงาน) | รหัส WO ที่เชื่อมโยง |
| Product (ผลิตภัณฑ์) | ชื่อผลิตภัณฑ์ |
| Assigned To (ผู้รับผิดชอบ) | บุคคลที่รับผิดชอบ |
| Severity (ความรุนแรง) | Critical (วิกฤต), Major (สำคัญ) หรือ Minor (เล็กน้อย) |

### การ์ดรายละเอียดข้อบกพร่อง (Defect Details Card)

| ฟิลด์ | คำอธิบาย |
|-------|-------------|
| Defect Type (ประเภทข้อบกพร่อง) | Dimensional, Surface, Material, Functional, Cosmetic |
| Quantity Affected (จำนวนที่ได้รับผลกระทบ) | จำนวนชิ้นงานที่มีข้อบกพร่อง |
| Defect Description (คำอธิบายข้อบกพร่อง) | คำอธิบายเต็มของความไม่สอดคล้อง |

### การ์ด Disposition และ Root Cause

| ฟิลด์ | คำอธิบาย |
|-------|-------------|
| Disposition (การจัดการ) | Rework, Remake, Scrap, Use As-Is, Return to Vendor |
| Root Cause (สาเหตุที่แท้จริง) | สาเหตุที่ข้อบกพร่องเกิดขึ้น (จำเป็นสำหรับ Rework/Remake) |

## การเปลี่ยนสถานะ (Status Transitions)

NCR มีวงจรชีวิตดังนี้:

```
Open → In Review → Resolved → Closed
```

### กฎการเปลี่ยนสถานะ (Transition Rules)

| จาก | ไปยัง | เงื่อนไข (Guard) |
|------|-----|-------|
| Open → In Review | ต้องกำหนด `assigned_to` (ผู้รับผิดชอบ) แล้ว |
| In Review → Resolved | ต้องกำหนด `disposition` (การจัดการ) แล้ว |
| Resolved → Closed | (ไม่มีเงื่อนไข) |

### วิธีการเปลี่ยนสถานะ

1. ที่หน้ารายละเอียด NCR ดูที่ส่วนหัว
2. ถัดจากป้ายสถานะ จะมี **ปุ่มเปลี่ยนสถานะ** แสดงสถานะถัดไปที่สามารถเปลี่ยนได้
3. คลิกปุ่ม (เช่น "→ In Review")
4. สถานะจะอัปเดตทันที
5. จะแสดงข้อความแจ้งเตือนสำเร็จ

[image: ncr-transition-buttons — ภาพใกล้ของปุ่มเปลี่ยนสถานะแสดง "→ In Review" และตัวเลือกอื่นๆ]

### ตัวอย่าง: วงจรชีวิต NCR แบบเต็ม

1. **Open** (เปิด) — NCR ถูกสร้างหลังจากการตรวจสอบไม่ผ่าน
2. คลิก **→ In Review** — ผู้รับผิดชอบเริ่มการสอบสวน
3. กำหนด Disposition (เช่น "Rework") และ Root Cause
4. คลิก **→ Resolved** — ระบบ Rework/Remake ดำเนินการโดยอัตโนมัติ
5. คลิก **→ Closed** — NCR ถูกปิดเป็นที่สิ้นสุด

## การ์ด Linked Work Order (ใบสั่งงานที่เชื่อมโยง)

หาก NCR ทำให้เกิดการ Rework หรือ Remake จะปรากฏการ์ด **Linked Work Order** แสดง:

- รหัส WO ของ Rework/Remake
- ป้ายสถานะ WO
- ป้าย **Rework ×N** (สำหรับ Disposition แบบ Rework)
- ป้าย **Remake** (สำหรับ Disposition แบบ Remake)
- ลิงก์ **View Work Order** — นำทางไปยังรายการ WO

[image: ncr-linked-wo — ภาพหน้าจอของการ์ด Linked Work Order แสดง WO ที่ถูก Rework พร้อมป้าย]

## ไทม์ไลน์สถานะ (Status Timeline)

ด้านล่าง ไทม์ไลน์แนวตั้งแสดงการเปลี่ยนสถานะทุกครั้ง:

- แต่ละรายการแสดง: ป้ายสถานะ, สถานะก่อนหน้า, เวลา และผู้ที่ทำการเปลี่ยน
- รายการล่าสุดอยู่ด้านล่าง
- เชื่อมต่อด้วยเส้นแนวตั้งพร้อมจุด

[image: ncr-status-timeline — ภาพหน้าจอของไทม์ไลน์สถานะแสดงการเปลี่ยนสถานะ 3-4 ครั้ง]

## สิ่งที่เกิดขึ้นเมื่อ NCR ถูก Resolved (แก้ไขแล้ว)

เมื่อคุณเปลี่ยนสถานะ NCR เป็น **Resolved** ระบบจะดำเนินการตาม Disposition โดยอัตโนมัติ:

| Disposition (การจัดการ) | การดำเนินการอัตโนมัติ |
|-------------|-----------------|
| **Rework** (ซ่อมแก้) | ระบบ Rework รีเซ็ตขั้นตอนที่ไม่ผ่านและขั้นตอนถัดไปเป็น pending (ดู [Rework & Remake](./07-rework-remake.md)) |
| **Remake** (ผลิตใหม่) | ระบบ Remake สร้าง Child WO ใหม่ (ดู [Rework & Remake](./07-rework-remake.md)) |
| **Scrap** (ทิ้ง) | ไม่มีการดำเนินการอัตโนมัติ — ชิ้นงานถูกทิ้ง |
| **Use As-Is** (ใช้ตามสภาพ) | ไม่มีการดำเนินการอัตโนมัติ — ชิ้นงานได้รับการยอมรับ |
| **Return to Vendor** (คืนผู้ขาย) | ไม่มีการดำเนินการอัตโนมัติ — ชิ้นงานถูกส่งคืน |

> **สำคัญ:** ตรวจสอบให้แน่ใจว่า Disposition และ Root Cause ถูกกำหนดอย่างถูกต้องก่อนทำการ Resolve เพราะระบบ Rework/Remake จะดำเนินการทันทีและไม่สามารถยกเลิกได้ง่าย
