# QC (Quality Control) — คู่มือผู้ใช้งาน

## ภาพรวม

คู่มือนี้ครอบคลุมทุกงานที่ทีม Quality Control (การควบคุมคุณภาพ) ดำเนินการในระบบ TAM3D MES ตั้งแต่การบันทึกการตรวจสอบ (Inspection) ไปจนถึงการจัดการรายงานความไม่สอดคล้อง (Non-Conformance Report หรือ NCR) และการสั่งการซ่อมแก้ (Rework) หรือผลิตใหม่ (Remake)

### การนำทาง (Navigation)

ในแถบด้านซ้าย (Sidebar) ให้ขยายเมนู **Quality** เพื่อดูรายการ:

| รายการเมนู | เส้นทาง (Path) | วัตถุประสงค์ |
|-----------|------|---------|
| Dashboard | `/quality/dashboard` | KPI ด้านคุณภาพ, กราฟ และแนวโน้ม |
| Inspections | `/quality/inspections` | บันทึกและเรียกดูผลการตรวจสอบ |
| NCR | `/quality/ncr` | จัดการรายงานความไม่สอดคล้อง |

[image: sidebar-quality-menu — ภาพหน้าจอของแถบด้านซ้ายที่แสดงเมนู "Quality" ขยายออก แสดง Dashboard, Inspections และ NCR]

### ขั้นตอนการทำงานของ QC (QC Workflow)

```
ชิ้นงานมาถึงขั้นตอน QC
      ↓
บันทึกการตรวจสอบ (Pass / Fail / Conditional Pass)
      ↓
หากไม่ผ่าน (Fail) → สร้าง NCR
      ↓
ตรวจสอบ NCR → กำหนดการจัดการ (Rework / Remake / Scrap / Use As-Is / Return to Vendor)
      ↓
แก้ไข NCR → ระบบดำเนินการตามการจัดการที่กำหนดโดยอัตโนมัติ
      ↓
ปิด NCR
```

### ข้อมูลอ้างอิงสถานะ (Status References)

**ผลการตรวจสอบ (Inspection Results):**

| ผลลัพธ์ | สี | ความหมาย |
|--------|-------|---------|
| Pass (ผ่าน) | เขียว | ชิ้นงานเป็นไปตามข้อกำหนด |
| Fail (ไม่ผ่าน) | แดง | ชิ้นงานไม่เป็นไปตามข้อกำหนด |
| Conditional Pass (ผ่านแบบมีเงื่อนไข) | เหลือง | ชิ้นงานได้รับการยอมรับโดยมีเงื่อนไข/ค่าเบี่ยงเบน |

**สถานะ NCR:**

| สถานะ | ความหมาย |
|--------|---------|
| Open (เปิด) | NCR ถูกสร้างแล้ว รอการตรวจสอบ |
| In Review (อยู่ระหว่างการตรวจสอบ) | มอบหมายให้ผู้รับผิดชอบแล้ว กำลังดำเนินการสอบสวน |
| Resolved (แก้ไขแล้ว) | ตัดสินใจและดำเนินการจัดการแล้ว (สั่ง Rework/Remake แล้ว) |
| Closed (ปิดแล้ว) | สถานะสุดท้าย — ไม่มีการดำเนินการเพิ่มเติม |
