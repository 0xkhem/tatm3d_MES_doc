# 5. NCR — สร้าง NCR

## การเข้าถึงฟอร์มสร้าง

1. ไปที่ **Quality → NCR**
2. คลิก **+ NCR**
3. ฟอร์มสร้างจะเปิดที่ `/quality/ncr/create`

หรือ: หลังจากการตรวจสอบไม่ผ่าน ให้คลิก **Create NCR** ในกล่องโต้ตอบแจ้งเตือน

[image: ncr-create-full — ภาพหน้าจอเต็มของฟอร์มสร้าง NCR]

---

## ฟิลด์ในฟอร์ม (Form Fields)

| ฟิลด์ | จำเป็น | คำอธิบาย |
|-------|----------|-------------|
| NCR Code (รหัส NCR) | อัตโนมัติ | สร้างอัตโนมัติ (อ่านอย่างเดียว, แบบ monospace) |
| Work Order (ใบสั่งงาน) | ใช่ | ดรอปดาวน์เลือก WO ที่ได้รับผลกระทบ |
| Defect Type (ประเภทข้อบกพร่อง) | ใช่ | ดรอปดาวน์: Dimensional, Surface, Material, Functional, Cosmetic |
| Defect Description (คำอธิบายข้อบกพร่อง) | ใช่ | ข้อความอิสระอธิบายรายละเอียดความไม่สอดคล้อง |
| Quantity Affected (จำนวนที่ได้รับผลกระทบ) | ใช่ | จำนวนชิ้นงานที่ได้รับผลกระทบ (จำนวนเต็มบวก) |
| Severity (ความรุนแรง) | ใช่ | ดรอปดาวน์: Critical (วิกฤต), Major (สำคัญ), Minor (เล็กน้อย) |
| Disposition (การจัดการ) | ไม่ | ดรอปดาวน์: Rework, Remake, Scrap, Use As-Is, Return to Vendor |
| Assigned To (ผู้รับผิดชอบ) | ไม่ | บุคคลที่รับผิดชอบการสอบสวน/แก้ไข |
| Root Cause (สาเหตุที่แท้จริง) | มีเงื่อนไข | จำเป็นเมื่อ Disposition เป็น Rework หรือ Remake |

[image: ncr-form-fields — ภาพใกล้ของฟอร์มแสดงฟิลด์ทั้งหมด]

## ขั้นตอนการสร้าง NCR

### ขั้นตอนที่ 1: เลือก Work Order

1. คลิกดรอปดาวน์ **Work Order**
2. เลือก WO ที่ได้รับผลกระทบ (เช่น "WO-2026-00042 — Motor Housing")

[image: ncr-select-wo — ภาพหน้าจอของดรอปดาวน์ WO]

### ขั้นตอนที่ 2: จำแนกประเภทข้อบกพร่อง

1. เลือก **Defect Type** (ประเภทข้อบกพร่อง):
   - **Dimensional** (มิติ) — ขนาดเกินค่าความคลาดเคลื่อน (ขนาด, ตำแหน่ง, เรขาคณิต)
   - **Surface** (พื้นผิว) — รอยขีดข่วน, รอยบุบ, ปัญหาความหยาบผิว
   - **Material** (วัสดุ) — วัสดุผิด, วัสดุมีข้อบกพร่อง, การปนเปื้อน
   - **Functional** (การทำงาน) — ชิ้นงานไม่ทำงานตามที่ออกแบบไว้
   - **Cosmetic** (ความสวยงาม) — ข้อบกพร่องทางสายตาที่ไม่กระทบการทำงาน

2. เขียน **Defect Description** (คำอธิบายข้อบกพร่อง) อย่างละเอียดเพื่ออธิบายสิ่งที่พบ

[image: ncr-defect-type — ภาพหน้าจอของดรอปดาวน์ Defect Type]

### ขั้นตอนที่ 3: กำหนดความรุนแรงและจำนวน

1. กรอก **Quantity Affected** — จำนวนชิ้นงานที่มีข้อบกพร่องนี้
2. เลือก **Severity** (ความรุนแรง):
   - **Critical** (วิกฤต) — มีความเสี่ยงด้านความปลอดภัยหรือการทำงานล้มเหลวทั้งหมด
   - **Major** (สำคัญ) — เบี่ยงเบนจากข้อกำหนดอย่างมีนัยสำคัญ
   - **Minor** (เล็กน้อย) — เบี่ยงเบนเล็กน้อย, ปัญหาด้านความสวยงาม

[image: ncr-severity-qty — ภาพหน้าจอของดรอปดาวน์ Severity และฟิลด์จำนวน]

### ขั้นตอนที่ 4: กำหนด Disposition (ไม่บังคับในขั้นตอนสร้าง)

Disposition (การจัดการ) กำหนดว่าจะดำเนินการอย่างไรกับชิ้นงานที่มีข้อบกพร่อง คุณสามารถกำหนดได้ตอนนี้หรือภายหลังในขั้นตอนตรวจสอบ

| Disposition (การจัดการ) | สิ่งที่เกิดขึ้นเมื่อ NCR ถูกแก้ไข (Resolved) |
|-------------|-----------------------------------|
| **Rework** (ซ่อมแก้) | ขั้นตอนที่ไม่ผ่านและขั้นตอนถัดไปจะถูกรีเซ็ตเป็น pending (รอดำเนินการ) WO ดำเนินต่อจากขั้นตอนที่ไม่ผ่าน |
| **Remake** (ผลิตใหม่) | WO เดิมจะถูกปิด และสร้าง Child WO (ใบสั่งงานลูก) ใหม่โดยคัดลอกขั้นตอนทั้งหมด |
| **Scrap** (ทิ้ง) | ชิ้นงานถูกทิ้ง ไม่มีการดำเนินการ WO อัตโนมัติ |
| **Use As-Is** (ใช้ตามสภาพ) | ชิ้นงานได้รับการยอมรับแม้จะมีข้อบกพร่อง ไม่มีการดำเนินการ WO |
| **Return to Vendor** (คืนผู้ขาย) | ชิ้นงานถูกส่งคืนผู้ขายวัสดุ ไม่มีการดำเนินการ WO |

[image: ncr-disposition — ภาพหน้าจอของดรอปดาวน์ Disposition]

### ขั้นตอนที่ 5: Root Cause (จำเป็นสำหรับ Rework/Remake)

หาก Disposition เป็น **Rework** หรือ **Remake** ฟิลด์ Root Cause (สาเหตุที่แท้จริง) จะกลายเป็นฟิลด์บังคับ ให้อธิบายว่าข้อบกพร่องเกิดขึ้นเพราะอะไร

### ขั้นตอนที่ 6: บันทึก

1. กำหนด **Assigned To** (ผู้รับผิดชอบ) หากต้องการ (ผู้ที่จะสอบสวน)
2. คลิก **Save** (บันทึก)
3. NCR จะถูกสร้างในสถานะ **Open** (เปิด)
4. จะแสดงข้อความแจ้งเตือนสำเร็จ

[image: ncr-save-success — ภาพหน้าจอแสดงข้อความแจ้งเตือนสำเร็จ]
