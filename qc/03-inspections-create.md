# 3. Inspections — สร้างการตรวจสอบ

## การเข้าถึงฟอร์มสร้าง

1. ไปที่ **Quality → Inspections**
2. คลิก **+ Inspection**
3. ฟอร์มสร้างจะเปิดที่ `/quality/inspections/create`

[image: inspection-create-full — ภาพหน้าจอเต็มของฟอร์มสร้าง Inspection]

---

## ฟิลด์ในฟอร์ม (Form Fields)

| ฟิลด์ | จำเป็น | คำอธิบาย |
|-------|----------|-------------|
| Inspection Code (รหัสตรวจสอบ) | อัตโนมัติ | สร้างอัตโนมัติ (อ่านอย่างเดียว, แบบ monospace) ใช้ระบบเลขรันนิ่งนัมเบอร์ |
| Work Order (ใบสั่งงาน) | ใช่ | ดรอปดาวน์เลือก WO ที่ต้องการตรวจสอบ |
| Process Step (ขั้นตอนกระบวนการ) | ไม่ | ดรอปดาวน์แสดงขั้นตอนจาก WO ที่เลือก เลือกขั้นตอนที่ต้องการตรวจสอบ |
| Sample Qty (จำนวนตัวอย่าง) | ไม่ | จำนวนชิ้นงานที่สุ่มตัวอย่าง |
| Inspected Qty (จำนวนที่ตรวจสอบ) | ไม่ | จำนวนชิ้นงานที่ตรวจสอบจริง |
| Result (ผลลัพธ์) | ใช่ | ดรอปดาวน์: Pass, Fail, Conditional Pass |
| Defect Count (จำนวนข้อบกพร่อง) | มีเงื่อนไข | จำนวนข้อบกพร่องที่พบ จำเป็นเมื่อผลลัพธ์เป็น Fail หรือ Conditional Pass |
| Defect Reason (สาเหตุข้อบกพร่อง) | ไม่ | ข้อความอิสระอธิบายข้อบกพร่อง |
| Inspector (ผู้ตรวจสอบ) | ไม่ | ชื่อผู้ตรวจสอบ |
| Notes (หมายเหตุ) | ไม่ | หมายเหตุเพิ่มเติม |

[image: inspection-form-fields — ภาพใกล้ของฟอร์มแสดงฟิลด์ทั้งหมดพร้อมข้อมูลตัวอย่าง]

## ขั้นตอนการบันทึกการตรวจสอบ

### ขั้นตอนที่ 1: เลือก Work Order

1. คลิกดรอปดาวน์ **Work Order**
2. เลือก WO ที่ต้องการตรวจสอบ (เช่น "WO-2026-00042 — Motor Housing")
3. ดรอปดาวน์ Process Step จะแสดงขั้นตอนของ WO นั้นโดยอัตโนมัติ

[image: inspection-select-wo — ภาพหน้าจอของดรอปดาวน์ WO พร้อมตัวเลือก]

### ขั้นตอนที่ 2: เลือก Process Step (ไม่บังคับ)

1. คลิกดรอปดาวน์ **Process Step**
2. เลือกขั้นตอนที่ได้ทำการตรวจสอบ (เช่น "5. Quality Check")

[image: inspection-select-step — ภาพหน้าจอของดรอปดาวน์ Process Step]

### ขั้นตอนที่ 3: กรอกจำนวน

1. กรอก **Sample Qty** — จำนวนชิ้นงานที่สุ่มตัวอย่าง
2. กรอก **Inspected Qty** — จำนวนชิ้นงานที่ตรวจสอบจริง

### ขั้นตอนที่ 4: บันทึกผลลัพธ์

1. คลิกดรอปดาวน์ **Result**
2. เลือกอย่างใดอย่างหนึ่ง:
   - **Pass** — ชิ้นงานทั้งหมดเป็นไปตามข้อกำหนด
   - **Fail** — ชิ้นงานไม่เป็นไปตามข้อกำหนด
   - **Conditional Pass** — ชิ้นงานได้รับการยอมรับโดยมีค่าเบี่ยงเบนที่บันทึกไว้

[image: inspection-select-result — ภาพหน้าจอของดรอปดาวน์ Result]

### ขั้นตอนที่ 5: บันทึกข้อบกพร่อง (กรณี Fail หรือ Conditional)

1. กรอก **Defect Count** (จำนวนข้อบกพร่อง)
2. กรอก **Defect Reason** (สาเหตุข้อบกพร่อง) (เช่น "รอยขีดข่วนบนผิวหน้า A", "ขนาดเกินค่าความคลาดเคลื่อน ±0.02mm")

### ขั้นตอนที่ 6: บันทึก

1. กรอกชื่อของคุณในฟิลด์ **Inspector**
2. เพิ่ม **Notes** (หมายเหตุ) หากต้องการ
3. คลิก **Save** (บันทึก)
4. จะแสดงข้อความแจ้งเตือนสำเร็จ: "Inspection created successfully."

[image: inspection-save-success — ภาพหน้าจอแสดงข้อความแจ้งเตือนสำเร็จ]

## การแจ้งเตือนสร้าง NCR (เมื่อผลลัพธ์เป็น Fail)

เมื่อคุณบันทึกการตรวจสอบที่มีผลลัพธ์ = **Fail** จะปรากฏกล่องโต้ตอบ:

> "การตรวจสอบนี้ไม่ผ่าน คุณต้องการสร้างรายงานความไม่สอดคล้อง (NCR) หรือไม่?"

| ปุ่ม | การดำเนินการ |
|--------|--------|
| **Create NCR** (สร้าง NCR) | นำทางไปยังฟอร์มสร้าง NCR |
| **Skip** (ข้าม) | ปิดกล่องโต้ตอบ — คุณสามารถสร้าง NCR ภายหลังได้ |

[image: inspection-ncr-prompt — ภาพหน้าจอของกล่องโต้ตอบแจ้งเตือน NCR หลังจากการตรวจสอบไม่ผ่าน]

นี่คือขั้นตอนการทำงานที่แนะนำ: การตรวจสอบไม่ผ่าน (Fail) → สร้าง NCR ทันทีขณะที่บริบทยังสด
