# 7. บอร์ดจัดตาราง — เครื่องมือจัดตารางอัตโนมัติ

## ภาพรวม

เครื่องมือจัดตารางอัตโนมัติสร้างตารางการผลิตที่ปรับให้เหมาะสมโดยการกำหนดการดำเนินงานที่ยังไม่ได้จัดตารางให้กับเครื่องจักรที่มีอยู่ ผู้วางแผนตั้งค่าพารามิเตอร์, ตรวจสอบตารางที่เสนอ และตัดสินใจยอมรับหรือยกเลิก

```
การดำเนินงานที่ยังไม่ได้จัดตาราง + เครื่องจักร + ปฏิทิน + กฎ
    ↓
เครื่องมือทำงาน (การคำนวณล้วนๆ)
    ↓
การกำหนดที่เสนอ (Ghost Overlay บน Timeline)
    ↓
ผู้วางแผนตรวจสอบ KPI → ยอมรับหรือยกเลิก
```

### ขั้นตอนที่ 1: เปิดกล่องโต้ตอบ Generate

1. บนบอร์ดจัดตาราง คลิก **⚡ Generate Schedule** ในแถบเครื่องมือ
2. กล่องโต้ตอบ Generate Schedule จะเปิดขึ้น

[image: auto-schedule-dialog — ภาพหน้าจอของกล่องโต้ตอบ Generate Schedule พร้อมฟิลด์การตั้งค่าทั้งหมด]

### ขั้นตอนที่ 2: ตั้งค่าพารามิเตอร์

| พารามิเตอร์ | คำอธิบาย | ค่าเริ่มต้น |
|-----------|-------------|---------|
| Horizon Start | วันแรกที่จะจัดตารางจาก | วันนี้ |
| Horizon End | วันสุดท้ายที่จะจัดตารางถึง | 2 สัปดาห์จากวันนี้ |
| Dispatching Rule | อัลกอริทึมสำหรับการเรียงลำดับการดำเนินงาน (ดูด้านล่าง) | EDD |
| Secondary Rule | ตัวตัดสินสำหรับ Setup-Aware Grouping เท่านั้น | — |
| Reschedule Mode | รวมการดำเนินงานที่จัดตารางแล้วในการปรับให้เหมาะสม | ปิด |

[image: auto-schedule-params — ภาพใกล้ของฟิลด์พารามิเตอร์ในกล่องโต้ตอบ]

### ขั้นตอนที่ 3: รันเครื่องมือ

1. คลิก **Generate**
2. ปุ่มแสดง Spinner ขณะที่เครื่องมือคำนวณ
3. การกำหนดที่เสนอปรากฏเป็น **Ghost Bar** (กึ่งโปร่งใส) บน Timeline

[image: auto-schedule-ghost-bars — ภาพหน้าจอของ Timeline แสดงแถบการดำเนินงาน Ghost/ที่เสนอซ้อนทับบนตารางที่มีอยู่]

### ขั้นตอนที่ 4: ตรวจสอบ KPI

**แผง KPI** ปรากฏด้านล่างแถบเครื่องมือแสดงตัวชี้วัดสถานการณ์:

| KPI | คำอธิบาย |
|-----|-------------|
| Makespan | วันปฏิทินทั้งหมดจากการกำหนดแรกถึงสุดท้าย |
| Late Orders | จำนวน WO ที่จัดตารางให้เสร็จหลังวันครบกำหนด |
| Utilization | กำลังการผลิตเครื่องจักรเฉลี่ยที่ใช้ (%) |
| Setup Time | ชั่วโมงเปลี่ยนงานทั้งหมด |
| Tardiness | ชั่วโมงความล่าช้าทั้งหมดในทุกคำสั่งซื้อที่ล่าช้า |

แผงยังแสดงกฎ Dispatching ที่ใช้ (เช่น "EDD", "Slack")

[image: auto-schedule-kpi-panel — ภาพหน้าจอของแผง KPI แสดงตัวชี้วัดทั้งห้า]

## กฎ Dispatching

### EDD — Earliest Due Date (วันครบกำหนดเร็วที่สุด)

จัดตารางการดำเนินงานที่มีวันครบกำหนดเร็วที่สุดก่อน

- เหมาะสำหรับ: เน้นการส่งมอบตรงเวลา
- ข้อเสีย: การเปลี่ยนงานเครื่องจักรมากขึ้น

### Priority-Weighted EDD

จัดตารางตามลำดับความสำคัญก่อน (urgent > high > normal > low) จากนั้นตามวันครบกำหนดในแต่ละระดับ

- เหมาะสำหรับ: คำสั่งซื้อด่วนและลูกค้า VIP
- ข้อเสีย: งานที่มีลำดับความสำคัญต่ำอาจล่าช้า

### SPT — Shortest Processing Time (เวลาประมวลผลสั้นที่สุด)

จัดตารางงานสั้นที่สุดก่อนเพื่อเพิ่มปริมาณงาน

- เหมาะสำหรับ: ล้างงานค้างจำนวนมากอย่างรวดเร็ว
- ข้อเสีย: งานยาวถูกผลักไปท้าย

### LPT — Longest Processing Time (เวลาประมวลผลยาวที่สุด)

จัดตารางงานยาวที่สุดก่อนเพื่อลด Makespan รวม

- เหมาะสำหรับ: ลดช่วงตารางโดยรวม
- ข้อเสีย: งานเล็กรอนานขึ้น

### Slack (Critical Ratio)

จัดตารางตามความตึงตัว: `slack = (เวลาจนถึงครบกำหนด) − (ชั่วโมงงานที่เหลือ)` Slack ติดลบ (ล่าช้าแล้ว) ไปก่อน

- เหมาะสำหรับ: เพิ่ม OTIF (On-Time In-Full) สูงสุด — ค่าเริ่มต้นที่ฉลาดที่สุด
- ข้อเสีย: ต้องการวันครบกำหนดและการประมาณที่แม่นยำ

### Setup-Aware Grouping

จัดกลุ่มการดำเนินงานตามกลุ่ม Setup (จากคำจำกัดความกระบวนการ) เพื่อลดเวลาเปลี่ยนงาน ใช้กฎรองภายในแต่ละกลุ่ม

- เหมาะสำหรับ: โรงงานที่มีต้นทุนการเปลี่ยนงานสูง
- ข้อเสีย: อาจล่าช้างานด่วนในกลุ่ม Setup ต่างกัน

[image: auto-schedule-rule-selection — ภาพหน้าจอของ Dropdown กฎ Dispatching แสดงตัวเลือกทั้งหมด]

## การเลือกกฎ

| สถานการณ์ | กฎที่แนะนำ |
|----------|-----------------|
| วันส่งมอบลูกค้าสำคัญมาก | Slack |
| มีคำสั่งซื้อด่วนและมาตรฐานผสมกัน | Priority-Weighted EDD |
| งานเล็กจำนวนมาก ต้องการล้างงานค้าง | SPT |
| งานใหญ่น้อยชิ้นที่ครอบงำตาราง | LPT |
| การเปลี่ยนงานบ่อยกินกำลังการผลิต | Setup-Aware Grouping |
| การจัดตารางตามกำหนดเวลาแบบง่าย | EDD |
| ไม่แน่ใจว่าจะใช้อะไร | เริ่มด้วย **Slack** |

> **เคล็ดลับ:** คุณสามารถสร้างหลายสถานการณ์ด้วยกฎต่างกันและเปรียบเทียบ KPI ก่อนยืนยัน ยกเลิกหนึ่ง ลองอีกอัน

## โหมดจัดตารางใหม่ (Reschedule Mode)

เมื่อ **Reschedule Mode** เปิด:

- การดำเนินงานที่จัดตารางแล้วรวมอยู่ในการปรับให้เหมาะสม
- การกำหนดภายใน **โซนแช่แข็ง** (วันนี้ + พารามิเตอร์ `frozen_days`, ค่าเริ่มต้น 2 วัน) ถูกล็อคและจะไม่ถูกย้าย
- ทุกอย่างนอกโซนแช่แข็งสามารถจัดเรียงใหม่ได้

ใช้เมื่อ:
- คำสั่งซื้อด่วนใหม่มาถึงและคุณต้องปรับให้เหมาะสมใหม่
- เครื่องจักรหยุดทำงานและคุณต้องกระจายงาน
- คุณต้องการปรับปรุงตารางที่มีอยู่โดยไม่รบกวนความมุ่งมั่นระยะใกล้

[image: auto-schedule-reschedule-mode — ภาพหน้าจอของ Toggle Reschedule Mode ในกล่องโต้ตอบ]

## วิธีที่เครื่องมือจัดการ Outside Processing

ขั้นตอน OPO **ไม่ได้กำหนดให้เครื่องจักร** แต่เครื่องมือถือว่าเป็นช่องว่างเวลา:

```
ขั้นตอนในบ้าน A → [ช่องว่าง OPO: จัดส่ง + ระยะเวลานำ] → ขั้นตอนในบ้าน B
```

1. เครื่องมือค้นหาขั้นตอนในบ้านสุดท้ายก่อนห่วงโซ่ OPO
2. วันจัดส่ง = วันทำงานถัดไปหลังจากขั้นตอนนั้นสิ้นสุด
3. วันกลับ = วันจัดส่ง + ระยะเวลานำซัพ (วันทำงาน)
4. ขั้นตอนในบ้านถัดไปไม่สามารถเริ่มก่อนวันกลับ

สิ่งนี้ทำให้ตารางเคารพระยะเวลานำซัพโดยอัตโนมัติ

## พารามิเตอร์ตาราง

ตั้งค่าใน **Knowledge → Production Schedule Modeling → Schedule Parameters**:

| พารามิเตอร์ | ค่าเริ่มต้น | คำอธิบาย |
|-----------|---------|-------------|
| `default_setup_minutes` | 30 | เวลาเปลี่ยนงานสำรองระหว่างกลุ่ม Setup ต่างกัน |
| `frozen_days` | 2 | วันนับจากวันนี้ที่ถูกล็อคในโหมดจัดตารางใหม่ |
