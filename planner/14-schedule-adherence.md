# 14. รายงานความสอดคล้องตาราง (Schedule Adherence Report)

## การเข้าถึงรายงาน

ไปที่ **Production → Schedule Adherence** หรือเปิด `/production/schedule-adherence`

[image: adherence-page — ภาพหน้าจอเต็มของหน้า Schedule Adherence แสดงการ์ด KPI, แผนภูมิการกระจาย Delta และตารางรายละเอียด]

---

## วัตถุประสงค์

รายงานความสอดคล้องตารางเปรียบเทียบตารางที่วางแผนกับการดำเนินงานจริง ตอบคำถาม: "เราปฏิบัติตามแผนได้ดีแค่ไหน?" ช่วยให้ผู้วางแผนระบุปัญหาเชิงระบบ — เครื่องจักรที่เริ่มช้าอย่างสม่ำเสมอ, การดำเนินงานที่เกินเวลาเสมอ หรือกะที่มีความสอดคล้องต่ำ

---

## ช่วงวันที่ที่ตั้งไว้ล่วงหน้า

เลือกช่วงเวลาโดยใช้ปุ่มที่ตั้งไว้ล่วงหน้า:

| ที่ตั้งไว้ล่วงหน้า | ช่วง |
|--------|-------|
| This Week | จันทร์ถึงอาทิตย์ของสัปดาห์ปัจจุบัน |
| Last Week | จันทร์ถึงอาทิตย์ของสัปดาห์ที่แล้ว |
| This Month | วันที่ 1 ถึงวันสุดท้ายของเดือนปัจจุบัน |
| Last 30 Days | วันนี้ลบ 29 วันถึงวันนี้ |

[image: adherence-date-presets — ภาพใกล้ของปุ่มที่ตั้งไว้ล่วงหน้าสำหรับช่วงวันที่]

---

## การ์ด KPI

ตัวชี้วัดหลักสี่รายการด้านบน:

| KPI | ไอคอน | คำอธิบาย |
|-----|------|-------------|
| Adherence Rate | 🎯 | เปอร์เซ็นต์ของการดำเนินงานที่เริ่มภายในระยะทนของเวลาที่วางแผน |
| Avg Start Delta | ⏱ | ความแตกต่างเฉลี่ยระหว่างเวลาเริ่มที่วางแผนและจริง (เป็นชั่วโมง) |
| On-Time Starts | ✅ | จำนวนการดำเนินงานที่เริ่มตรงเวลา |
| Late Starts | ⚠ | จำนวนการดำเนินงานที่เริ่มช้า (การเตือนสีแดงถ้าสูง) |

[image: adherence-kpi-cards — ภาพใกล้ของการ์ด KPI สี่ใบพร้อมค่าตัวอย่าง]

---

## แผนภูมิการกระจาย Start Delta

แผนภูมิแท่งแสดงการกระจายของ Delta เวลาเริ่ม:

- **แกน X:** กลุ่ม Delta (เช่น "On time", "0–1h late", "1–2h late", "2–4h late", ">4h late", "Early")
- **แกน Y:** จำนวนการดำเนินงาน
- **สี:** เขียวสำหรับตรงเวลา, อำพันสำหรับล่าช้าเล็กน้อย, แดงสำหรับล่าช้ามาก

แผนภูมินี้เผยรูปแบบ — ถ้าการดำเนินงานส่วนใหญ่รวมกันในกลุ่ม "1–2h late" อาจมีความล่าช้าเชิงระบบ (เช่น ปัญหาการส่งมอบกะ)

[image: adherence-delta-chart — ภาพหน้าจอของแผนภูมิแท่งการกระจาย Start Delta]

---

## ตารางรายละเอียด

ตารางที่เรียงลำดับได้แสดงทุกการดำเนินงานที่จัดตารางพร้อมข้อมูลแผนกับจริง:

| คอลัมน์ | คำอธิบาย |
|--------|-------------|
| WO Code | รหัส Work Order |
| Step | ชื่อขั้นตอนกระบวนการ |
| Machine | เครื่องจักรที่กำหนด |
| Planned Start | วันที่และชั่วโมงที่จัดตาราง |
| Actual Start | เมื่อขั้นตอนเริ่มจริง |
| Start Delta | ความแตกต่างเป็นชั่วโมง (บวก = ช้า, ลบ = เร็ว) |
| Planned Duration | ชั่วโมงที่ประมาณ |
| Actual Duration | ชั่วโมงที่ทำงานจริง |
| Duration Delta | ความแตกต่าง (บวก = เกินเวลา, ลบ = เสร็จเร็ว) |

### การเรียงลำดับ

คลิกส่วนหัวคอลัมน์ใดก็ได้เพื่อเรียงลำดับ คลิกอีกครั้งเพื่อกลับ ตัวบ่งชี้การเรียงลำดับ (▲/▼) แสดงทิศทางการเรียงลำดับปัจจุบัน

### สีเซลล์ Delta

| ค่า | สี |
|-------|-------|
| ลบ (เร็ว) | ข้อความเขียวพร้อมคำนำหน้า "−" |
| ศูนย์ | ข้อความปกติ |
| บวก (ช้า/เกินเวลา) | ข้อความแดงพร้อมคำนำหน้า "+" |
| ไม่มีข้อมูล | เส้นประสีเทา "—" |

[image: adherence-detail-table — ภาพหน้าจอของตารางรายละเอียดพร้อม 5–6 แถวแสดงค่า Delta ต่างกันในสี]

## การค้นหา

ใช้ช่องค้นหาเพื่อกรองตามรหัส WO, ชื่อขั้นตอน หรือชื่อเครื่องจักร

[image: adherence-search — ภาพหน้าจอของช่องค้นหาพร้อมผลลัพธ์ที่กรอง]

---

## การส่งออก

คลิกปุ่ม **Export** (⬇) เพื่อดาวน์โหลดมุมมองปัจจุบันเป็นไฟล์ CSV มีประโยชน์สำหรับการวิเคราะห์เพิ่มเติมในสเปรดชีต

[image: adherence-export — ภาพหน้าจอแสดงปุ่ม Export]

## กรณีการใช้งานสำหรับผู้วางแผน

| สถานการณ์ | สิ่งที่ต้องมองหา |
|----------|-----------------|
| "เราปฏิบัติตามตารางไหม?" | ตรวจสอบ KPI Adherence Rate — เป้าหมายคือ >85% |
| "การดำเนินงานไหนล่าช้าอย่างสม่ำเสมอ?" | เรียงลำดับตารางตาม Start Delta จากมากไปน้อย |
| "เครื่องจักรเฉพาะทำให้เกิดความล่าช้าไหม?" | ค้นหาตามชื่อเครื่องจักรและตรวจสอบรูปแบบ Delta |
| "การประมาณเวลาของเราแม่นยำไหม?" | ตรวจสอบ Duration Delta — การเกินเวลาอย่างสม่ำเสมอหมายความว่าการประมาณต่ำเกินไป |
| "กะไหนมีความสอดคล้องดีกว่า?" | กรองตามช่วงวันที่และเปรียบเทียบการดำเนินงานช่วงเช้ากับบ่าย |
| "ฉันควรปรับตารางไหม?" | ถ้าความสอดคล้องต่ำ พิจารณารันจัดตารางอัตโนมัติใหม่พร้อมการประมาณที่อัปเดต |
