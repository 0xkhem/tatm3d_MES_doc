# คู่มือการเพิ่มประสิทธิภาพเวลา Setup

เวลา Setup คือตัวทำลายประสิทธิภาพที่มองไม่เห็นในโรงงาน HMLV เครื่องจักรที่ใช้เวลา 45 นาทีในการเปลี่ยน Setup ระหว่างงานสูญเสียเวลามากกว่า 6 ชั่วโมงต่อสัปดาห์ไปกับเวลาที่ไม่ได้ตัดชิ้นงาน — ก่อนที่เศษโลหะชิ้นแรกจะหลุดออกมา คู่มือนี้อธิบายวิธีใช้ Scheduling Engine ของ TAM3D เพื่อลดการสูญเสียนั้น

---

## ทำไมเวลา Setup ถึงสำคัญกว่าใน HMLV

ในการผลิต High-Volume เวลา Setup ถูกกระจายไปยังชิ้นงานหลายพัน ชิ้น ใน HMLV คุณอาจผลิตชิ้นงาน 5 ชิ้นแล้วเปลี่ยนไปทำอะไรที่แตกต่างกันโดยสิ้นเชิง การเปลี่ยน Setup ทุกครั้งคือต้นทุนจริง:

- การเปลี่ยน Fixture และการ Tram ใหม่
- การเปลี่ยนเครื่องมือและการป้อน Offset
- การโหลดโปรแกรมและการ Dry-run ตรวจสอบ
- การตรวจสอบชิ้นแรกก่อนรัน Batch

โรงงานที่รัน 8 งานต่างกันต่อเครื่องต่อสัปดาห์ที่ 30 นาทีต่อครั้ง สูญเสีย 4 ชั่วโมง — นั่นคือครึ่งกะ — ไปกับ Setup เพียงอย่างเดียว

---

## วิธีที่ระบบจำลองเวลา Setup

### Setup Groups

ทุก Process Definition สามารถมี `setup_group` — ป้ายกำกับที่บอก Scheduler ว่างานใดใช้ Fixture, Tooling หรือตระกูลโปรแกรมเดียวกัน

ตัวอย่าง:
```
Aluminum-Fixture-A    → ชิ้นงาน Bracket อลูมิเนียมทั้งหมดที่ใช้ Vise Setup เดียวกัน
Steel-Turning         → งาน Shaft เหล็กทั้งหมดบนเครื่อง Lathe
Titanium-5Axis        → ชิ้นงาน Aerospace ไทเทเนียมทั้งหมดบนเครื่อง 5 แกน
Manual-Deburr         → การ Deburring ด้วยมือทั้งหมด (ไม่มีการเปลี่ยน Setup)
```

เมื่องานสองงานติดกันบนเครื่องเดียวกันมี `setup_group` เดียวกัน Scheduler จะถือว่าเวลาเปลี่ยน Setup เป็น **0 นาที** — ช่างเพียงแค่โหลดชิ้นงานถัดไป

### Setup Time Matrix

สำหรับงานที่มี Setup Group *ต่างกัน* คุณกำหนดเวลาเปลี่ยน Setup ระหว่างแต่ละคู่:

```
Aluminum-Fixture-A → Steel-Turning:     45 นาที
Steel-Turning → Aluminum-Fixture-A:     45 นาที
Aluminum-Fixture-A → Aluminum-Housing:  15 นาที
```

ตั้งค่าได้ที่: **Knowledge → Production Schedule Modeling → Schedule Parameters**

หากคู่ใดไม่อยู่ใน Matrix ระบบจะใช้ `default_setup_minutes` เป็น Fallback (ค่าเริ่มต้น: 30 นาที)

## การเลือก Dispatching Rule ที่เหมาะสม

### เมื่อไหรควรใช้ Setup-Aware Grouping

ใช้ Rule นี้เมื่อ:
- โรงงานของคุณมีต้นทุนการเปลี่ยน Setup สูง (>20 นาทีต่อครั้ง)
- มีงานหลายชิ้นในตระกูลชิ้นงานเดียวกันรันพร้อมกัน
- การใช้งานเครื่องจักรเป็นข้อจำกัดหลัก (ไม่ใช่วันส่งมอบ)

**วิธีการทำงาน:** Engine จัดกลุ่มการดำเนินงานทั้งหมดตาม `setup_group` แล้วจัดตารางแต่ละกลุ่มติดกันบนเครื่องเดียวกัน ภายในแต่ละกลุ่มจะใช้ Rule รอง (EDD, SPT หรือ Priority-Weighted EDD) เพื่อเรียงลำดับงาน

**ตัวอย่าง:** มีงาน Bracket อลูมิเนียม 6 งานและงาน Shaft เหล็ก 4 งาน หากไม่มีการจัดกลุ่ม Scheduler อาจสลับระหว่างงานทั้งสอง — 10 ครั้งที่ 45 นาที = สูญเสีย 7.5 ชั่วโมง ด้วย Setup-Aware Grouping จะรันงานอลูมิเนียมทั้ง 6 งานก่อน (0 ครั้งภายในกลุ่ม) แล้วจึงรันงานเหล็กทั้ง 4 งาน (1 ครั้งระหว่างกลุ่ม) = เปลี่ยน Setup เพียง 1 ครั้งที่ 45 นาที

**ข้อแลกเปลี่ยน:** งานบางงานอาจรอนานขึ้นหากอยู่ใน Setup Group ต่างจากที่กำลังรันอยู่ หากงาน Shaft เหล็กมีกำหนดส่งมอบที่ตึงแต่เครื่องกำลังรันงานอลูมิเนียมอยู่ งานนั้นจะถูกเลื่อนออกไป

### เมื่อไหรควรใช้ EDD หรือ Slack แทน

หากวันส่งมอบตึงกว่าต้นทุน Setup ให้เปลี่ยนเป็น:
- **Slack (Critical Ratio)** — ค่าเริ่มต้นที่ดีที่สุดสำหรับโรงงานที่เน้น OTIF พิจารณาทั้งความใกล้ชิดของ Deadline และขนาดงาน
- **EDD** — ง่ายกว่า เรียงตาม Deadline ก่อน

Rule เหล่านี้ไม่จัดกลุ่มตาม Setup ดังนั้นต้นทุนการเปลี่ยน Setup จะสูงกว่า — แต่ไม่มีงานใดถูกเลื่อนเพราะ Setup Group ของมัน

### ตารางเลือก Rule

| สถานการณ์ | ใช้ Rule นี้ |
|-----------|------------|
| งานคล้ายกันหลายชิ้น การเปลี่ยน Setup แพง | Setup-Aware Grouping |
| วันส่งมอบตึงทุกงาน | Slack (Critical Ratio) |
| มีทั้งงานด่วนและงานปกติ | Priority-Weighted EDD |
| ต้องการล้าง Backlog เร็ว | SPT |
| ไม่แน่ใจ | เริ่มด้วย Slack แล้วเปรียบเทียบ KPI |

## ขั้นตอน: การเพิ่มประสิทธิภาพตารางการผลิตสำหรับ Setup

### 1. ตั้งค่า Setup Groups บน Process Definitions

ไปที่ **Knowledge → Process Definitions** สำหรับแต่ละ Definition ตั้งค่าฟิลด์ `setup_group`:

| Process Definition | Setup Group |
|-------------------|-------------|
| CNC Milling (อลูมิเนียม) | `Aluminum-Fixture-A` |
| CNC Milling (เหล็ก) | `Steel-Fixture-B` |
| CNC Turning | `Steel-Turning` |
| Deburring | `Manual-Deburr` |
| CMM Inspection | `QC-Station` |

### 2. ตั้งค่า Setup Time Matrix

ไปที่ **Knowledge → Production Schedule Modeling → Schedule Parameters**

ตั้งค่า `default_setup_minutes` (เช่น 30) และเพิ่มคู่เฉพาะที่คุณทราบเวลาเปลี่ยน Setup จริง

### 3. สร้างสองสถานการณ์และเปรียบเทียบ

เปิด **Scheduling Board** → คลิก ⚡ Generate Schedule

**สถานการณ์ A:** รันด้วย Slack (Critical Ratio)
- บันทึก KPI: Total Setup Hours, Late Orders, Makespan

**สถานการณ์ B:** รันด้วย Setup-Aware Grouping + EDD รอง
- เปรียบเทียบ: Total Setup Hours ควรต่ำกว่า; Late Orders อาจสูงกว่า

หากสถานการณ์ B ประหยัด Setup ได้ 3 ชั่วโมงแต่ทำให้มี Late Order เพิ่ม 1 รายการ คุณมีการแลกเปลี่ยนที่ต้องประเมิน หากประหยัด Setup โดยไม่มี Late Order เพิ่ม ให้ยอมรับ

### 4. ยอมรับสถานการณ์ที่ดีกว่า

คลิก **Accept** เพื่อ Commit การมอบหมายที่เสนอไปยังฐานข้อมูล Timeline Grid จะอัปเดตทันที

## ตัวอย่างจริง: โรงงาน Motor Housing

**สถานการณ์:** มี WO ที่ Active 12 รายการบนเครื่อง CNC Mill 5 แกน ผสมระหว่าง Housing อลูมิเนียม (8 งาน) และ Bracket ไทเทเนียม (4 งาน) การเปลี่ยนระหว่างตระกูลวัสดุแต่ละครั้งใช้เวลา 60 นาที

**ไม่มีการจัดกลุ่ม (EDD):**
- 11 ครั้งที่สลับระหว่างตระกูล × 60 นาที = สูญเสีย 11 ชั่วโมง
- งานทั้งหมดจัดตารางตาม Deadline — Late Order 2 รายการ

**ด้วย Setup-Aware Grouping + EDD รอง:**
- 1 ครั้ง (กลุ่มอลูมิเนียม → กลุ่มไทเทเนียม) × 60 นาที = สูญเสีย 1 ชั่วโมง
- งานภายในแต่ละกลุ่มเรียงตาม Deadline
- กู้คืนเวลา Setup 10 ชั่วโมง
- Late Order เท่าเดิม 2 รายการ (ข้อจำกัด Deadline ไม่เปลี่ยน)

**ผลลัพธ์สุทธิ:** เวลาตัดชิ้นงานที่มีประสิทธิผลเพิ่มขึ้น 10 ชั่วโมงต่อสัปดาห์

## การติดตามประสิทธิภาพ Setup

หลังจากรันการผลิต ใช้ **Schedule Adherence Report** (Production → Dashboard → Schedule Adherence) เพื่อเปรียบเทียบเวลาที่วางแผนกับเวลาจริง หากเวลา Setup จริงเกินกว่าที่วางแผนอย่างสม่ำเสมอ ให้อัปเดต Setup Matrix ด้วยข้อมูลจริง

Metric สำคัญที่ต้องดู: **Total Setup Hours** ในแผง KPI หากตัวเลขนี้เพิ่มขึ้นทุกสัปดาห์ Setup Group ของคุณอาจต้องปรับปรุง หรือมีตระกูลงานใหม่ที่ถูกเพิ่มเข้ามาโดยไม่มีการจัดกลุ่มที่เหมาะสม

## ข้อผิดพลาดที่พบบ่อย

**ไม่กำหนด Setup Groups เลย** ทุกการดำเนินงานถูกถือว่าไม่ซ้ำกัน ดังนั้นทุกงานติดกันจะเสีย `default_setup_minutes` คุณสูญเสียประโยชน์ของการจัดกลุ่มทั้งหมด

**Setup Groups มากเกินไป** หากทุก Process Definition มี Group ไม่ซ้ำกัน การจัดกลุ่มไม่มีประโยชน์ ตั้งเป้าที่ 5–10 กลุ่มที่สะท้อนตระกูล Fixture/Tooling จริง

**ละเลย Secondary Rule** ภายใน Setup Group งานยังต้องเรียงลำดับ ใช้ EDD เป็น Secondary Rule เพื่อให้งานเร่งด่วนที่สุดภายในแต่ละกลุ่มรันก่อน

**รัน Setup-Aware Grouping เมื่อ Deadline วิกฤต** หากมีงาน 3 งานที่ครบกำหนดพรุ่งนี้และอยู่ใน Setup Group ต่างกัน การจัดกลุ่มจะเลื่อนบางงานออกไป เปลี่ยนเป็น Slack หรือ EDD เมื่อแรงกดดันการส่งมอบสูง
