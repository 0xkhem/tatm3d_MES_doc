# 13. การติดตามย้อนกลับคำสั่งซื้อ (Order Traceability)

## การเข้าถึงหน้าติดตามย้อนกลับ

ไปที่ **Production → Traceability** หรือเปิด `/production/traceability`

[image: traceability-page — ภาพหน้าจอเต็มของหน้า Order Traceability แสดงช่องค้นหาและคำสั่งซื้อที่ติดตามพร้อมห่วงโซ่การไหล, ขั้นตอน WO และ Timeline]

---

## วัตถุประสงค์

เมื่อลูกค้าถามว่า "คำสั่งซื้อของฉันอยู่ที่ไหน?" หน้าติดตามย้อนกลับตอบได้ในมุมมองเดียว แสดงวงจรชีวิตทั้งหมดของคำสั่งซื้อตั้งแต่ใบเสนอราคาจนถึงการจัดส่ง

---

## การค้นหา

ช่องค้นหาด้านบนรับ:
- **รหัส SO** (เช่น `SO-2026-008`)
- **รหัส WO** (เช่น `WO-2026-00042`)
- **รหัส Quote** (เช่น `QT-2026-015`)
- **ชื่อลูกค้า** (เช่น `XPeng Motors`)
- **ชื่อผลิตภัณฑ์** (เช่น `Motor Housing`)

พิมพ์คำค้นหาและกด Enter ระบบค้นหา Entity ที่ตรงกันและติดตามห่วงโซ่ทั้งหมด

[image: traceability-search — ภาพใกล้ของช่องค้นหาพร้อมคำค้นหาที่ป้อน]

---

## ห่วงโซ่การไหล (Flow Chain)

ส่วนบนแสดง Entity ที่เชื่อมโยง:

```
QT-2026-015 ──→ SO-2026-008 ──→ PRJ-2026-003
(Accepted)      (In Production)   (Active)
```

แต่ละ Entity แสดงรหัสและสถานะปัจจุบัน คลิกรหัสใดก็ได้เพื่อนำทางไปยังหน้ารายละเอียด

ด้านล่างห่วงโซ่:
- ชื่อลูกค้า
- ชื่อผลิตภัณฑ์และวัสดุ
- วันครบกำหนดพร้อมวันที่เหลือ

[image: traceability-flow-chain — ภาพหน้าจอของส่วนห่วงโซ่การไหลแสดง Quote → SO → Project ที่เชื่อมโยงพร้อมป้ายสถานะ]

---

## การแสดงผลขั้นตอน Work Order

สำหรับแต่ละ Work Order ที่เชื่อมโยง ไปป์ไลน์แนวนอนแสดงขั้นตอนทั้งหมด:

```
┌────┐  ┌────┐  ┌────┐  ┌────┐  ┌────┐  ┌────┐
│ DC │→│ DB │→│ CNC│→│ OX │→│ QC │→│ PK │
│ ✅ │  │ ✅ │  │ 🔵 │  │ ⬜ │  │ ⬜ │  │ ⬜ │
└────┘  └────┘  └────┘  └────┘  └────┘  └────┘
```

| ไอคอน | สถานะ |
|------|--------|
| ✅ เขียว | เสร็จสิ้น |
| 🔵 น้ำเงิน (กะพริบ) | กำลังดำเนินการ |
| ⬜ เทา | รอดำเนินการ |
| ⏭ หรี่ | ข้ามไป |

ตัวบ่งชี้เพิ่มเติม:
- ไอคอน 🚚 รถบรรทุกบนขั้นตอน Outside Processing
- สามเหลี่ยม ⚠ เตือนบนขั้นตอนที่มี NCR

ด้านล่างไปป์ไลน์:
- ชื่อและสถานะขั้นตอนปัจจุบัน
- เครื่องจักรที่กำหนด (ถ้ามี)
- แถบความคืบหน้าจำนวน (เช่น "12/20 pcs reported")

[image: traceability-step-pipeline — ภาพหน้าจอของการแสดงผลไปป์ไลน์ขั้นตอนสำหรับ Work Order]

---

## ส่วน Outside Processing

ถ้าขั้นตอน WO ใดเกี่ยวข้องกับ Outside Processing ส่วนเฉพาะจะแสดง:

- อ้างอิง OPO และชื่อขั้นตอน
- ชื่อซัพ
- สถานะ (Pending, Shipped, Late)
- วันจัดส่งและวันกลับ

[image: traceability-opo-section — ภาพหน้าจอของส่วน Outside Processing]

---

## ส่วนคุณภาพ

แสดงข้อมูลที่เกี่ยวข้องกับคุณภาพสำหรับคำสั่งซื้อที่ติดตาม:

- ผลการตรวจสอบ (ผ่าน/ไม่ผ่าน)
- ระเบียน NCR (ถ้ามี)
- ประวัติ Rework/Remake

[image: traceability-quality-section — ภาพหน้าจอของส่วนคุณภาพ]

## Timeline

บันทึกเหตุการณ์ตามลำดับเวลาแสดงทุกเหตุการณ์สำคัญ:

| เหตุการณ์ | แหล่งที่มา |
|-------|--------|
| Quote สร้าง/ยอมรับ | Timestamp ของ Quote |
| SO สร้าง | วันที่สร้าง Sales Order |
| WO ปล่อย | Timestamp การเปลี่ยนสถานะ WO |
| ขั้นตอนเริ่ม/เสร็จสิ้น | actual_start/actual_end ของขั้นตอน |
| OPO จัดส่ง/รับ | วันจัดส่ง/รับ OPO |
| การตรวจสอบบันทึก | วันที่สร้างการตรวจสอบ |
| NCR เปิด | วันที่สร้าง NCR |
| WO เสร็จสมบูรณ์ | Timestamp การเสร็จสมบูรณ์ WO |

เหตุการณ์เรียงลำดับล่าสุดก่อน

[image: traceability-timeline — ภาพหน้าจอของส่วน Timeline แสดงเหตุการณ์ตามลำดับเวลา]

## การตรวจจับคอขวด

มุมมองติดตามย้อนกลับไฮไลต์คอขวดโดยอัตโนมัติ:

| คอขวด | ตัวบ่งชี้ |
|------------|-----------|
| ขั้นตอนทำงานเกินระยะเวลาที่วางแผน | "Running over by X hours" |
| OPO ล่าช้า (เกินวันที่คาดว่าจะกลับ) | "Waiting on vendor: X days late" |
| ขั้นตอนพร้อมแต่ยังไม่ได้จัดตาราง | "Ready to start — not yet scheduled" |
| WO ถูก Hold | "On hold: [สาเหตุ]" |

[image: traceability-bottleneck — ภาพหน้าจอแสดงตัวบ่งชี้คอขวดบนขั้นตอน]

## กรณีการใช้งาน

| สถานการณ์ | สิ่งที่ต้องค้นหา |
|----------|---------------|
| ลูกค้าโทรถามเรื่องคำสั่งซื้อ | ค้นหาตามรหัส SO หรือชื่อลูกค้า |
| "ทำไม WO นี้ถึงล่าช้า?" | ค้นหาตามรหัส WO, ตรวจสอบตัวบ่งชี้คอขวด |
| "อะไรบล็อกชิ้นส่วนนี้?" | ค้นหาตามชื่อผลิตภัณฑ์, ตรวจสอบไปป์ไลน์ขั้นตอน |
| "ซัพส่งชิ้นส่วนกลับมาแล้วหรือยัง?" | ค้นหาตามรหัส WO, ตรวจสอบส่วน OPO |
| "QC ผ่านชุดนี้แล้วหรือยัง?" | ค้นหาตามรหัส WO, ตรวจสอบส่วนคุณภาพ |
