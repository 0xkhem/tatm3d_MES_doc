# การส่งมอบบางส่วน / Split Delivery

ใน HMLV ลูกค้ามักต้องการชิ้นงานเป็น Batch — "ส่ง 20 ชิ้นตอนนี้ ส่วนที่เหลือเมื่อพร้อม" หรือปัญหาคุณภาพทำให้ชิ้นงานบางส่วนเสียหายระหว่างการรัน และคุณต้องส่งชิ้นที่ผ่านในขณะที่ผลิตชิ้นที่เสียหายใหม่ คู่มือนี้ครอบคลุมวิธีจัดการการส่งมอบแบบแยกส่วนใน TAM3D

---

## ความท้าทายหลัก

โมเดล Work Order ของ TAM3D ติดตามจำนวนเดียวสำหรับงานเดียว WO สำหรับ "Motor Housing 50 ชิ้น" คืองานหนึ่งหน่วย เมื่อลูกค้าต้องการ 20 ชิ้นก่อน คุณต้องการวิธี:

1. ติดตามการส่งมอบบางส่วนกับ Sales Order
2. ให้การผลิตดำเนินต่อไปสำหรับ 30 ชิ้นที่เหลือ
3. รักษา Traceability จาก SO → การส่งมอบบางส่วน → WO ที่เหลือ

---

## แนวทางที่ 1: แยก Work Order ตั้งแต่การสร้าง

แนวทางที่สะอาดที่สุดคือแยกคำสั่งซื้อเป็น WO แยกกันก่อนเริ่มการผลิต

**เมื่อไหรควรใช้:** คุณรู้ล่วงหน้าว่าลูกค้าต้องการการส่งมอบแบบแบ่งระยะ (เช่น "20 ชิ้นภายใน 15 มีนาคม, 30 ชิ้นภายใน 1 เมษายน")

**วิธีทำ:**

1. เมื่อแปลง Sales Order Line Item เป็น Work Orders ให้สร้าง WO สองรายการแทนที่จะเป็นหนึ่ง:
   - WO-2026-00001: จำนวน 20, วันครบกำหนด 15 มีนาคม
   - WO-2026-00002: จำนวน 30, วันครบกำหนด 1 เมษายน

2. WO ทั้งสองอ้างอิง SO Line Item และ Process Route เดียวกัน

3. จัดตารางแยกกันบน Scheduling Board WO แรกได้รับการจัดตาราง Priority เพื่อให้ถึงวันที่เร็วกว่า

4. เมื่อ WO-2026-00001 เสร็จสิ้น สามารถบรรจุและส่งได้อย่างอิสระ SO ยังคงอยู่ใน "In Production" จนกว่า WO-2026-00002 จะเสร็จสิ้นด้วย

**ข้อดี:** Traceability เต็มรูปแบบ การจัดตารางอิสระ วันครบกำหนดที่ชัดเจนต่อ Batch

**ข้อเสีย:** Setup สองครั้งแยกกันหากชิ้นงานต้องการ Fixture พิจารณาว่าต้นทุน Setup คุ้มค่ากับความยืดหยุ่นในการจัดตารางหรือไม่ หรือคุณสามารถรันทั้งสองจำนวนใน Setup เดียวแล้วบรรจุ/ส่ง 20 ชิ้นแรกเมื่อออกจากเครื่อง

---

## แนวทางที่ 2: แยกระหว่างการผลิต

บางครั้งการตัดสินใจแยกเกิดขึ้นหลังจากการผลิตเริ่มแล้ว — ลูกค้าโทรมาขอการส่งมอบบางส่วนก่อนของสิ่งที่ทำเสร็จแล้ว

**เมื่อไหรควรใช้:** WO อยู่ใน in_progress แล้วและจำนวนบางส่วนเสร็จสิ้น

**วิธีทำ:**

1. เปิด Work Order บันทึกจำนวนที่เสร็จสิ้นปัจจุบัน (มองเห็นได้ในความคืบหน้าของขั้นตอน)

2. หาก WO อยู่ที่ขั้นตอนที่ชิ้นงานบางส่วนเสร็จและพร้อมส่ง:
   - เสร็จสิ้นขั้นตอนปัจจุบันสำหรับจำนวนที่เสร็จ
   - ใช้ **Quantity Report** (ช่างรายงานจำนวนที่เสร็จต่อขั้นตอน) เพื่อบันทึกจำนวนชิ้นที่ผ่าน

3. สำหรับชิ้นงานที่กำลังส่งตอนนี้:
   - สร้าง **Outbound Receipt** ใน Inventory (Inventory → Outbound) สำหรับจำนวนบางส่วน
   - อ้างอิง WO Code และ SO Code ในหมายเหตุ Receipt เพื่อ Traceability

4. สำหรับชิ้นงานที่เหลือ:
   - WO ดำเนินการผลิตต่อสำหรับจำนวนที่เหลือ
   - ไม่จำเป็นต้องเปลี่ยนแปลงระบบ — WO ติดตามจำนวนรวม และการส่งมอบบางส่วนถูกบันทึกใน Inventory

5. อัปเดตหมายเหตุ Sales Order เพื่อสะท้อนวันที่และจำนวนการส่งมอบบางส่วน

**ข้อจำกัด:** ระบบปัจจุบันไม่แยก WO ระหว่างการรันเป็นสองระเบียนแยกกัน การส่งมอบบางส่วนถูกติดตามผ่าน Inventory Transactions ไม่ใช่เป็น WO Completion Event แยกต่างหาก หากต้องการการติดตามการแยกอย่างเป็นทางการ ให้ใช้แนวทางที่ 1 ตั้งแต่ต้น

---

## แนวทางที่ 3: Remake สำหรับการเสียหายบางส่วน

สถานการณ์นี้เกิดขึ้นเมื่อปัญหาคุณภาพทำให้ชิ้นงานบางส่วนเสียหายและคุณต้องส่งชิ้นที่ดีในขณะที่ผลิตชิ้นที่เสียหายใหม่

**สถานการณ์:** WO สำหรับ 50 ชิ้น การตรวจสอบ QC พบชิ้นงาน 8 ชิ้นที่มีข้อบกพร่องด้านมิติ ลูกค้าต้องการ 42 ชิ้นที่ดีตอนนี้ ส่วน 8 ชิ้นจะผลิตใหม่

**วิธีทำ:**

1. สร้างระเบียน **Inspection** สำหรับ Batch (Quality → Inspections → Create)
   - ตั้ง inspected_qty: 50, defect_count: 8, result: Fail

2. สร้าง **NCR** (Quality → NCR → Create) สำหรับ 8 ชิ้นที่มีข้อบกพร่อง
   - ตั้ง Disposition: **Remake**
   - มอบหมายให้วิศวกรที่รับผิดชอบ

3. Resolve NCR ด้วย Remake Disposition ระบบจะ:
   - สร้าง Child WO สำหรับจำนวน Remake (8 ชิ้น) โดยอัตโนมัติ
   - เชื่อมโยง Child WO กับ NCR ต้นฉบับเพื่อ Traceability
   - WO ต้นฉบับสามารถทำเครื่องหมายว่าเสร็จสิ้นสำหรับ 42 ชิ้นที่ดี

4. ส่ง 42 ชิ้นที่ดีผ่าน Outbound Receipt ใน Inventory

5. จัดตาราง Remake WO (8 ชิ้น) บน Scheduling Board จะปรากฏใน Unscheduled Sidebar

**ผลลัพธ์:** SO แสดง WO ต้นฉบับว่าเสร็จสิ้น (ส่ง 42 ชิ้น) และ Remake WO ว่ากำลังผลิต (8 ชิ้น) Traceability เต็มรูปแบบจากข้อบกพร่อง → NCR → Remake → การส่งมอบ

## การติดตามการส่งมอบบางส่วนบน Sales Order

**Delivery Promise Signal** ของ Sales Order สะท้อนสถานะการส่งมอบโดยรวม เมื่อ WO บางรายการเสร็จสิ้นและรายการอื่นยังอยู่ในการผลิต:

- Signal พิจารณา WO ทั้งหมดที่เชื่อมโยงกับ SO
- หาก WO ใดยังอยู่ในการผลิตเกินวันครบกำหนด SO Signal จะเปลี่ยนเป็นสีเหลืองหรือแดง
- ใช้หน้า **Order Traceability** (Production → Traceability) เพื่อดูภาพรวมทั้งหมด: SO → Project → WO ทั้งหมด → สถานะขั้นตอน

สำหรับการสื่อสารกับลูกค้า ทีมขายสามารถดูหน้ารายละเอียด SO ซึ่งแสดงสถานะของแต่ละ Line Item และ WO ที่เชื่อมโยง

---

## Inventory Traceability สำหรับการส่งมอบบางส่วน

การส่งมอบบางส่วนทุกครั้งควรมี **Outbound Receipt** ที่สอดคล้องกันใน Inventory:

- ไปที่ **Inventory → Outbound**
- สร้าง Receipt พร้อม:
  - Material: ชิ้นงานสำเร็จรูป
  - Quantity: จำนวนการส่งมอบบางส่วน
  - Reference: WO Code และ SO Code ในฟิลด์หมายเหตุ

ซึ่งสร้าง Audit Trail ว่าส่งอะไร เมื่อไหร่ และกับคำสั่งซื้อใด หน้า Inventory Balance จะสะท้อนการลดลงของสต็อกสินค้าสำเร็จรูป

---

## แนวปฏิบัติที่แนะนำ

สำหรับโรงงาน HMLV ที่ทำการส่งมอบแบบแบ่งระยะเป็นประจำ แนวทางที่สะอาดที่สุดคือ **แยกที่ระดับ SO** — สร้าง Line Items แยกกันสำหรับแต่ละ Milestone การส่งมอบ แต่ละรายการมีวันครบกำหนดของตัวเอง ซึ่งไหลตามธรรมชาติไปยัง WO แยกกันที่มีการจัดตารางอิสระและการติดตามการส่งมอบที่ชัดเจน

หลีกเลี่ยงการแยก WO ระหว่างการผลิตเว้นแต่จะถูกบังคับโดยเหตุการณ์คุณภาพ ระบบรองรับได้ แต่ Traceability ไม่สะอาดเท่ากับการวางแผนการแยกล่วงหน้า
