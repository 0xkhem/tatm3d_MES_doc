# คู่มือการตีความ KPI

ตัวเลขที่ไม่มีบริบทคือสัญญาณรบกวน คู่มือนี้อธิบายว่า KPI แต่ละตัวใน VulcornOne หมายความว่าอย่างไร ตัวเลขที่ดีสำหรับโรงงาน CNC แบบ HMLV ควรเป็นเท่าไร และที่สำคัญที่สุด — ต้องทำอย่างไรเมื่อตัวเลขไม่ดี

---

## ดู KPI ได้ที่ไหน

KPI แสดงใน 3 ที่:

1. **Home Dashboard** (`/home`) — แถบ KPI ด้านบน แสดงจำนวน WO ที่กำลังดำเนินการ, อัตราส่งมอบตรงเวลา และรายการที่ต้องให้ความสนใจ
2. **Production Dashboard** (`/production/dashboard`) — การ์ด KPI เต็มรูปแบบ, แถบกระจายสถานะ, WIP aging, งานตามกระบวนการ และความคืบหน้า WO
3. **Schedule Adherence Report** (`/production/dashboard/schedule-adherence`) — เปรียบเทียบแผน vs. จริง, อัตราการยึดตามแผน, การวิเคราะห์ความแตกต่าง
4. **Scheduling Board KPI Panel** — แสดงหลังจากรัน auto-schedule แสดง metric เฉพาะสถานการณ์ (makespan, ออเดอร์ล่าช้า, ชั่วโมง setup, utilization)

---

## อัตราส่งมอบตรงเวลา (On-Time Delivery Rate)

**คำจำกัดความ:** เปอร์เซ็นต์ของ Work Order ที่เสร็จสมบูรณ์ตรงเวลาหรือก่อนกำหนดส่ง

`On-Time Delivery = (WO ที่เสร็จตรงเวลา) / (WO ที่เสร็จทั้งหมด) × 100`

### ตัวเลขที่ดีควรเป็นเท่าไร?

| อัตรา | การประเมิน |
|------|-----------|
| ≥ 95% | ดีเยี่ยม — การจัดตารางและการวางแผนกำลังการผลิตทำงานได้ดี |
| 90–94% | ดี — มีปัญหาเล็กน้อย ตรวจสอบ 5–10% ที่ล่าช้า |
| 80–89% | น่าเป็นห่วง — ปัญหาเชิงระบบ น่าจะเป็นเรื่องกำลังการผลิตหรือการจัดตาราง |
| < 80% | วิกฤต — ความน่าเชื่อถือในการส่งมอบมีปัญหา ต้องแก้ไขทันที |

สำหรับโรงงาน HMLV เป้าหมายที่เป็นจริงคือ 90%+ ส่วน 95%+ ทำได้ด้วยวินัยในการจัดตารางที่ดี ต่ำกว่า 85% หมายความว่าลูกค้าได้รับสินค้าล่าช้าเป็นประจำ — นั่นคือความเสี่ยงทางธุรกิจ

### อะไรทำให้อัตราต่ำ?

**กำหนดส่งที่มองโลกสวยเกินไป** ฝ่ายขายรับปากวันส่งโดยไม่ตรวจสอบ Delivery Promise Signal แก้ไข: บังคับให้ตรวจสอบสัญญาณก่อนยืนยันวันส่งทุกครั้ง

**เครื่องจักรหยุดทำงานโดยไม่ได้วางแผน** เครื่องเสียและไม่สามารถย้ายงานได้ทันเวลา แก้ไข: รักษากำลังการผลิตสำรองบนเครื่องจักรที่สำคัญ ใช้ขั้นตอนการย้ายงานทันทีเมื่อเครื่องหยุดทำงาน

**การส่งงานนอกล่าช้า (Outside Processing)** ผู้รับเหมาส่งชิ้นงานกลับล่าช้า ทำให้ขั้นตอนถัดไปติดขัด แก้ไข: เพิ่มวันบัฟเฟอร์ในวันคืนงาน OPO ที่คาดไว้ ติดตาม OPO panel ทุกวัน

**การทำซ้ำและการผลิตใหม่ (Rework/Remake)** ปัญหาคุณภาพทำให้ต้องรีเซ็ตขั้นตอนและเพิ่มวันในการเสร็จ WO แก้ไข: ติดตามอัตรา rework ตาม process definition แก้ไขสาเหตุที่แท้จริง

**ประมาณเวลา cycle ต่ำเกินไป** เวลามาตรฐานใน process definition มองโลกสวยเกินไป แก้ไข: ใช้ Schedule Adherence Report เปรียบเทียบชั่วโมงที่วางแผน vs. จริง อัปเดต standard_time_minutes

### วิธีตรวจสอบ

1. ไปที่ **Production → Work Orders** กรองตามสถานะ: Completed เรียงตามวันกำหนดส่ง
2. ดู WO ที่เสร็จหลังวันกำหนดส่ง — มีอะไรเหมือนกัน?
3. ตรวจสอบ **Schedule Adherence Report** — WO ที่ล่าช้ามักล่าช้าที่ขั้นตอนหรือเครื่องจักรเดียวกันหรือไม่?
4. ถ้าเป็นเครื่องจักรเดียวกันเสมอ: คอขวดด้านกำลังการผลิต
5. ถ้าเป็นขั้นตอนเดียวกันเสมอ: ประมาณเวลา cycle ต่ำเกินไปหรือปัญหาคุณภาพ
6. ถ้าสุ่ม: น่าจะเป็นการส่งงานนอกล่าช้าหรือออเดอร์เร่งด่วนรบกวน

## อัตราการเสร็จ WO (WO Completion Rate)

**คำจำกัดความ:** เปอร์เซ็นต์ของ Work Order ที่ถึงสถานะ `completed` จาก WO ทั้งหมดที่ไม่ถูกยกเลิกในช่วงเวลาหนึ่ง

### ตัวเลขที่ดีควรเป็นเท่าไร?

ขึ้นอยู่กับเวลา cycle เฉลี่ยของ WO โรงงานที่มี cycle เฉลี่ย 2 สัปดาห์จะมีอัตราเสร็จรายสัปดาห์ต่ำกว่าโรงงานที่มีงาน 2 วัน metric ที่มีประโยชน์กว่าคือ **อัตราเสร็จเทียบกับอัตราปล่อยงาน** — คุณเสร็จ WO เร็วเท่ากับที่ปล่อยงานหรือไม่?

ถ้าอัตราเสร็จ < อัตราปล่อยงานหลายสัปดาห์ติดต่อกัน WIP ของคุณกำลังเพิ่มขึ้น นั่นคือสัญญาณเตือน

**รูปแบบที่ดี:** อัตราเสร็จ ≈ อัตราปล่อยงาน (WIP คงที่)  
**รูปแบบเตือน:** อัตราเสร็จ < อัตราปล่อยงาน (WIP เพิ่มขึ้น งานค้างสะสม)  
**รูปแบบน่าเป็นห่วง:** อัตราเสร็จ > อัตราปล่อยงานเป็นเวลานาน (กำลังเคลียร์งานค้าง อาจจะมีงานน้อยเกินไปเร็วๆ นี้)

### วิธีใช้

ไปที่ **Production → Dashboard** แถบกระจายสถานะแสดงการแบ่ง WO ทั้งหมดตามสถานะ ถ้า `in_progress` เพิ่มขึ้นทุกสัปดาห์ในขณะที่ `completed` คงที่ คุณมีปัญหา throughput

## Work Order ที่กำลังดำเนินการ (Active Work Orders)

**คำจำกัดความ:** จำนวน WO ที่อยู่ในสถานะ `released` หรือ `in_progress`

### ตัวเลขที่ดีควรเป็นเท่าไร?

ไม่มีเป้าหมายสากล — ขึ้นอยู่กับขนาดโรงงาน คำถามที่มีประโยชน์คือ: **ตัวเลขนี้เพิ่มขึ้น คงที่ หรือลดลง?**

- เพิ่มขึ้นทุกสัปดาห์: คุณรับงานมากกว่าที่ทำเสร็จ ต้องเพิ่มกำลังการผลิตหรือชะลอการรับออเดอร์
- คงที่: สถานะคงตัวที่ดี
- ลดลง: กำลังเคลียร์งานค้าง ดีถ้าตั้งใจ น่าเป็นห่วงถ้าการรับออเดอร์ลดลง

metric ที่นำไปปฏิบัติได้มากกว่าคือ **WIP aging** — WO ที่กำลังดำเนินการอยู่นานแค่ไหนแล้ว?

### WIP Aging

Production Dashboard แสดงส่วน WIP aging ซึ่งแบ่ง WO ที่กำลังดำเนินการตามระยะเวลาที่อยู่ในสถานะ `in_progress`:

| อายุ | ความหมาย |
|-----|----------|
| 0–3 วัน | ปกติ — เพิ่งเริ่ม |
| 4–7 วัน | เฝ้าดู — ควรจะเสร็จเร็วๆ นี้ |
| 8–14 วัน | น่าเป็นห่วง — ทำไมยังเปิดอยู่? |
| > 14 วัน | มีปัญหา — ตรวจสอบทันที |

WO ที่อยู่ในสถานะ `in_progress` นานกว่า 14 วัน อาจเป็นเพราะ:
- ติดอยู่ที่ขั้นตอนส่งงานนอก (ตรวจสอบ OPO)
- ถูกพักงาน (ตรวจสอบเหตุผลการพัก)
- มีปัญหาคุณภาพ (ตรวจสอบ NCR ที่เปิดอยู่)
- การรายงานความคืบหน้าขั้นตอนไม่ถูกต้อง (พนักงานไม่รายงาน)

คลิกเข้าไปใน WO ที่ aging และตรวจสอบความคืบหน้าขั้นตอน คำตอบมักจะชัดเจน

## อัตราการยึดตามแผน (Schedule Adherence Rate)

**คำจำกัดความ:** เปอร์เซ็นต์ของงานที่จัดตารางไว้ที่เริ่มภายในช่วงเวลาที่ยอมรับได้ของเวลาเริ่มที่วางแผนไว้

`Adherence = (งานที่เริ่มตรงเวลา) / (งานที่จัดตารางทั้งหมด) × 100`

ดูได้ที่: **Production → Dashboard → Schedule Adherence**

### ตัวเลขที่ดีควรเป็นเท่าไร?

| อัตรา | การประเมิน |
|------|-----------|
| ≥ 85% | ดี — ตารางของคุณเป็นจริงและพนักงานทำตาม |
| 70–84% | พอรับได้ — มีการเลื่อนบ้าง ตรวจสอบรูปแบบ |
| < 70% | แย่ — ตารางของคุณไม่เป็นจริงหรือไม่ถูกปฏิบัติตาม |

### วิธีตีความ

**Adherence ต่ำ + เวลา cycle ถูกต้อง = ปัญหาการจัดตาราง** งานถูกจัดวางแน่นเกินไป ไม่มีบัฟเฟอร์สำหรับความแปรปรวนของ setup, ความพร้อมของพนักงาน หรือความล่าช้าเล็กน้อย เพิ่ม slack ในขอบเขตการจัดตาราง

**Adherence ต่ำ + เวลา cycle ไม่ถูกต้อง = ปัญหาคุณภาพข้อมูล** standard_time_minutes ของคุณผิด งานใช้เวลานานกว่าที่วางแผนอย่างสม่ำเสมอ ใช้ข้อมูลความแตกต่างในรายงาน adherence เพื่อระบุว่า process definition ใดต้องอัปเดตเวลา

**Adherence ต่ำบนเครื่องจักรเฉพาะ = คอขวด** เครื่องจักรหนึ่งล่าช้ากว่าตารางอย่างสม่ำเสมอในขณะที่เครื่องอื่นตรงเวลา เครื่องนั้นคือข้อจำกัดของคุณ ทุกอย่างที่อยู่ถัดไปจะล่าช้า

**Adherence ต่ำในวันเฉพาะ = ปัญหาการจัดกะ** ถ้า adherence วันจันทร์ต่ำกว่าวันอื่นอย่างสม่ำเสมอ ตรวจสอบ Shift Roster — เช้าวันจันทร์มีคนน้อยเกินไปหรือไม่?

### การใช้ข้อมูลความแตกต่าง

Schedule Adherence Report แสดงชั่วโมงที่วางแผน vs. ชั่วโมงจริงต่องาน เรียงตามความแตกต่าง (จริง − แผน) จากมากไปน้อย งานที่อยู่ด้านบนคือการประมาณเวลาที่ผิดพลาดมากที่สุด

ถ้า "CNC Milling Op10 — Steel 4140" ใช้เวลาเกินแผน 40% อย่างสม่ำเสมอ ให้อัปเดต standard_time_minutes การเปลี่ยนแปลงเพียงครั้งเดียวนี้จะปรับปรุงความแม่นยำของตารางสำหรับ WO ในอนาคตทุกตัวที่ใช้ process definition นั้น

## KPI ของ Scheduling Board (หลังรัน Auto-Schedule)

KPI เหล่านี้แสดงใน KPI panel หลังจากรัน auto-scheduling engine อธิบายตารางที่เสนอก่อนที่คุณจะยอมรับ

### Makespan

**คำจำกัดความ:** จำนวนวันปฏิทินทั้งหมดจากงานแรกที่จัดตารางถึงงานสุดท้าย

**ความหมาย:** ใช้เวลานานแค่ไหนในการทำงานทั้งหมดในขอบเขตการจัดตาราง สั้นกว่าดีกว่า — หมายความว่างานเสร็จเร็วขึ้น

**เมื่อไรต้องดำเนินการ:** ถ้า makespan ยาวเกินวันสิ้นสุดขอบเขต บางงานไม่สามารถจัดตารางได้ ตรวจสอบ Unscheduled Sidebar — งานเหล่านั้นต้องได้รับความสนใจ (ขยายขอบเขตหรือเพิ่มกำลังการผลิต)

### ออเดอร์ล่าช้า (Late Orders)

**คำจำกัดความ:** จำนวน Work Order ในตารางที่เสนอที่คาดว่าจะเสร็จหลังวันกำหนดส่ง

**นี่คือ KPI ที่สำคัญที่สุดใน panel** ออเดอร์ล่าช้าเป็นศูนย์หมายความว่าตารางที่เสนอตรงตามข้อผูกพันทั้งหมด ตัวเลขใดๆ > 0 หมายความว่าคุณมีปัญหาการส่งมอบในตารางที่เสนอ

**ต้องทำอย่างไรกับออเดอร์ล่าช้า:**
1. จดว่า WO ใดล่าช้า (เลื่อนเมาส์ไปบน timeline เพื่อระบุ)
2. ตรวจสอบว่าสามารถย้ายให้เร็วขึ้นโดยย้ายไปเครื่องจักรอื่นได้หรือไม่
3. ตรวจสอบว่าวันกำหนดส่งเป็นจริงหรือไม่ — ตั้งโดยไม่ตรวจสอบกำลังการผลิตหรือเปล่า?
4. ถ้าหลีกเลี่ยงไม่ได้ ยอมรับตารางและแจ้งฝ่ายขายทันทีเพื่อให้ติดต่อลูกค้า

**อย่ายอมรับตารางที่มีออเดอร์ล่าช้าโดยไม่มีแผน** การยอมรับไม่ได้ทำให้ปัญหาหายไป — แค่หมายความว่าคุณยอมรับที่จะส่งล่าช้า

### Total Tardiness (ความล่าช้ารวม)

**คำจำกัดความ:** ผลรวมของชั่วโมงล่าช้าทั้งหมดของออเดอร์ที่ล่าช้า (วันล่าช้า × 8 ชั่วโมงต่อวัน)

เสริม Late Orders ออเดอร์ล่าช้า 2 ตัวที่มี tardiness รวม 8 ชั่วโมงแตกต่างมากจากออเดอร์ล่าช้า 2 ตัวที่มี 80 ชั่วโมง กรณีแรกอาจกู้คืนได้ กรณีหลังคือความล้มเหลวในการส่งมอบอย่างร้ายแรง

### Avg Utilization (การใช้งานเฉลี่ย)

**คำจำกัดความ:** เปอร์เซ็นต์ของกำลังการผลิตเครื่องจักรทั้งหมดที่มี (ทุกเครื่อง ทุกวันในขอบเขต) ที่ถูกจัดตาราง

| Utilization | การประเมิน |
|-------------|-----------|
| < 60% | งานน้อย — มีกำลังการผลิตสำรองมาก |
| 60–80% | สมดุล — สมดุลที่ดีระหว่างภาระงานและบัฟเฟอร์ |
| 80–90% | สูง — มีพื้นที่จำกัดสำหรับเหตุการณ์ไม่คาดคิด |
| > 90% | เกินกำลัง — เหตุการณ์ไม่คาดคิดใดๆ จะทำให้ออเดอร์ล่าช้า |

**กับดัก:** Utilization สูงดูดีบนกระดาษแต่เปราะบาง เครื่องเสีย ปัญหาคุณภาพ หรือออเดอร์เร่งด่วนจะทำให้ส่งมอบล่าช้าทันที โรงงาน HMLV ควรตั้งเป้า utilization 70–80% เพื่อรักษาความยืดหยุ่น

**Utilization ต่ำแต่มีออเดอร์ล่าช้า** = คุณมีคอขวด เครื่องจักรหนึ่งเกินกำลังในขณะที่เครื่องอื่นว่าง ค่าเฉลี่ยซ่อนปัญหา ตรวจสอบ utilization strip ต่อเครื่อง

### Total Setup Hours (ชั่วโมง Setup รวม)

**คำจำกัดความ:** ผลรวมของเวลาเปลี่ยนงานทั้งหมดในตารางที่เสนอ

น้อยกว่าดีกว่า นี่คือผลลัพธ์โดยตรงของการปรับ setup ให้เหมาะสม เปรียบเทียบตัวเลขนี้ระหว่างตาราง Slack-rule และตาราง Setup-Aware Grouping เพื่อวัดการประหยัด setup

ถ้า Total Setup Hours สูงเมื่อเทียบกับ Total Scheduled Hours กลุ่ม setup ของคุณอาจต้องปรับปรุง — หรือคุณมีงานหลายตระกูลทำงานพร้อมกันมากเกินไป

## การใช้ Production Dashboard เพื่อหาคอขวด

Production Dashboard (`/production/dashboard`) คือการตรวจสุขภาพรายสัปดาห์ของคุณ นี่คือแนวทางเป็นระบบในการระบุคอขวด

### ขั้นตอนที่ 1: ตรวจสอบแถบกระจายสถานะ

แถบแสดง WO ทั้งหมดแบ่งตามสถานะ: draft, released, in_progress, on_hold, completed, cancelled

**สัญญาณอันตราย:** ส่วน `on_hold` ใหญ่ WO ที่ถูกพักจำนวนมากหมายความว่าการผลิตถูกขัดจังหวะบ่อย ตรวจสอบเหตุผลการพัก — เป็นเครื่องจักรเดียวกันหรือไม่? วัสดุเดียวกัน? ขั้นตอนเดียวกัน?

**สัญญาณอันตราย:** ส่วน `released` ใหญ่แต่ `in_progress` เล็ก WO ถูกปล่อยแต่ไม่ถูกเริ่ม พนักงานไม่หยิบงาน (ตรวจสอบ Shift Roster — เครื่องจักรมีคนประจำหรือไม่?) หรือ scheduling board จัดวางไว้ไกลเกินไปในอนาคต

### ขั้นตอนที่ 2: ตรวจสอบ WIP aging

เรียงตามอายุจากมากไปน้อย WO ที่อยู่ in-progress นานที่สุดคือการส่งมอบที่เสี่ยงที่สุด เปิดแต่ละตัวและตรวจสอบ:
- ติดอยู่ที่ขั้นตอนไหน?
- มี NCR ที่เปิดอยู่หรือไม่?
- มี OPO ที่เกินกำหนดหรือไม่?
- ขั้นตอนถูกมอบหมายให้เครื่องจักรที่อยู่ระหว่างซ่อมบำรุงหรือไม่?

### ขั้นตอนที่ 3: ตรวจสอบส่วนงานตามกระบวนการ

ส่วนนี้แสดงความคืบหน้าระดับขั้นตอนของ WO ที่กำลังดำเนินการทั้งหมด มองหาขั้นตอนที่อยู่ในสถานะ `in_progress` นานโดยไม่เสร็จ ถ้าขั้นตอน "CNC Milling Op10" aging ทั้งหมด เครื่อง CNC Mill คือคอขวดของคุณ

### ขั้นตอนที่ 4: เปรียบเทียบกับ Schedule Adherence

ไปที่ Schedule Adherence Report กรองตามเครื่องจักรหรือ process definition ที่ระบุในขั้นตอนที่ 3 ถ้าเครื่อง/ขั้นตอนนั้นแสดงความแตกต่างสูงอย่างสม่ำเสมอ (จริง >> แผน) คุณมี:
- ปัญหากำลังการผลิต (งานมากเกินไปสำหรับเครื่อง)
- ปัญหาเวลา cycle (เวลามาตรฐานผิด)
- ปัญหาคุณภาพ (rework เพิ่มเวลาที่ไม่ได้วางแผน)

### ขั้นตอนที่ 5: ดำเนินการ

| สิ่งที่พบ | การดำเนินการ |
|---------|------------|
| เครื่องจักรหนึ่งเกินกำลังเสมอ | เพิ่มกะที่สอง ส่งงานนอกบางส่วน หรือลดการรับออเดอร์สำหรับเครื่องประเภทนั้น |
| เวลา cycle ผิดอย่างสม่ำเสมอ | อัปเดต standard_time_minutes ใน process definition |
| อัตรา rework สูงในขั้นตอนเฉพาะ | ตรวจสอบสาเหตุ — เครื่องมือ, จิ๊ก, การฝึกอบรมพนักงาน |
| WO จำนวนมากถูกพักเพราะวัสดุ | ปรับปรุงการวางแผนวัสดุ — ปล่อย WO เมื่อยืนยันว่ามีวัสดุพร้อมเท่านั้น |
| OPO เกินกำหนดจำนวนมาก | หาผู้รับเหมาสำรอง เพิ่มวันบัฟเฟอร์ใน lead time ปรับปรุงการสื่อสารกับผู้รับเหมา |

| KPI | เป้าหมาย | ตำแหน่ง | การดำเนินการหลักถ้าต่ำกว่าเป้า |
