# การ Setup หลายขั้นตอนบนเครื่องเดียว

ชิ้นงานที่ซับซ้อนมักต้องการหลาย Setup บนเครื่องเดียวกัน — Op10 (Setup แรก), Op20 (พลิก/จับยึดใหม่), Op30 (หมุน 4 แกน) และอื่นๆ นี่เป็นแนวปฏิบัติมาตรฐานบนเครื่อง CNC Mill 5 แกนและเครื่อง Lathe หลายแกน คู่มือนี้อธิบายวิธีจำลองสิ่งเหล่านี้อย่างถูกต้องใน Process Routes ของ TAM3D และวิธีที่ Scheduler จัดการกับมัน

---

## การ Setup หลายขั้นตอนคืออะไร?

"Setup" ในการกลึง CNC คือการกำหนดค่า Fixture ที่แตกต่างกัน ชิ้นงานชิ้นเดียวอาจต้องการ:

- **Op10:** จับใน Vise กลึงหน้าบนและรูปแบบรู
- **Op20:** พลิกชิ้นงาน จับบนฟีเจอร์ Op10 กลึงหน้าล่าง
- **Op30:** หมุน 90° กลึงโปรไฟล์ด้านข้าง

แต่ละการดำเนินงานคือขั้นตอนแยกกันใน Process Route แม้ว่าทั้งหมดจะรันบนเครื่องเดียวกัน ความแตกต่างหลักจากงานขั้นตอนเดียวคือแต่ละการดำเนินงานมี:

- เวลา Setup ของตัวเอง (การจับยึดใหม่, การ Probe ใหม่, การเปลี่ยนโปรแกรม)
- Cycle Time ของตัวเอง
- Tooling ที่อาจแตกต่างกัน

---

## การจำลองชิ้นงาน Multi-Op ใน Process Routes

### แนวทางที่แนะนำ: หนึ่งขั้นตอนต่อหนึ่งการดำเนินงาน

สร้าง Process Definition แยกกันสำหรับแต่ละการดำเนินงาน แล้วเรียงลำดับใน Process Route

**ตัวอย่าง: Hydraulic Manifold Block (5 แกน, 3 Setup)**

| ลำดับ | Process Definition | Setup Group | ชั่วโมงโดยประมาณ | หมายเหตุ |
|-------|-------------------|-------------|-----------------|---------|
| 10 | CNC Milling Op10 | Steel-Manifold | 3.0h | หน้า, รูปแบบรู, ฟีเจอร์บน |
| 20 | CNC Milling Op20 | Steel-Manifold | 2.5h | พลิก, หน้าล่าง, Cross-drilling |
| 30 | CNC Milling Op30 | Steel-Manifold | 1.5h | โปรไฟล์ด้านข้าง, เกลียว Port |
| 40 | Deburring | Manual-Deburr | 1.0h | ทุกหน้า |
| 50 | CMM Inspection | QC-Station | 0.5h | ตรวจสอบ 100% |
| 60 | Packing | Pack-Station | 0.2h | ใส่โฟม |

**ทำไมต้องแยกขั้นตอนแทนที่จะเป็น "CNC Milling" ขั้นตอนใหญ่ขั้นตอนเดียว?**

- แต่ละการดำเนินงานสามารถจัดตารางได้อย่างอิสระ หาก Op10 เสร็จตอนบ่ายสาม Op20 สามารถเริ่มทันทีหรือจัดตารางสำหรับเช้าวันถัดไป — Scheduler จัดการการเรียงลำดับ
- ช่างรายงานการเสร็จสิ้นต่อการดำเนินงาน ให้การติดตามความคืบหน้าที่ละเอียด
- หาก Op20 มีปัญหาคุณภาพ คุณสามารถ Rework เฉพาะการดำเนินงานนั้นโดยไม่ต้อง Reset Op10
- เวลา Setup ระหว่างการดำเนินงานถูกติดตามแยกจาก Cycle Time

### รูปแบบการตั้งชื่อสำหรับ Process Definitions แบบ Multi-Op

ใช้รูปแบบการตั้งชื่อที่สอดคล้องกันเพื่อให้ช่างและผู้วางแผนระบุการดำเนินงานได้ทันที:

```
CNC Milling Op10 — [ตระกูลชิ้นงาน]
CNC Milling Op20 — [ตระกูลชิ้นงาน]
CNC Milling Op30 — [ตระกูลชิ้นงาน]
```

หรือสำหรับงาน Lathe:
```
CNC Turning Op10 (Chuck) — [ตระกูลชิ้นงาน]
CNC Turning Op20 (Tailstock) — [ตระกูลชิ้นงาน]
```

สร้างสิ่งเหล่านี้ใน **Knowledge → Process Definitions** แต่ละ Definition ควรมี:
- `equipment_type` เดียวกัน (ทั้งหมดรันบนเครื่องประเภทเดียวกัน)
- `setup_group` เดียวกันหากเป็นตระกูลวัสดุ/Fixture เดียวกัน
- `standard_time_minutes` ที่แม่นยำสำหรับแต่ละการดำเนินงาน

## วิธีที่ Scheduler จัดการลำดับ Multi-Op

Scheduling Engine บังคับการเรียงลำดับขั้นตอน: Op20 ไม่สามารถเริ่มได้จนกว่า Op10 จะเสร็จสิ้น ซึ่งเป็นอัตโนมัติ — คุณไม่จำเป็นต้องกำหนดค่า

**ความหมายบน Timeline:**

หาก Op10 ถูกจัดตารางสำหรับวันจันทร์ 08:00–11:00 บนเครื่อง EQP-0003 Op20 จะถูกจัดตารางไม่เร็วกว่าวันจันทร์ 11:00 บนเครื่องเดียวกัน (หรือช้ากว่าหากเครื่องยุ่ง)

**เครื่องเดียวกันกับเครื่องต่างกัน:**

การดำเนินงานทั้งสามสามารถรันบนเครื่องเดียวกัน (พบบ่อยที่สุด) หรือบนเครื่องต่างกันของประเภทเดียวกัน Scheduler มอบหมายแต่ละการดำเนินงานให้เครื่องที่ดีที่สุดที่มีอย่างอิสระ โดยเคารพข้อจำกัดลำดับ

หากโรงงานของคุณมีเครื่อง 5 แกนสองเครื่องและ Op10 เสร็จบน Mill A Op20 สามารถจัดตารางบน Mill B หาก Mill A ยุ่ง — Scheduler จัดการสิ่งนี้โดยอัตโนมัติ

**เวลา Setup ระหว่างการดำเนินงาน:**

หาก Op10 และ Op20 มี `setup_group` เดียวกัน (เช่น `Steel-Manifold`) Scheduler จะถือว่าการเปลี่ยน Setup ระหว่างกันเป็น 0 นาที — ช่างเพียงแค่จับยึดชิ้นงานใหม่ ซึ่งถูกต้องสำหรับสถานการณ์ Multi-Op ส่วนใหญ่ที่เครื่องยังคง Setup สำหรับตระกูลชิ้นงานเดียวกัน

หากการดำเนินงานต้องการการ Reset เครื่องเต็มรูปแบบระหว่างกัน (ผิดปกติ) ให้กำหนด Setup Group ต่างกันและเพิ่มเวลาเปลี่ยน Setup ใน Setup Matrix

## ขั้นตอน Setup แยกสำหรับงานที่ซับซ้อน

สำหรับชิ้นงานที่ซับซ้อนมากที่ Setup เองเป็นกิจกรรมที่สำคัญและต้องใช้ทักษะ (เช่น การ Level แผ่นขนาดใหญ่อย่างแม่นยำ, การจัดตำแหน่ง Fixture 5 แกนที่ซับซ้อน) ให้พิจารณาเพิ่มขั้นตอน Setup ที่ชัดเจน:

| ลำดับ | Process Definition | ชั่วโมงโดยประมาณ | หมายเหตุ |
|-------|-------------------|-----------------|---------|
| 10 | CNC Setup & Programming | 2.0h | สร้าง Fixture, Prove-out โปรแกรม, ตรวจสอบชิ้นแรก |
| 20 | CNC Milling Op10 | 6.0h | Rough และ Finish |
| 30 | CNC Milling Op20 | 4.0h | พลิก, Finish ฟีเจอร์ที่เหลือ |

**เมื่อไหรควรใช้ขั้นตอน Setup แยก:**
- Setup ใช้เวลามากกว่า 2 ชั่วโมง
- คนต่างกันทำ Setup กับการกลึง (วิศวกร Setup กับช่าง)
- Setup ต้องการการตรวจสอบชิ้นแรกก่อนรัน Batch
- คุณต้องการติดตามเวลา Setup แยกจากเวลาการกลึงในรายงาน

**เมื่อไหรไม่ควรใช้:**
- Setup ใช้เวลาน้อยกว่า 30 นาทีและทำโดยช่างคนเดียวกัน
- คุณจับเวลา Setup ใน `standard_time_minutes` ของขั้นตอนการกลึงแล้ว

## ประสบการณ์ช่างสำหรับชิ้นงาน Multi-Op

จากมุมมองของช่างบน **Operator Panel**:

1. พวกเขาเห็น Op10 ใน Task Queue เริ่มต้น กลึง Setup แรก รายงานการเสร็จสิ้น
2. Op20 พร้อมใช้งานทันทีใน Queue ของพวกเขา (หรือ Queue ของช่างกะถัดไปหากเป็นกะต่างกัน)
3. พวกเขาหยิบ Op20 จับยึดชิ้นงานใหม่ กลึง Setup ที่สอง รายงานการเสร็จสิ้น
4. และต่อไปตลอดทุกการดำเนินงาน

มุมมองรายละเอียดขั้นตอนแสดงชื่อการดำเนินงาน (เช่น "CNC Milling Op20") และหมายเหตุที่คุณเพิ่ม (เช่น "พลิกชิ้นงานบนหน้า Datum ของ Op10, Probe X/Y/Z Zero") ใช้ฟิลด์หมายเหตุเพื่อสื่อสารคำแนะนำ Setup กับช่าง

## การติดตามความคืบหน้า Multi-Op

บนหน้า **Work Order Detail** แถบความคืบหน้าขั้นตอนแสดงการเสร็จสิ้นตลอดทุกการดำเนินงาน ชิ้นงาน 3 Op ที่เสร็จ 2/3 การดำเนินงานแสดง ~67% ความคืบหน้า

บน **Production Kanban** การ์ด WO แสดงชื่อขั้นตอน Active ปัจจุบัน — ดังนั้นคุณสามารถเห็นได้ทันทีว่า `WO-2026-00042` อยู่ที่ "CNC Milling Op20" โดยไม่ต้องเปิดหน้ารายละเอียด

บน **TV Monitor** WO แบบ Multi-Op แสดงขั้นตอนปัจจุบันและเวลาที่ผ่านไป ให้พื้นที่การผลิตมองเห็นแบบ Live ว่าแต่ละชิ้นงานอยู่ที่ไหนในลำดับการดำเนินงาน

## ข้อผิดพลาดที่พบบ่อย

**รวมทุกการดำเนินงานเป็นขั้นตอนเดียว** ขั้นตอน "CNC Milling 7.0h" เดียวสำหรับชิ้นงาน 3 Op สูญเสียความละเอียดทั้งหมด คุณไม่สามารถเห็นว่าการดำเนินงานใดกำลังรัน ไม่สามารถ Rework การดำเนินงานเฉพาะ และ Scheduler ไม่สามารถเรียงลำดับได้อย่างอิสระ

**ไม่ตั้งเวลามาตรฐานต่อการดำเนินงาน** หาก Op10 คือ 3 ชั่วโมงและ Op20 คือ 2 ชั่วโมงแต่คุณป้อน 5 ชั่วโมงสำหรับขั้นตอนเดียว Scheduler บล็อกเครื่องเป็นเวลา 5 ชั่วโมงต่อเนื่อง การแยกเป็นสองขั้นตอนให้ Scheduler อาจใช้เครื่องสำหรับงานอื่นระหว่างการดำเนินงาน (เช่น หากมีการรอ Queue ระหว่าง Op10 และ Op20)

**ใช้ชื่อขั้นตอนเดียวกันสำหรับทุกการดำเนินงาน** "CNC Milling" × 3 สร้างความสับสนสำหรับช่างและทำให้รายงานอ่านยาก ใส่หมายเลขการดำเนินงานในชื่อขั้นตอนเสมอ

**ลืมตั้งค่า Equipment Type** แต่ละขั้นตอนการดำเนินงานต้องมี `equipment_type` ที่ถูกต้องเพื่อให้ Scheduler รู้ว่าเครื่องใดมีสิทธิ์ หาก Op10 ต้องการ 5 แกน Mill แต่ Op20 สามารถรันบน 3 แกนได้ ให้ตั้งค่าต่างกัน — ซึ่งให้ Scheduler มีความยืดหยุ่นมากขึ้น
