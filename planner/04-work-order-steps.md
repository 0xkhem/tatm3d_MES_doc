# 4. Work Orders — การจัดการขั้นตอน (Step Management)

## วงจรชีวิตของขั้นตอน (Step Lifecycle)

ใบสั่งผลิตแต่ละรายการประกอบด้วยลำดับขั้นตอน (operations) ที่เรียงตามลำดับ ขั้นตอนเป็นไปตามวงจรชีวิตนี้:

```
pending → in_progress → completed
  │           │
  └───────────┴──→ skipped
```

---

## กฎการเรียงลำดับขั้นตอน (Step Sequencing Rules)

- ขั้นตอนต้องเสร็จสมบูรณ์**ตามลำดับ** — คุณไม่สามารถเริ่มขั้นตอนที่ 3 ได้จนกว่าขั้นตอนที่ 1 และ 2 จะอยู่ในสถานะสุดท้าย (completed หรือ skipped)
- เมื่อ**ขั้นตอนแรก**เริ่มต้น WO จะเปลี่ยนสถานะจาก Released → In Progress โดยอัตโนมัติ
- เมื่อ**ทุกขั้นตอน**อยู่ในสถานะสุดท้ายและมีอย่างน้อยหนึ่งขั้นตอนที่ completed WO จะเสร็จสมบูรณ์โดยอัตโนมัติ

---

## การดำเนินการของขั้นตอน (Step Actions)

แต่ละขั้นตอนแสดงปุ่มการดำเนินการตามสถานะปัจจุบันและสถานะของ WO:

| สถานะขั้นตอน | การดำเนินการที่ใช้ได้ | เงื่อนไข |
|-------------|-------------------|-----------|
| Pending (รอดำเนินการ) | **Start** (เริ่ม) | ขั้นตอนก่อนหน้าทั้งหมดต้องเป็นสถานะสุดท้าย; WO ต้องเป็น released หรือ in_progress |
| In Progress (กำลังดำเนินการ) | **Complete** (เสร็จสิ้น), **Skip** (ข้าม) | WO ต้องไม่เป็น on_hold หรือ cancelled |
| Completed (เสร็จสิ้น) | (ไม่มี) | สถานะสุดท้าย |
| Skipped (ข้าม) | (ไม่มี) | สถานะสุดท้าย |

ถ้า WO เป็น **Draft**, **On Hold**, **Cancelled**, หรือ **Completed** ปุ่มการดำเนินการทุกขั้นตอนจะแสดงตัวบ่งชี้ 🔒 **Locked** (ล็อค)

[image: step-actions-buttons — ภาพขยายของแถวขั้นตอนแสดงปุ่ม Start, Complete, และ Skip ในสถานะต่างๆ]

---

## การเริ่มขั้นตอน (Starting a Step)

1. ขยาย WO เพื่อดูขั้นตอน
2. ค้นหาขั้นตอน pending ถัดไป (ขั้นตอนก่อนหน้าทั้งหมดต้อง completed/skipped)
3. คลิก **Start** (▶)
4. ขั้นตอนจะเปลี่ยนเป็น **In Progress**
5. ถ้านี่เป็นขั้นตอนแรก WO จะเปลี่ยนเป็น **In Progress** โดยอัตโนมัติ
6. เวลา `actual_start` จะถูกบันทึก

[image: step-start-action — ภาพหน้าจอแสดงปุ่ม Start บนขั้นตอน pending ที่ขั้นตอนก่อนหน้าเสร็จสมบูรณ์แล้ว]

---

## การทำขั้นตอนให้เสร็จสิ้น (Completing a Step)

1. ค้นหาขั้นตอนที่มีสถานะ **In Progress**
2. คลิก **Complete** (✓)
3. ขั้นตอนจะเปลี่ยนเป็น **Completed**
4. เวลา `actual_end` จะถูกบันทึก
5. ถ้าทุกขั้นตอนอยู่ในสถานะสุดท้ายแล้ว WO จะเสร็จสมบูรณ์โดยอัตโนมัติ

---

## การข้ามขั้นตอน (Skipping a Step)

1. ค้นหาขั้นตอนที่มีสถานะ **Pending** หรือ **In Progress**
2. คลิก **Skip** (⏭)
3. ขั้นตอนจะเปลี่ยนเป็น **Skipped** — จะไม่บล็อกขั้นตอนถัดไป

> **กรณีใช้งาน:** ขั้นตอนไม่จำเป็นสำหรับล็อตนี้ (เช่น ไม่ต้องการการเคลือบผิว)

---

## การมอบหมายเครื่องจักรให้กับขั้นตอน (Assigning a Machine to a Step)

แต่ละขั้นตอนสามารถมอบหมายเครื่องจักรได้:

1. ในแถวขั้นตอนที่ขยาย ค้นหาคอลัมน์ **Machine**
2. คลิกดรอปดาวน์ตัวเลือกเครื่องจักร
3. ดรอปดาวน์จะแสดงเครื่องจักรที่กรองตามประเภทอุปกรณ์ที่ขั้นตอนต้องการ
4. เลือกเครื่องจักร
5. การมอบหมายจะถูกบันทึกทันที

[image: step-machine-selector — ภาพขยายของดรอปดาวน์ตัวเลือกเครื่องจักรบนแถวขั้นตอน แสดงเครื่องจักรที่กรองแล้ว]

> **หมายเหตุ:** การมอบหมายเครื่องจักรที่นี่แยกจากการมอบหมายตาราง สิ่งนี้กำหนดว่าเครื่องจักรใดจะดำเนินการขั้นตอน; Scheduling Board กำหนด*เมื่อไหร่*ที่จะทำงาน

---

## การติดตามจำนวน (Quantity Tracking)

สำหรับขั้นตอนที่ติดตามผลผลิต:

| ฟิลด์ | คำอธิบาย |
|-------|-------------|
| Completed Qty (จำนวนที่เสร็จ) | จำนวนชิ้นส่วนที่ดีที่ผลิตได้ในขั้นตอนนี้ |
| Defect Qty (จำนวนของเสีย) | จำนวนชิ้นส่วนที่มีข้อบกพร่องในขั้นตอนนี้ |
| Order Qty (จำนวนสั่ง) | จำนวนรวมที่สั่ง (จาก WO) |

ผู้ปฏิบัติงานรายงานจำนวนผ่าน Operator Panel หรือ Mobile Scan ผู้วางแผนสามารถดูความคืบหน้าได้ที่นี่

---

## เวลาทำงานจริง (Active Run Time)

สำหรับขั้นตอนที่มีสถานะ **In Progress** ระบบจะคำนวณเวลาทำงานจริง:

```
Active Run Time = (now − actual_start) − cumulative_hold_minutes
```

สิ่งนี้ไม่รวมเวลาที่ WO ถูกพัก ทำให้เห็นภาพเวลาการกลึง/ผลิตจริงที่แม่นยำ

[image: step-run-time — ภาพขยายของขั้นตอนที่แสดงตัวบ่งชี้เวลาทำงานจริง]

---

## ขั้นตอนงานส่งผลิตภายนอก (Outside Processing Steps)

ขั้นตอนที่ใช้คำจำกัดความกระบวนการบริการภายนอกจะถูกทำเครื่องหมายด้วยไอคอน 🚚 รถบรรทุก ขั้นตอนเหล่านี้:

- ไม่สามารถเริ่ม/เสร็จสิ้นด้วยตนเองได้ (จัดการผ่านโมดูล Outside Processing)
- แสดงชื่อผู้ขายและระยะเวลานำส่ง (lead time)
- ไม่สามารถลากได้ในแถบด้านข้างของ Scheduling
- ทำหน้าที่เป็นช่วงเวลาว่างในระบบจัดตาราง

[image: step-outside-processing — ภาพขยายของแถวขั้นตอนที่แสดงไอคอนรถบรรทุกและข้อมูลผู้ขายสำหรับขั้นตอนงานส่งผลิตภายนอก]
