# การตรวจสอบชิ้นแรก (First Article Inspection — FAI)

First Article Inspection คือการตรวจสอบชิ้นงานชิ้นแรกจาก Setup ใหม่อย่างครบถ้วนก่อนรัน Batch ที่เหลือ เป็นแนวปฏิบัติมาตรฐานในการกลึงความแม่นยำแบบ HMLV — การพบข้อผิดพลาด Setup บนชิ้นที่ 1 ถูกกว่าการค้นพบบนชิ้นที่ 50 มาก คู่มือนี้อธิบายวิธีจำลอง FAI ใน Process Routes และ Quality Workflow ของ TAM3D

---

## ทำไม FAI ถึงสำคัญใน HMLV

ในการผลิต High-Volume FAI เป็นเหตุการณ์ครั้งเดียวต่อหมายเลขชิ้นงาน ใน HMLV ทุกงานคือ Setup ใหม่โดยพื้นฐาน — Fixture ต่างกัน, โปรแกรมต่างกัน, ช่างต่างกัน FAI ไม่ใช่ตัวเลือก แต่เป็นการป้องกันหลักต่อการเสียหายของ Batch

ต้นทุนของการข้าม FAI:
- ข้อผิดพลาด Offset 0.1mm บน Batch 50 ชิ้น = ชิ้นงาน 50 ชิ้นที่เสียหาย
- ต้นทุน Rework หรือ Remake + ความล่าช้าของลูกค้า
- การตรวจสอบคุณภาพของลูกค้าที่อาจเกิดขึ้น

ต้นทุนของการทำ FAI:
- เวลาตรวจสอบ CMM หรือด้วยมือ 20–60 นาทีต่อ Setup
- ชิ้นงานหนึ่งชิ้นที่ถูกเก็บไว้จนกว่าการตรวจสอบจะผ่าน

การคำนวณชัดเจน ความท้าทายคือการทำให้ FAI เป็นส่วนหนึ่งของ Workflow อย่างเป็นระบบแทนที่จะเป็นการตัดสินใจเฉพาะกิจ

---

## การจำลอง FAI ใน Process Routes

### แนวทางที่ 1: ขั้นตอน FAI เฉพาะใน Route (แนะนำ)

เพิ่มขั้นตอน "First Article Inspection" ที่ชัดเจนทันทีหลังจากการดำเนินงานการกลึงแรก

**ตัวอย่าง: Motor Housing (อลูมิเนียม, 5 แกน)**

| ลำดับ | Process Definition | ชั่วโมงโดยประมาณ | หมายเหตุ |
|-------|-------------------|-----------------|---------|
| 10 | CNC Milling Op10 | 4.0h | Rough + Finish, ชิ้นแรกเท่านั้น |
| 20 | First Article Inspection | 0.5h | CMM ตรวจสอบมิติเต็มรูปแบบ, 1 ชิ้น |
| 30 | CNC Milling Op10 (Batch Run) | 8.0h | รันจำนวนที่เหลือหลัง FAI ผ่าน |
| 40 | Deburring | 1.0h | ทุกชิ้น |
| 50 | CMM Inspection | 0.5h | ตัวอย่าง 10% ของ Batch |
| 60 | Packing | 0.3h | |

**วิธีการทำงาน:**
- ขั้นตอน 10 กลึงชิ้นแรก
- ขั้นตอน 20 คือ FAI — ผู้ตรวจสอบ QC วัดชิ้นแรกอย่างครบถ้วน
- หาก FAI ผ่าน ขั้นตอน 30 รัน Batch ที่เหลือ
- หาก FAI ล้มเหลว NCR ถูกสร้างและ Setup ถูกแก้ไขก่อนรัน Batch

**ข้อดี:** FAI เป็น Gate บังคับใน Workflow ช่างไม่สามารถเริ่ม Batch Run (ขั้นตอน 30) จนกว่าขั้นตอน FAI (ขั้นตอน 20) จะเสร็จสิ้น ระบบบังคับสิ่งนี้ผ่านกฎการเรียงลำดับขั้นตอน

### แนวทางที่ 2: FAI เป็นส่วนหนึ่งของขั้นตอนการกลึงแรก

สำหรับชิ้นงานที่ง่ายกว่าหรือเมื่อช่างคนเดียวกันทำทั้งการกลึงและการตรวจสอบ:

| ลำดับ | Process Definition | ชั่วโมงโดยประมาณ | หมายเหตุ |
|-------|-------------------|-----------------|---------|
| 10 | CNC Milling + FAI | 4.5h | กลึงชิ้นแรก, ตรวจสอบ, แล้วรัน Batch |
| 20 | Deburring | 0.5h | |
| 30 | CMM Inspection | 0.3h | ตรวจสอบตัวอย่าง |
| 40 | Packing | 0.2h | |

**ข้อดี:** Route ง่ายกว่า ขั้นตอนน้อยกว่า
**ข้อเสีย:** FAI ไม่ใช่เหตุการณ์ที่ติดตามแยกต่างหาก หากชิ้นแรกล้มเหลว คุณไม่สามารถสร้าง NCR กับขั้นตอน "FAI" เฉพาะ — มันถูกฝังอยู่ในขั้นตอนการกลึง ใช้แนวทางที่ 1 สำหรับชิ้นงานใดก็ตามที่ความล้มเหลวของ FAI เป็นความเสี่ยงจริง

## Workflow FAI ในทางปฏิบัติ

### ขั้นตอนที่ 1 — ช่างกลึงชิ้นแรก

ช่างเริ่มขั้นตอน 10 (CNC Milling Op10) บน Operator Panel พวกเขากลึงชิ้นหนึ่งและรายงานการเสร็จสิ้นบางส่วน (จำนวน: 1)

จากนั้น Hold ขั้นตอนหรือเสร็จสิ้นพร้อมหมายเหตุ: "ชิ้นแรกพร้อมสำหรับ FAI"

### ขั้นตอนที่ 2 — ผู้ตรวจสอบ QC ทำการตรวจสอบ

ผู้ตรวจสอบ QC เปิด **Quality → Inspections → Create**

กรอก:
- **Project:** เชื่อมโยงกับ Project
- **Work Order:** เชื่อมโยงกับ WO
- **Step:** เชื่อมโยงกับขั้นตอน FAI (ขั้นตอน 20)
- **Sample Qty:** 1
- **Inspected Qty:** 1
- **Result:** Pass / Fail / Conditional Pass

สำหรับ FAI เต็มรูปแบบ ผู้ตรวจสอบวัดทุกมิติสำคัญบนแบบ บันทึกผลลัพธ์และรายละเอียดข้อบกพร่องใดๆ

### ขั้นตอนที่ 3a — FAI ผ่าน

ผู้ตรวจสอบ QC ทำเครื่องหมาย Inspection เป็น **Pass**

ขั้นตอน FAI (ขั้นตอน 20) สามารถเสร็จสิ้นได้แล้ว ช่างดำเนินการต่อไปยังขั้นตอน 30 (Batch Run)

ระบบบันทึกการผ่าน FAI กับ WO ซึ่งมองเห็นได้ในหน้า Order Traceability และรายละเอียด WO

### ขั้นตอนที่ 3b — FAI ล้มเหลว

ผู้ตรวจสอบ QC ทำเครื่องหมาย Inspection เป็น **Fail**

1. สร้าง **NCR** (Quality → NCR → Create):
   - เชื่อมโยงกับ WO และ Inspection ที่ล้มเหลว
   - Defect Type: Dimensional (พบบ่อยที่สุดสำหรับความล้มเหลวของ FAI)
   - Root Cause: Setup Error / Program Offset / Fixture Issue
   - Disposition: **Rework** (แก้ไข Setup และกลึงชิ้นแรกใหม่)

2. มอบหมาย NCR ให้วิศวกรหรือผู้ทำ Setup ที่รับผิดชอบ

3. วิศวกรแก้ไข Setup (ปรับ Offset, จับยึดใหม่, อัปเดตโปรแกรม)

4. NCR ถูก Resolve ด้วย Rework Disposition ระบบ Reset ขั้นตอน FAI กลับเป็น Pending

5. ช่างกลึงชิ้นแรกใหม่และ FAI ถูกทำซ้ำ

**สำคัญ:** Batch Run (ขั้นตอน 30) ถูกบล็อกจนกว่าขั้นตอน FAI จะเสร็จสิ้น ช่างไม่สามารถข้ามไปข้างหน้าได้ ซึ่งถูกบังคับโดยกฎการเรียงลำดับขั้นตอน — ขั้นตอนไม่สามารถเริ่มได้จนกว่าขั้นตอนก่อนหน้าทั้งหมดจะเป็น Terminal (เสร็จสิ้นหรือข้าม)

## FAI สำหรับขั้นตอน Outside Processing

เมื่อชิ้นงานออกไปรับการชุบผิว (Anodizing, Plating) และกลับมา การตรวจสอบเทียบเท่า FAI เป็นแนวปฏิบัติมาตรฐานก่อนรับ Batch

จำลองสิ่งนี้เป็นขั้นตอนการตรวจสอบทันทีหลังจากขั้นตอน Outside Processing กลับมา:

| ลำดับ | Process Definition | หมายเหตุ |
|-------|-------------------|---------|
| 30 | Surface Treatment | Outside Processing (OPO สร้างอัตโนมัติ) |
| 40 | Post-Treatment Inspection | ตรวจสอบความหนาของ Coating, การยึดเกาะ, สี |
| 50 | CMM Inspection | ตรวจสอบมิติที่ไม่ได้รับผลกระทบจากการชุบ |

ขั้นตอน Post-Treatment Inspection เชื่อมโยงกับ Equipment Type ของ QC Station เมื่อ OPO ถูกทำเครื่องหมายว่าเสร็จสิ้น (ชิ้นงานกลับจากซัพ) ขั้นตอนการตรวจสอบจะพร้อมใช้งาน

## การติดตามประสิทธิภาพ FAI

ใช้ **Quality Dashboard** (Quality → Dashboard) เพื่อติดตาม Metric ที่เกี่ยวข้องกับ FAI:

- **อัตราผ่านตาม Process Definition:** หาก "CNC Milling Op10" มีอัตราผ่านครั้งแรก 70% กระบวนการ Setup ของคุณต้องการความสนใจ
- **การกระจายข้อบกพร่อง:** ความล้มเหลวของ FAI ส่วนใหญ่เป็น Dimensional หรือ Surface? ซึ่งชี้ไปยังสาเหตุรากฐานเฉพาะ (Offset กับ Tooling กับวัสดุ)
- **จำนวน Rework ต่อ WO:** ฟิลด์ `rework_count` บน WO เพิ่มขึ้นกับแต่ละ Rework จำนวน Rework สูงบนชิ้นงานประเภทเฉพาะบ่งชี้ความไม่เสถียรของกระบวนการ

กรองรายการ NCR ตาม Defect Type "Dimensional" และ Root Cause "Setup Error" เพื่อดูประวัติความล้มเหลวของ FAI

## การตั้งค่า FAI Process Definitions

สร้างสิ่งเหล่านี้ใน **Knowledge → Process Definitions**:

| ชื่อ | Equipment Type | Setup Group | เวลามาตรฐาน |
|-----|---------------|-------------|------------|
| First Article Inspection | CMM / Inspection Station | QC-Station | 30 นาที |
| Post-Treatment Inspection | Inspection Station | QC-Station | 20 นาที |
| In-Process Inspection | Inspection Station | QC-Station | 15 นาที |

มอบหมายสิ่งเหล่านี้ให้ Equipment Type ของบทบาท QC Inspector เพื่อให้ปรากฏใน Task Queue ของผู้ตรวจสอบ QC ไม่ใช่ Queue ของช่างกลึง

## เอกสาร FAI

สำหรับลูกค้าที่ต้องการเอกสาร FAI (พบบ่อยในยานยนต์และ Aerospace):

1. ระเบียน Inspection ใน TAM3D จับผลการวัด, ผู้ตรวจสอบ, วันที่ และสถานะผ่าน/ล้มเหลว
2. Export ข้อมูลการตรวจสอบผ่านโมดูล Quality เพื่อรวมใน First Article Inspection Report (FAIR) ของคุณ
3. เชื่อมโยงการแก้ไขแบบที่อนุมัติกับ WO ผ่าน Drawing Manager เพื่อให้ FAI สามารถติดตามย้อนกลับไปยังการแก้ไขแบบเฉพาะ

สำหรับข้อกำหนด FAI แบบ AS9102 หรือ PPAP TAM3D จับข้อมูลหลัก (ผลมิติ, ระเบียนการตรวจสอบ, ประวัติ NCR) ที่ป้อนเข้าสู่แพ็คเกจเอกสาร FAI อย่างเป็นทางการของคุณ

## อ้างอิงด่วน: Decision Tree ของ FAI

```
งานใหม่หรือ Setup ใหม่?
    ↓ ใช่
เพิ่มขั้นตอน FAI ใน Process Route (ลำดับ 20, หลังการดำเนินงานกลึงแรก)
    ↓
ช่างกลึงชิ้นแรก
    ↓
ผู้ตรวจสอบ QC สร้างระเบียน Inspection
    ↓
ผ่าน? ──→ เสร็จสิ้นขั้นตอน FAI → ช่างรัน Batch
    ↓ ล้มเหลว
สร้าง NCR → Disposition: Rework
    ↓
วิศวกรแก้ไข Setup
    ↓
NCR Resolved → ขั้นตอน FAI Reset → ทำซ้ำการตรวจสอบ
```
