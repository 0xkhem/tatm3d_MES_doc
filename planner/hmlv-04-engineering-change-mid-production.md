# การเปลี่ยนแปลงทางวิศวกรรมระหว่างการผลิต

ลูกค้าส่งแบบแก้ไขหลังจาก Work Order เริ่มดำเนินการแล้ว หรือวิศวกรของคุณพบข้อผิดพลาดด้านค่าความเผื่อที่ต้องการการกลึงใหม่ คู่มือนี้ครอบคลุมวิธีจัดการการเปลี่ยนแปลงทางวิศวกรรม (EC) โดยไม่สูญเสียประวัติการผลิตหรือสร้างความวุ่นวายในการจัดตาราง

---

## ปัญหา

เมื่อการแก้ไขแบบมาถึงระหว่างการผลิต คุณเผชิญกับคำถามหลายข้อ:

- ชิ้นงานที่กลึงแล้วสามารถนำมาใช้ได้หรือต้องทิ้ง?
- ขั้นตอนใดต้องทำซ้ำ?
- จะอัปเดต Process Route โดยไม่สูญเสียประวัติของสิ่งที่ทำไปแล้วได้อย่างไร?
- จะสื่อสารการเปลี่ยนแปลงกับช่างได้อย่างไร?

Workflow Hold → Update → Resume ของ TAM3D จัดการสิ่งนี้ได้อย่างสะอาด

---

## Workflow: การเปลี่ยนแปลงทางวิศวกรรมบน WO ที่กำลังดำเนินการ

### ขั้นตอนที่ 1 — Hold Work Order

ก่อนทำการเปลี่ยนแปลงใดๆ ให้หยุดการผลิตบน WO ที่ได้รับผลกระทบ

1. เปิด Work Order
2. คลิก **Hold**
3. เลือกหรือป้อนเหตุผลการ Hold: "Engineering Change" หรือ "Drawing Revision"
4. สถานะ WO เปลี่ยนเป็น `on_hold` Task Queue ของช่างจะแสดง WO ว่า Hold พร้อมเหตุผลที่แสดง

Timestamp การ Hold ถูกบันทึก เวลาที่ WO ใช้ใน Hold ถูกติดตามแยกจาก Active Run Time ดังนั้น Metric การผลิตของคุณยังคงแม่นยำ

### ขั้นตอนที่ 2 — ประเมินผลกระทบ

ตรวจสอบความคืบหน้าขั้นตอนของ WO เพื่อเข้าใจว่าทำอะไรไปแล้ว:

- ขั้นตอนที่ทำเครื่องหมาย `completed` — งานเสร็จแล้ว ตัดสินใจว่าขั้นตอนเหล่านี้ต้องทำซ้ำตาม EC หรือไม่
- ขั้นตอน `in_progress` ปัจจุบัน — งานถูกขัดจังหวะ ขั้นตอนนี้ต้องได้รับการประเมินใหม่
- ขั้นตอนที่ยัง `pending` — ยังไม่เริ่ม สามารถอัปเดตได้อย่างอิสระ

**สามสถานการณ์:**

| สถานการณ์ | สิ่งที่เกิดขึ้น | การดำเนินการ |
|-----------|--------------|-------------|
| EC ส่งผลต่อขั้นตอน Pending เท่านั้น | การกลึงจนถึงตอนนี้ไม่มีปัญหา | อัปเดตขั้นตอน Pending, Resume |
| EC ต้องการการกลึงขั้นตอนที่เสร็จแล้วใหม่ | ชิ้นงานต้องการ Rework | ใช้เส้นทาง NCR Rework |
| EC ทำให้ชิ้นงานทั้งหมดเป็น Scrap | ชิ้นงานใช้ไม่ได้ | ใช้เส้นทาง NCR Remake |

### ขั้นตอนที่ 3a — EC ส่งผลต่อขั้นตอน Pending เท่านั้น (พบบ่อยที่สุด)

หากการเปลี่ยนแปลงทางวิศวกรรมส่งผลต่อการดำเนินงานที่ยังไม่เริ่มเท่านั้น (เช่น การเปลี่ยนค่าความเผื่อบนฟีเจอร์ที่กลึงในขั้นตอน 40 และคุณกำลังอยู่ที่ขั้นตอน 20):

1. เปิด Work Order → **Edit**
2. อัปเดตขั้นตอนที่เกี่ยวข้อง: เปลี่ยน Process Definition, อัปเดตชั่วโมงที่ประมาณ, เพิ่มหมายเหตุพร้อมหมายเลขอ้างอิง EC
3. หาก Process Route เองต้องเปลี่ยน (เช่น ต้องการขั้นตอน Grinding ใหม่) ให้อัปเดต Process Route ใน Knowledge ก่อน แล้วเชื่อมโยงใหม่กับ WO
4. บันทึก WO
5. Resume WO: คลิก **Resume** (หรือเปลี่ยนสถานะกลับเป็น `in_progress`)

ช่างทำงานต่อจากที่หยุด ขั้นตอนที่อัปเดตปรากฏใน Task Queue พร้อมคำแนะนำใหม่

### ขั้นตอนที่ 3b — EC ต้องการการกลึงขั้นตอนที่เสร็จแล้วใหม่ (Rework)

หากการเปลี่ยนแปลงทางวิศวกรรมหมายความว่าขั้นตอนที่เสร็จแล้วบางขั้นตอนต้องทำซ้ำ:

1. สร้าง **Quality Inspection** สำหรับชิ้นงานที่ได้รับผลกระทบ (Quality → Inspections → Create)
   - Result: Fail
   - Defect Reason: "Engineering Change — Drawing Revision Rev X"

2. สร้าง **NCR** (Quality → NCR → Create)
   - เชื่อมโยงกับ WO และ Inspection ที่ล้มเหลว
   - Disposition: **Rework**
   - มอบหมายให้วิศวกรที่รับผิดชอบ

3. Resolve NCR ด้วย Rework ระบบจะ:
   - Reset ขั้นตอนที่ล้มเหลวและขั้นตอนถัดไปทั้งหมดกลับเป็น `pending` โดยอัตโนมัติ
   - ล้างเวลาเริ่มต้น/สิ้นสุดจริงบนขั้นตอนที่ Reset
   - ตั้ง WO กลับเป็น `in_progress`
   - เพิ่ม `rework_count` บน WO เพื่อ Traceability

4. อัปเดตขั้นตอนกระบวนการด้วยคำแนะนำ/ค่าความเผื่อที่แก้ไข

5. Task Queue ของช่างตอนนี้แสดงขั้นตอนที่ Reset เป็น Pending พวกเขาเริ่มใหม่จากจุด Rework

### ขั้นตอนที่ 3c — EC ทำให้ชิ้นงานทั้งหมดเป็น Scrap (Remake)

หากการเปลี่ยนแปลงทางวิศวกรรมเผยให้เห็นว่าชิ้นงานที่กลึงแล้วผิดพลาดโดยพื้นฐานและไม่สามารถนำมาใช้ได้:

1. สร้าง Inspection และ NCR ตามด้านบน
2. ตั้ง Disposition: **Remake**
3. Resolve NCR ระบบจะ:
   - ปิด WO ต้นฉบับ
   - สร้าง Child WO ใหม่พร้อม Process Route และจำนวนเดียวกัน
   - เชื่อมโยง Child WO กับ NCR เพื่อ Traceability เต็มรูปแบบ
   - สร้าง Scrap Inventory Transaction หาก WO มี Material เชื่อมโยง

4. อัปเดตขั้นตอนของ Child WO ด้วย Route/คำแนะนำที่แก้ไข

5. จัดตาราง Child WO บน Scheduling Board จะปรากฏใน Unscheduled Sidebar

## การอัปเดตแบบ

หากการเปลี่ยนแปลงทางวิศวกรรมมาพร้อมไฟล์แบบที่แก้ไข:

1. ไปที่ **Drawing Manager** (เข้าถึงได้จาก Sidebar)
2. ค้นหาแบบที่เชื่อมโยงกับ WO หรือชิ้นงานนี้
3. อัปโหลดไฟล์แบบที่แก้ไข เพิ่มหมายเหตุเวอร์ชัน (เช่น "Rev B — การเปลี่ยนค่าความเผื่อบนเส้นผ่านศูนย์กลาง Bore ตาม EC-2026-047")

แบบที่อัปเดตจะมองเห็นได้ทันทีสำหรับช่างเมื่อพวกเขาเปิดรายละเอียดขั้นตอนของ WO

---

## การสื่อสารการเปลี่ยนแปลงกับช่าง

เมื่อ WO ถูก Resume (หรือ Remake WO ถูก Release และจัดตาราง):

- Task Queue ของ **Operator Panel** ของช่างอัปเดตโดยอัตโนมัติ
- หากขั้นตอนถูก Reset ผ่าน Rework ช่างจะเห็นขั้นตอนเหล่านั้นเป็น Pending อีกครั้ง
- มุมมองรายละเอียดขั้นตอนแสดงหมายเหตุที่คุณเพิ่มระหว่างการอัปเดต — ใช้สิ่งนี้เพื่อสื่อสารหมายเลขอ้างอิง EC และสิ่งที่เปลี่ยนแปลง

สำหรับการสื่อสารเร่งด่วน ใช้ระบบ **Abnormal Report**: ยื่น Abnormal Report บน WO พร้อมรายละเอียด EC ซึ่งสร้างระเบียนที่มี Timestamp และปรากฏใน Attention Panel บน Home Dashboard

---

## การติดตามการเปลี่ยนแปลงทางวิศวกรรมตลอดเวลา

ทุก Rework และ Remake ถูกบันทึกบน WO:

- ฟิลด์ `rework_count` บน WO เพิ่มขึ้นกับแต่ละ Rework
- ระเบียน NCR เชื่อมโยง EC กับ WO และขั้นตอนเฉพาะ
- หน้า **Order Traceability** แสดงประวัติทั้งหมด: WO ต้นฉบับ → NCR → Rework/Remake → Child WO

ใช้ **Quality Dashboard** เพื่อติดตามอัตรา Rework ตาม Process Definition หากการดำเนินงานเฉพาะสร้าง EC อย่างสม่ำเสมอ อาจบ่งชี้ปัญหาการเสนอราคาหรือการวางแผนกระบวนการ Upstream

---

## การป้องกันการหยุดชะงักจาก EC

**ล็อกแบบก่อน Release WO** กำหนดนโยบายว่า WO จะถูก Release เฉพาะหลังจากแบบได้รับการอนุมัติอย่างเป็นทางการ ใช้ Drawing Manager เพื่อติดตามสถานะการแก้ไข

**ใช้กระบวนการ Quote สำหรับการจับต้นทุน EC** เมื่อลูกค้าขอ EC ให้สร้าง Quote Line Item ที่แก้ไขเพื่อจับต้นทุนการกลึงเพิ่มเติม ซึ่งป้องกันไม่ให้งาน EC ถูกดูดซับเป็นต้นทุนที่ไม่ได้ติดตาม

**ตั้งค่า "Frozen Zone" บน Scheduling Board** พารามิเตอร์ `frozen_days` ของ Scheduling Engine ปกป้องการมอบหมายระยะใกล้จากการถูกย้ายระหว่างการ Re-optimization ซึ่งหมายความว่า EC บน WO หนึ่งจะไม่เลื่อนงานอื่นที่กำลังจะเริ่มโดยไม่ตั้งใจ
