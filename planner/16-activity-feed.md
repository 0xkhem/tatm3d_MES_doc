# 16. Activity Feed — ศูนย์กลางการสื่อสาร Work Order

## ภาพรวม

Activity Feed ให้ประวัติที่สมบูรณ์ของ Work Order ทุกรายการ — เหตุการณ์ระบบและความคิดเห็นของทีมในไทม์ไลน์เดียว เปิดได้จากแผ่นรายละเอียด WO, Scheduling Board, Kanban หรือมุมมอง Traceability

---

## การเปิด Activity Feed

มองหาไอคอน 💬 พร้อมป้ายจำนวน ปรากฏในหลายตำแหน่ง:

### จากหน้า Work Orders

1. เปิด **Production → Work Orders**
2. คลิกแถว WO ใดก็ได้เพื่อขยาย หรือเปิดแผ่นรายละเอียด
3. คลิกปุ่ม **Activity** (ไอคอน 💬 + จำนวน) ในส่วนการดำเนินการของส่วนหัว
4. Activity Sheet จะเลื่อนเข้ามาจากด้านขวา

[image: activity-feed-wo-detail — ภาพหน้าจอของแผ่นรายละเอียด WO พร้อมปุ่ม Activity ที่ไฮไลท์ในส่วนหัว]

### จาก Scheduling Board

1. เปิด **Production → Scheduling Board**
2. คลิกแถบการดำเนินงานใดก็ได้บนไทม์ไลน์
3. ในป๊อปอัพรายละเอียดการดำเนินงาน คลิกไอคอน **Activity**

[image: activity-feed-scheduling — ภาพหน้าจอของป๊อปอัพรายละเอียดการดำเนินงานพร้อมไอคอน Activity]

### จาก Kanban Board

1. เปิด **Production → Work Orders → Kanban**
2. การ์ด WO แต่ละใบแสดงไอคอนกิจกรรมพร้อมป้ายจำนวนในส่วนท้าย
3. คลิกเพื่อเปิดฟีดสำหรับ WO นั้น

[image: activity-feed-kanban — ภาพหน้าจอของการ์ด Kanban พร้อมไอคอนกิจกรรมและป้ายจำนวน]

---

## การอ่านฟีด

ฟีดแสดงรายการทั้งหมดตามลำดับเวลาย้อนกลับ (ใหม่สุดก่อน)

### เหตุการณ์ระบบ (System Events)

รายการบรรทัดเดียวแบบกะทัดรัดพร้อมไอคอนจาง:

```
↔ สถานะเปลี่ยนจาก Released เป็น In Progress — กฤษ — 2 ชม. ที่แล้ว
📅 จัดตาราง: CNC Milling Op10 → 5-Axis Mill, จันทร์ 08:00 — กฤษ — 2 ชม. ที่แล้ว
✅ ขั้นตอนเสร็จสิ้น: CNC Milling Op10 — สมชาย — 1 ชม. ที่แล้ว
🚚 OPO ส่งออกไปยัง Anodizing Co. — กฤษ — 30 นาทีที่แล้ว
```

แต่ละเหตุการณ์แสดง: ไอคอน, คำอธิบาย, ชื่อผู้ดำเนินการ, และเวลาสัมพัทธ์

[image: activity-feed-system-events — ภาพหน้าจอของฟีดแสดงแถวเหตุการณ์ระบบหลายรายการ]

### ความคิดเห็น (Comments)

รายการรูปแบบการ์ดพร้อมข้อมูลผู้เขียนและเนื้อหา:

```
┌─────────────────────────────────────────────┐
│ มล  มาลี · QC Inspector · 45 นาทีที่แล้ว [QC] │
│                                              │
│ ชิ้นแรกวัดแล้ว — ขนาดรูเกินค่าพิกัด 0.02mm    │
│ พักรอวิศวกรตรวจสอบ                             │
│                                              │
│ 📷 [รูปย่อ] [รูปย่อ]                           │
│ 📎 inspection-report.pdf (2.1 MB)            │
└─────────────────────────────────────────────┘
```

แต่ละความคิดเห็นแสดง: อักษรย่อผู้เขียน, ชื่อ, ป้ายบทบาท, เวลา, แท็กหมวดหมู่, ข้อความ, รูปย่อ (คลิกเพื่อขยาย), และลิงก์ไฟล์ (คลิกเพื่อดาวน์โหลด)

[image: activity-feed-comments — ภาพหน้าจอของฟีดแสดงการ์ดความคิดเห็นพร้อมรูปภาพและไฟล์แนบ]

---

## การโพสต์ความคิดเห็น

1. ที่ด้านล่างของ Activity Sheet ค้นหาพื้นที่ป้อนความคิดเห็น
2. พิมพ์บันทึกของคุณในช่องข้อความ (ข้อความตัวอย่าง: "เพิ่มบันทึก...")
3. ตัวเลือกเพิ่มเติม:
   - คลิก 📷 เพื่อแนบรูปภาพ (สูงสุด 5 รูป, สูงสุด 10MB ต่อรูป)
   - คลิก 📎 เพื่อแนบไฟล์ (สูงสุด 3 ไฟล์, สูงสุด 25MB ต่อไฟล์)
   - เลือก **แท็กหมวดหมู่** จากดรอปดาวน์ (QC, โลจิสติกส์, วิศวกรรม, ลูกค้า, งานด่วน, เหตุผลหยุด, ทั่วไป)
4. คลิก **ส่ง**
5. ความคิดเห็นจะปรากฏที่ด้านบนของฟีด

[image: activity-feed-comment-input — ภาพหน้าจอของพื้นที่ป้อนความคิดเห็นพร้อมปุ่มรูปภาพ/ไฟล์และดรอปดาวน์หมวดหมู่]

### การแนบรูปภาพ

รูปภาพเป็นไฟล์แนบที่พบบ่อยที่สุด — หลักฐาน QC, สภาพชิ้นงาน, รูปการตั้งค่า

1. คลิกปุ่ม 📷 กล้อง
2. เลือกรูปภาพหนึ่งรูปหรือมากกว่า (JPEG, PNG, WebP)
3. รูปย่อตัวอย่างจะปรากฏด้านล่างช่องข้อความ คลิก ✕ เพื่อลบ
4. ส่งความคิดเห็น รูปภาพจะถูกเก็บถาวรกับ WO

[image: activity-feed-photo-preview — ภาพหน้าจอแสดงรูปย่อตัวอย่างในพื้นที่ป้อนก่อนส่ง]

### การแนบไฟล์

สำหรับเอกสารเช่น รายงานการตรวจสอบ, แบบที่อัปเดต, หรือการติดต่อกับผู้รับเหมาช่วง

1. คลิกปุ่ม 📎 คลิปหนีบกระดาษ
2. เลือกไฟล์ (PDF, DOCX, XLSX, DXF, STEP — สูงสุด 25MB ต่อไฟล์)
3. ชิปชื่อไฟล์จะปรากฏด้านล่างช่องข้อความ
4. ส่งความคิดเห็น

---

## การกรองฟีด

ใช้แถบกรองที่ด้านบนของฟีด:

| ตัวกรอง | แสดง |
|--------|-------|
| ทั้งหมด | ทุกอย่าง (ค่าเริ่มต้น) |
| เหตุการณ์ | เหตุการณ์ระบบเท่านั้น |
| ความคิดเห็น | ความคิดเห็นของผู้ใช้เท่านั้น |
| QC | ความคิดเห็นที่แท็ก QC |
| โลจิสติกส์ | ความคิดเห็นที่แท็กโลจิสติกส์ |
| วิศวกรรม | ความคิดเห็นที่แท็กวิศวกรรม |
| ลูกค้า | ความคิดเห็นที่แท็กลูกค้า |
| งานด่วน | ความคิดเห็นที่แท็กงานด่วน |

ใช้ประโยชน์เมื่อต้องการดูเฉพาะบันทึกที่เกี่ยวกับ QC หรือเฉพาะเหตุการณ์ระบบเพื่อเข้าใจประวัติสถานะ

[image: activity-feed-filters — ภาพหน้าจอของแถบกรองพร้อมปุ่มแท็กหมวดหมู่]

---

## การลบความคิดเห็น

คุณสามารถลบความคิดเห็นของตัวเองได้ภายใน 15 นาทีหลังโพสต์:

1. ค้นหาความคิดเห็นของคุณในฟีด
2. คลิกปุ่ม **🗑 ลบ** (มองเห็นได้เฉพาะบนความคิดเห็นของคุณเอง ภายใน 15 นาที)
3. ความคิดเห็นจะถูกลบ

หลังจาก 15 นาที ความคิดเห็นจะกลายเป็นถาวรเพื่อความสมบูรณ์ของบันทึกการตรวจสอบ

---

## กรณีใช้งานสำหรับผู้วางแผน (Planner)

### การสื่อสารงานด่วน (Rush Order)

เมื่อลูกค้าโทรมาเร่งงาน:

1. เปิด Activity Feed ของ WO
2. โพสต์ความคิดเห็นแท็ก **งานด่วน**: "ลูกค้าโทรมา — ต้องการ 10 ชิ้นภายในวันศุกร์ เพิ่มความสำคัญ"
3. เปลี่ยนความสำคัญ WO ระบบจะบันทึก: "ความสำคัญเปลี่ยนจาก Normal เป็น High"
4. จัดตารางใหม่บนบอร์ด ระบบจะบันทึก: "ย้ายตาราง: Op10 → CNC Mill #3, พุธ 08:00"

ตอนนี้ทุกคนที่ดู WO นี้จะเห็นบริบททั้งหมด: ทำไมความสำคัญเปลี่ยน ใครร้องขอ และตารางถูกปรับอย่างไร

### การติดตามความล่าช้าของผู้รับเหมาช่วง

เมื่อผู้รับเหมาช่วงส่งงานช้า:

1. ระบบบันทึกอัตโนมัติ: "OPO เกินกำหนด — คาดว่าจะกลับ 3 เม.ย. ยังไม่ได้รับ"
2. โพสต์ความคิดเห็นแท็ก **โลจิสติกส์**: "โทรหา SurfaceTech — ชิ้นงานล่าช้าเนื่องจากปัญหาอุปกรณ์ ETA ใหม่: 7 เม.ย."
3. เมื่อชิ้นงานมาถึง ระบบบันทึก: "OPO เสร็จสิ้น — รับ 7 เม.ย."

### การบันทึกการเปลี่ยนแปลงทางวิศวกรรม

เมื่อแบบฉบับแก้ไขมาถึงระหว่างการผลิต:

1. โพสต์ความคิดเห็นแท็ก **วิศวกรรม**: "ได้รับแบบ Rev B — เปลี่ยนค่าพิกัดขนาดรูตาม EC-2026-047"
2. แนบไฟล์แบบที่อัปเดต (📎 drawing-rev-b.pdf)
3. พัก WO ระบบบันทึก: "สถานะเปลี่ยนเป็น On Hold — เหตุผล: Engineering Change"
4. หลังอัปเดตขั้นตอน กลับมาทำงาน ระบบบันทึก: "สถานะเปลี่ยนเป็น In Progress"
