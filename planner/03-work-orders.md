# 3. Work Orders — รายการและการจัดการ

## การเข้าถึง Work Orders

นำทางไปที่ **Production → Work Orders** หรือไปที่ `/production/work-orders`

[image: work-orders-page-full — ภาพหน้าจอเต็มของหน้า Work Orders แสดงแท็บสถานะ, แถบเครื่องมือ, และตาราง WO]

---

## เลย์เอาต์ของหน้า (Page Layout)

### แท็บสถานะ (Status Tabs)

แท็บแนวนอนด้านบนกรอง WO ตามสถานะ:

| แท็บ | แสดง |
|-----|-------|
| All (ทั้งหมด) | ใบสั่งผลิตทุกรายการ |
| Draft (ร่าง) | ยังไม่ได้ปล่อย |
| Released (ปล่อยแล้ว) | พร้อมสำหรับการจัดตาราง |
| In Progress (กำลังดำเนินการ) | กำลังทำงานอยู่ |
| On Hold (พักงาน) | หยุดชั่วคราว |
| Completed (เสร็จสิ้น) | เสร็จแล้ว |
| Cancelled (ยกเลิก) | ยกเลิกแล้ว |

คลิกแท็บเพื่อกรอง จำนวนของแต่ละสถานะจะแสดงให้เห็น

[image: work-orders-status-tabs — ภาพขยายของแถบแท็บสถานะพร้อมจำนวน]

### แถบเครื่องมือ (Toolbar)

- ปุ่ม **+ Work Order** — สร้าง WO ใหม่ด้วยตนเอง
- **กล่องค้นหา** — กรองตามรหัส WO, ชื่อผลิตภัณฑ์, หรือลูกค้า
- ปุ่ม **Export** — ดาวน์โหลดมุมมองปัจจุบันเป็น CSV
- **การตั้งค่าคอลัมน์** — สลับคอลัมน์ที่มองเห็น

[image: work-orders-toolbar — ภาพขยายของแถบเครื่องมือพร้อมปุ่มการดำเนินการและการค้นหา]

## ตาราง Work Order

| คอลัมน์ | คำอธิบาย |
|--------|-------------|
| Thumbnail (รูปย่อ) | มุมมอง Isometric ของชิ้นส่วน (จากใบเสนอราคา/SO ที่เชื่อมโยง) |
| Order Code (รหัสคำสั่ง) | รหัส WO (เช่น `WO-2026-00001`) แบบอักษร Monospace |
| Product (ผลิตภัณฑ์) | ชื่อชิ้นส่วนและสเปค |
| Customer (ลูกค้า) | ชื่อลูกค้า |
| Qty (จำนวน) | จำนวนสั่งผลิต |
| Priority (ความสำคัญ) | ป้าย: urgent (แดง), high (ส้ม), normal (เขียว), low (เทา) |
| Status (สถานะ) | ป้ายสถานะแบบมีสี |
| Steps (ขั้นตอน) | ตัวบ่งชี้ความคืบหน้า (เช่น "3/6 steps") |
| Due Date (วันกำหนด) | วันที่เป้าหมายเสร็จสมบูรณ์พร้อมตัวบ่งชี้เกินกำหนด |
| Actions (การดำเนินการ) | เมนูดรอปดาวน์พร้อมการดำเนินการที่ใช้ได้ |

[image: work-orders-table — ภาพหน้าจอของตารางแสดง 4-5 ตัวอย่าง WO พร้อมสถานะและความสำคัญที่แตกต่างกัน]

## การเปลี่ยนสถานะ Work Order (Work Order Status Transitions)

ผู้วางแผนสามารถเปลี่ยนสถานะ WO ผ่านเมนูดรอปดาวน์การดำเนินการ:

### กฎการเปลี่ยนสถานะ (Transition Rules)

| จาก | ไป | เงื่อนไข |
|------|-----|-------|
| Draft → Released | WO ต้องมีอย่างน้อย 1 ขั้นตอน |
| Released → In Progress | (เปลี่ยนอัตโนมัติเมื่อขั้นตอนแรกเริ่ม) |
| Released → On Hold | ต้องระบุเหตุผลการพัก |
| In Progress → On Hold | ต้องระบุเหตุผลการพัก |
| On Hold → Released | กลับมาทำงานต่อ |
| On Hold → In Progress | กลับมาทำงานต่อ |
| Any → Completed | ทุกขั้นตอนต้องเป็นสถานะสุดท้าย (completed หรือ skipped) |
| Any → Cancelled | ต้องระบุเหตุผลการยกเลิก |

[image: work-orders-status-actions — ภาพหน้าจอของเมนูดรอปดาวน์การดำเนินการบน WO แสดงการเปลี่ยนสถานะที่ใช้ได้]

### การปล่อย Work Order (Releasing a Work Order)

1. ค้นหา WO ที่มีสถานะ **Draft** ในตาราง
2. คลิกเมนูการดำเนินการ (⋯)
3. เลือก **Release**
4. WO จะเปลี่ยนเป็นสถานะ **Released** และพร้อมสำหรับการจัดตาราง

[image: work-orders-release — ภาพหน้าจอแสดงการดำเนินการ Release ในเมนูดรอปดาวน์]

### การพัก WO (Putting a WO On Hold)

1. คลิกเมนูการดำเนินการบน WO ที่ดำเนินการอยู่
2. เลือก **Hold**
3. กล่องโต้ตอบจะปรากฏขึ้นเพื่อขอ **เหตุผลการพัก** (เลือกจากหมวดหมู่เหตุผลความผิดปกติ)
4. สามารถสลับ **Self-resumable** (อนุญาตให้ผู้ปฏิบัติงานกลับมาทำงานต่อได้โดยไม่ต้องขออนุมัติจากผู้วางแผน)
5. คลิก **Confirm**
6. WO จะเปลี่ยนเป็นสถานะ **On Hold** เวลาที่พักจะถูกติดตาม

[image: work-orders-hold-dialog — ภาพหน้าจอของกล่องโต้ตอบเหตุผลการพักพร้อมดรอปดาวน์เหตุผลและตัวสลับ Self-resumable]

### การกลับมาทำงานต่อ (Resuming a WO)

1. คลิกเมนูการดำเนินการบน WO ที่มีสถานะ **On Hold**
2. เลือก **Resume** (→ Released หรือ → In Progress)
3. เวลาพักสะสมจะถูกบันทึกใน `cumulative_hold_minutes`

---

## การขยาย Work Order (รายละเอียดขั้นตอน)

1. คลิกแถว WO ใดก็ได้เพื่อขยาย
2. ส่วนที่ขยายจะแสดงขั้นตอนทั้งหมดตามลำดับพร้อมข้อมูล:
   - หมายเลขและชื่อขั้นตอน
   - ป้ายสถานะ (pending, in_progress, completed, skipped)
   - ประเภทอุปกรณ์ที่ต้องใช้
   - เครื่องจักรที่มอบหมาย (พร้อมตัวเลือกเครื่องจักร)
   - ผู้ปฏิบัติงานที่มอบหมาย
   - ปุ่มการดำเนินการ (Start, Complete, Skip)

[image: work-orders-expanded-steps — ภาพหน้าจอของ WO ที่ขยายแล้วแสดงรายการขั้นตอนพร้อมป้ายสถานะ การมอบหมายเครื่องจักร และปุ่มการดำเนินการ]

---

## การสร้าง Work Order ด้วยตนเอง

1. คลิก **+ Work Order** ในแถบเครื่องมือ
2. กรอกแบบฟอร์ม:
   - รหัสคำสั่ง (สร้างอัตโนมัติหรือกรอกเอง)
   - ชื่อผลิตภัณฑ์, สเปค, วัสดุ
   - ลูกค้า (ดรอปดาวน์)
   - จำนวนและหน่วย
   - ความสำคัญ (urgent, high, normal, low)
   - วันกำหนด
   - เส้นทางกระบวนการ (ดรอปดาวน์ — จะเติมขั้นตอนให้อัตโนมัติ)
3. คลิก **Save**

[image: work-orders-create-form — ภาพหน้าจอของฟอร์มสร้าง Work Order]

> **หมายเหตุ:** WO ส่วนใหญ่ถูกสร้างอัตโนมัติจาก Sales Orders (ดู [คู่มือทีมขาย — สร้าง Work Orders](../sales/07-sales-orders-to-work-orders.md)) การสร้างด้วยตนเองใช้สำหรับงาน ad-hoc

## Work Traveler (เอกสารติดตามงาน)

WO แต่ละรายการมีเอกสาร **Work Traveler** สำหรับพิมพ์:

1. คลิกเมนูการดำเนินการบน WO
2. เลือก **Print Traveler** (🖨)
3. หน้าที่ปรับให้เหมาะสำหรับการพิมพ์จะเปิดขึ้นแสดงรายละเอียด WO, รายการตรวจสอบขั้นตอน, และ QR code สำหรับสแกน

[image: work-orders-traveler — ภาพหน้าจอของมุมมองพิมพ์ Work Traveler]

## ป้าย Rework และ Remake

WO ที่มีต้นกำเนิดจากปัญหาคุณภาพจะแสดงป้ายพิเศษ:

| ป้าย | ความหมาย |
|-------|---------|
| Rework ×N | WO นี้ถูกทำซ้ำ N ครั้ง (ขั้นตอนถูกรีเซ็ตจาก NCR) |
| Remake | WO นี้ถูกสร้างเป็นงานทำใหม่จาก NCR (WO ลูก) |

[image: work-orders-rework-badge — ภาพขยายของการ์ด WO แสดงป้าย Rework และ Remake]
