# Scheduling (การจัดตารางการผลิต)

## คืออะไร?

โมดูล Scheduling คือพื้นที่ทำงานหลักของผู้วางแผน เป็นบอร์ด 3 เลนที่แสดงงานกลึงในโรงงาน งานซัพ และคิว QC ทั้งหมดในมุมมองเดียวที่ตรงกัน — เพื่อให้เห็นภาพรวมการผลิตทั้งหมดโดยไม่ต้องสลับหน้าจอ

## เค้าโครง 3 เลน

```
┌──────────────────────────────────────────────────────┐
│  เลน 1: Timeline Grid                                │
│  แถวเครื่องจักร × คอลัมน์ชั่วโมง (08:00 – 20:00)    │
│  ลากงานไปวางบนเครื่องจักรเพื่อจัดตาราง              │
├──────────────────────────────────────────────────────┤
│  เลน 2: Outside Processing Panel                     │
│  แถวซัพ × คอลัมน์วัน                          │
│  แถบ Gantt: วันจัดส่ง → วันที่คาดว่าจะรับกลับ        │
├──────────────────────────────────────────────────────┤
│  เลน 3: QC Queue Panel                               │
│  การ์ดจัดกลุ่มตามวันสำหรับชิ้นส่วนที่มาถึงการตรวจสอบ │
└──────────────────────────────────────────────────────┘
```

ทั้งสามเลนมีการไฮไลต์ข้ามเลน — เมื่อวางเมาส์บน WO ในเลนใดก็ตาม จะไฮไลต์ WO นั้นในทุกเลนและหรี่รายการอื่น

## Timeline Grid (เลน 1)

พื้นที่จัดตารางหลัก แต่ละแถวคือเครื่องจักร แต่ละคอลัมน์คือชั่วโมงของวันทำงาน แถบ Operation แสดงว่าขั้นตอน WO ไหนถูกมอบหมายให้เครื่องจักรไหนและเมื่อไหร่

- ลากงานที่ยังไม่ได้จัดตารางจาก Sidebar ไปวางบนแถวเครื่องจักรเพื่อมอบหมาย
- ลากแถบที่มีอยู่เพื่อจัดตารางใหม่
- ระบบตรวจจับความขัดแย้งป้องกันการมอบหมายซ้อนทับบนเครื่องจักรเดียวกัน

## มุมมอง By Line Item (ตามรายการสินค้า)

สลับไปแท็บ "By Line Item" เพื่อดูตารางจากมุมมองคำสั่งซื้อลูกค้าแทนมุมมองเครื่องจักร

- แถวคือ Work Order จัดกลุ่มภายใต้หัวข้อ Sales Order ที่พับได้
- หัวข้อ SO แสดง: รหัส SO, ชื่อลูกค้า, จำนวน WO และวันกำหนดส่ง
- แสดงคำเตือนสีแดงหาก SO เกินกำหนดส่งแล้ว
- WO ที่ไม่มี Sales Order จะแสดงในกลุ่ม "งานเดี่ยว" ด้านล่าง
- คอลัมน์วันแสดงว่าขั้นตอนไหนถูกจัดตารางในแต่ละวันสำหรับแต่ละ WO
- คลิกชิปขั้นตอนเพื่อดูรายละเอียด (เหมือนมุมมองเครื่องจักร)
- มุมมองนี้เป็นแบบอ่านอย่างเดียว — สลับไป "By Machine" เพื่อลากจัดตารางใหม่

มุมมองนี้ตอบคำถาม: "คำสั่งซื้อของลูกค้าจะเสร็จเมื่อไหร่?" และ "WO ไหนเป็นคอขวดของการส่งมอบ SO?"

## การจัดตารางอัตโนมัติ

แทนที่จะวางงานด้วยตนเอง สามารถรันระบบจัดตารางอัตโนมัติได้ โดยจัดส่งงานที่ยังไม่ได้จัดตารางทั้งหมดด้วยกฎหนึ่งในสี่กฎ:

| กฎ | ทำอะไร |
|----|--------|
| EDD | Earliest Due Date — ให้ความสำคัญงานที่ใกล้ถึงกำหนดก่อน |
| Priority-Weighted EDD | EDD โดยใช้ Priority ของ WO เป็นตัวตัดสิน |
| SPT | Shortest Processing Time — เพิ่ม Throughput สูงสุด |
| Setup-Aware Grouping | จัดกลุ่ม Setup ที่คล้ายกันบนเครื่องเดียวกันเพื่อลดเวลาเปลี่ยน |

## Shift Roster (ตารางกะทำงาน)

จับคู่ช่างปฏิบัติงานกับเครื่องจักรในแต่ละกะ Roster เป็นตารางเครื่องจักร × กะ ผู้วางแผนมอบหมายว่าช่างคนไหนดูแลเครื่องจักรไหน ข้อมูลนี้ส่งต่อไปยัง Operator Panel เพื่อให้ช่างเห็นงานที่ได้รับมอบหมาย

## Schedule Adherence Report (รายงานความสอดคล้องตาราง)

เปรียบเทียบตารางที่วางแผนไว้กับการดำเนินงานจริง แสดงว่างานไหนเริ่ม/เสร็จตรงเวลา งานไหนล่าช้า และล่าช้าเท่าไหร่ มีประโยชน์สำหรับระบุคอขวดที่เกิดซ้ำ

## แนวคิดหลัก

| คำศัพท์ | ความหมาย |
|---------|----------|
| Operation | ขั้นตอน WO ที่สามารถมอบหมายให้เครื่องจักรได้ |
| Assignment | Operation ที่วางบนเครื่องจักรในวันและชั่วโมงที่กำหนด |
| Unscheduled sidebar | รายการ Operation ที่ยังไม่ได้มอบหมายให้เครื่องจักรใด |
| Lead time gap | เวลาที่สำรองไว้สำหรับงานซัพระหว่างขั้นตอนในโรงงาน |
