# Inventory (คลังสินค้า)

## คืออะไร?

โมดูล Inventory ติดตามการเคลื่อนไหวของวัสดุและชิ้นส่วนเข้าและออกจากโรงงาน ให้มุมมอง real-time ว่ามีอะไรในสต็อก อะไรถูกใช้ไป และอะไรอยู่ในกระบวนการผลิตอยู่

## วิธีการทำงาน

ทุกการเคลื่อนไหวของวัสดุถูกบันทึกเป็น Transaction — Inbound (รับสินค้าเข้า) หรือ Outbound (จ่ายออก) ยอดคงเหลือสต็อกของแต่ละวัสดุคือผลรวมสะสมของ Transaction ทั้งหมด

## Inbound (รับสินค้าเข้า)

Transaction Inbound บันทึกวัสดุที่มาถึงคลังสินค้า:

- การส่งมอบจากซัพพลายเออร์
- การรับคืนจากซัพ (เช่น ชิ้นส่วนกลับจาก Outside Processing)
- การปรับปรุง (แก้ไขจำนวนสต็อก)

แต่ละรายการ Inbound บันทึก: วัสดุ, จำนวน, หมายเลข Lot/Batch, วันที่ และอ้างอิง (หมายเลข PO หรือ WO)

## Outbound (จ่ายออก)

Transaction Outbound บันทึกวัสดุที่ออกจากคลังสินค้า:

- จ่ายให้การผลิต (วัตถุดิบที่ WO ใช้)
- จัดส่งให้ลูกค้า (สินค้าสำเร็จรูป)
- ตัดออก (เขียนออกหลังการตัดสินใจ NCR)

## Stock Balance (ยอดคงเหลือ)

หน้า Stock Balance แสดงจำนวนคงเหลือปัจจุบันต่อวัสดุ สามารถกรองตามหมวดหมู่วัสดุหรือค้นหาตามรหัส/ชื่อ

## In-Plant Materials (วัสดุในกระบวนการ)

In-Plant Materials ติดตามชิ้นส่วนกึ่งสำเร็จรูปที่อยู่ในกระบวนการผลิต — ไม่ได้อยู่ในคลังสินค้า ยังไม่ได้จัดส่ง ให้ visibility ของ WIP (Work In Progress) โดยไม่ต้องตรวจสอบ WO แต่ละใบ

## Inventory Summary (สรุปสินค้าคงคลัง)

มุมมองรวมแสดงมูลค่าและจำนวนรวมของวัสดุทั้งหมด จัดกลุ่มตามหมวดหมู่ มีประโยชน์สำหรับการตรวจสอบสต็อกเป็นระยะ

## แนวคิดหลัก

| คำศัพท์ | ความหมาย |
|---------|----------|
| Inbound | วัสดุที่เข้ามาในสต็อก |
| Outbound | วัสดุที่ออกจากสต็อก |
| Transaction | การเคลื่อนไหวที่บันทึกครั้งเดียว (เข้าหรือออก) |
| Stock Balance | จำนวนคงเหลือปัจจุบันต่อวัสดุ |
| In-Plant | วัสดุที่อยู่ในการผลิต (WIP) |
| Lot / Batch | ตัวระบุการตรวจสอบย้อนกลับสำหรับกลุ่มชิ้นส่วน |
