# 5. คิวงาน (Task Queue)

## การเข้าถึงคิวงาน

1. กด **งานทั้งหมด (All Tasks)** ในแถบดำเนินการด่วนด้านล่าง หรือกดปุ่ม "งานทั้งหมด" บนแดชบอร์ด
2. คิวงานจะเปิดขึ้นแสดงขั้นตอนทั้งหมดที่กำหนดให้กับเครื่องจักรของคุณ

[image: task-queue-full — ภาพหน้าจอคิวงานทั้งหมดแสดง 5–6 แถวงาน]

---

## รายการงาน

แต่ละแถวงานแสดงข้อมูลดังนี้:

| องค์ประกอบ | คำอธิบาย |
|------------|----------|
| Sequence badge (ป้ายลำดับ) | หมายเลขขั้นตอน (สีน้ำเงินหากดำเนินการได้ สีเทาหากยังไม่ได้) |
| WO Code (รหัสใบสั่งผลิต) | รหัสใบสั่งผลิตแบบ monospace |
| Step status (สถานะขั้นตอน) | ป้าย: pending (รอดำเนินการ), in_progress (กำลังดำเนินการ), completed (เสร็จสิ้น), skipped (ข้าม) |
| Rework badge (ป้ายงานซ่อม) | ป้ายสีส้ม "Rework ×N" หาก WO ถูกทำซ้ำ |
| Remake badge (ป้ายผลิตใหม่) | ป้ายสีม่วง "Remake" หาก WO ถูกสร้างจาก NCR |
| Step name (ชื่อขั้นตอน) | ชื่อกระบวนการ (เช่น "CNC Machining") |
| Product name (ชื่อผลิตภัณฑ์) | ชิ้นงานที่กำลังผลิต |
| Arrow (ลูกศร) | เครื่องหมาย → แสดงว่าสามารถกดเพื่อดูรายละเอียดได้ |

[image: task-queue-row — ภาพขยายแถวงานเดียวแสดงองค์ประกอบทั้งหมด]

## งานที่ดำเนินการได้ vs งานที่ยังดำเนินการไม่ได้

- **งานที่ดำเนินการได้ (Actionable tasks)** มีป้ายลำดับสีน้ำเงินและขอบด้านซ้ายสีน้ำเงิน เป็นขั้นตอนที่คุณสามารถเริ่มหรือเสร็จสิ้นได้ทันที
- **งานที่ยังดำเนินการไม่ได้ (Non-actionable tasks)** มีป้ายลำดับสีเทา เป็นขั้นตอนที่รอให้ขั้นตอนก่อนหน้าเสร็จก่อน

[image: task-queue-actionable — ภาพหน้าจอแสดงความแตกต่างทางภาพระหว่างงานที่ดำเนินการได้ (สีน้ำเงิน) และงานที่ยังดำเนินการไม่ได้ (สีเทา)]

---

## การกดที่งาน

กดที่แถวงานใดก็ได้เพื่อเปิด **มุมมองรายละเอียดขั้นตอน (Step Detail View)** (ดูหัวข้อถัดไป)

---

## คิวว่างเปล่า

หากไม่มีงานที่กำหนดให้กับเครื่องจักรของคุณ จะแสดงข้อความ:

> "ไม่มีงาน" พร้อมไอคอนคลิปบอร์ด

[image: task-queue-empty — ภาพหน้าจอสถานะคิวงานว่างเปล่า]

---

## การเรียงลำดับงาน

งานจะถูกเรียงลำดับตาม:
1. งานที่ดำเนินการได้ก่อน (ขั้นตอนที่คุณทำได้ทันที)
2. จากนั้นตามลำดับความสำคัญของ WO (เร่งด่วน > สูง > ปกติ > ต่ำ)
3. จากนั้นตามลำดับขั้นตอนภายในแต่ละ WO
