# ภาพรวมโมดูล TAM3D MES

เอกสารนี้อธิบายแต่ละโมดูลในระบบว่าคืออะไร ทำอะไรได้บ้าง และเชื่อมต่อกับส่วนอื่นอย่างไร ไม่จำกัดเฉพาะบทบาทใดบทบาทหนึ่ง

---

## 1. [Sales (การขาย)](./01-sales.md)

จัดการวงจรชีวิตของคำสั่งซื้อตั้งแต่ต้นจนจบ เริ่มจากใบเสนอราคา (Quote) ที่สามารถแชร์ลิงก์ให้ลูกค้าดูได้โดยไม่ต้องล็อกอิน เมื่อลูกค้าตอบรับจะแปลงเป็น Sales Order พร้อม Line Items ที่ระบุชิ้นส่วน จำนวน และราคา

ฟีเจอร์หลัก: ใบเสนอราคา, Sales Order, Line Items, PDF export, Delivery Promise Signal (ตัวบ่งชี้สีเขียว/เหลือง/แดงว่าโรงงานสามารถส่งมอบตามวันที่ขอได้หรือไม่)

---

## 2. [Production (การผลิต)](./02-production.md)

ติดตามการผลิตทุก Work Order ตั้งแต่สร้างจนเสร็จ Work Order คืองานผลิตชิ้นเดียวสำหรับชิ้นส่วนและจำนวนที่กำหนด แต่ละ WO มีขั้นตอน (Steps) ตามเส้นทางกระบวนการ Project จัดกลุ่ม WO หลายใบที่เกี่ยวข้องกับ Sales Order เดียวกัน

ฟีเจอร์หลัก: Work Orders, Projects, Production Kanban, Production Dashboard (KPI + กราฟ), Order Traceability (ติดตามย้อนกลับจาก SO ถึงขั้นตอนสุดท้าย), Abnormal Reports

---

## 3. [Scheduling (การจัดตารางการผลิต)](./03-scheduling.md)

บอร์ดจัดตารางแบบ 3 เลนที่ให้ผู้วางแผนมองเห็นภาพรวมการผลิตทั้งหมดในหน้าเดียว เลนที่ 1 คือ Timeline Grid สำหรับมอบหมายงานให้เครื่องจักรในช่วงเวลา 08:00–20:00 เลนที่ 2 คือ Outside Processing Panel แสดงงานที่ส่งออกไปยังซัพ เลนที่ 3 คือ QC Queue Panel แสดงงานที่รอตรวจสอบคุณภาพ

ระบบรองรับการจัดตารางอัตโนมัติด้วยกฎ EDD, SPT และ Setup-Aware Grouping รวมถึง Shift Roster สำหรับจับคู่ช่างกับเครื่องจักร และ Schedule Adherence Report สำหรับวิเคราะห์ว่าการผลิตเป็นไปตามแผนหรือไม่

---

## 4. [Outside Processing (งานส่งผลิตภายนอก)](./04-outside-processing.md)

จัดการงานที่ต้องส่งออกไปให้ซัพภายนอกทำ เช่น การชุบผิว การอบชุบ เมื่อ Work Order มีขั้นตอนที่เป็น Outside Service ระบบจะสร้าง Outside Processing Order (OPO) ขึ้นอัตโนมัติ

OPO ติดตามวงจรชีวิต: Pending → Shipped → In Transit → Completed พร้อมแจ้งเตือนเมื่อเกินวันที่คาดว่าจะรับกลับ รองรับการพิมพ์ Packing Slip และ export CSV นอกจากนี้ยังมีหน้าจัดการข้อมูลซัพ (Vendors) แยกต่างหาก

---

## 5. [Quality (คุณภาพ)](./05-quality.md)

บันทึกและติดตามผลการตรวจสอบคุณภาพของชิ้นงาน เมื่อตรวจสอบไม่ผ่านจะสร้าง NCR (Non-Conformance Report) เพื่อบันทึกข้อบกพร่อง สาเหตุ และการตัดสินใจ

NCR รองรับสองเส้นทาง: Rework (ซ่อมแก้ไขขั้นตอนที่มีปัญหาแล้วผลิตต่อ) และ Remake (สร้าง Work Order ใหม่ทั้งใบ) Quality Dashboard แสดง KPI อัตราผ่าน/ไม่ผ่าน และการกระจายของข้อบกพร่อง

---

## 6. [Inventory (คลังสินค้า)](./06-inventory.md)

ติดตามการเคลื่อนไหวของวัสดุและชิ้นส่วนในโรงงาน รองรับการรับสินค้าเข้า (Inbound) การจ่ายออก (Outbound) และดูยอดคงเหลือ (Stock Balance) แบบ real-time

ฟีเจอร์หลัก: รายการเคลื่อนไหว, ยอดคงเหลือต่อวัสดุ, สรุปสต็อก, In-Plant Materials (วัสดุที่อยู่ระหว่างกระบวนการผลิต)

---

## 7. [Equipment (เครื่องจักร)](./07-equipment.md)

ติดตามสถานะและการซ่อมบำรุงของเครื่องจักรทุกเครื่องในโรงงาน Factory Floor Overview แสดงภาพรวมเครื่องจักรแบ่งตาม Workshop และ Workstation

ฟีเจอร์หลัก: สถานะเครื่องจักร (พร้อมใช้/ซ่อมบำรุง/หยุด), Maintenance Tasks (งานซ่อมบำรุงตามแผน), Maintenance Requests (คำร้องขอซ่อมบำรุงจากช่าง)

---

## 8. [Tooling (เครื่องมือตัด)](./08-tooling.md)

จัดการวงจรชีวิตของเครื่องมือตัด CNC ตั้งแต่การกำหนดประเภทเครื่องมือ (Tool Definition) ไปจนถึงการติดตามเครื่องมือแต่ละชิ้น (Tool Instance) และการมอบหมายให้เครื่องจักร

ระบบติดตามอายุการใช้งานเครื่องมือใน 3 มิติ: จำนวนครั้งที่ตัด, เวลาที่ใช้งาน, จำนวนชิ้นงาน และแสดงสถานะ OK / Warning / Critical เมื่อเครื่องมือใกล้หมดอายุ

---

## 9. [Knowledge / Master Data (ข้อมูลหลัก)](./09-knowledge.md)

รากฐานของระบบทั้งหมด โมดูลอื่นทุกโมดูลต้องอาศัยข้อมูลที่ตั้งค่าไว้ที่นี่ก่อนจึงจะทำงานได้

ครอบคลุม: วัสดุ (Materials), คำจำกัดความกระบวนการ (Process Definitions), เส้นทางกระบวนการ (Process Routes), ประเภทเครื่องจักรและเครื่องจักร, ช่างปฏิบัติงานและทักษะ, ลูกค้าและซัพ, การจำลองโรงงาน (Factory Modeling — Workshop/Workstation), การจำลองตารางกะ (Schedule Modeling — Shifts/Work Calendar), Knowledge Health Dashboard (ตรวจสอบความสมบูรณ์ของข้อมูล)

---

## 10. [Operator Interface (หน้าจอช่างปฏิบัติงาน)](./10-operator-interface.md)

หน้าจอแยกสำหรับช่างในหน้างาน ออกแบบสำหรับมือถือ เข้าสู่ระบบด้วย PIN 4 หลัก ไม่ต้องใช้ชื่อผู้ใช้และรหัสผ่าน

Operator Home Dashboard (`/operator`) แสดง 6 การ์ดหลัก: คิวงาน, สแกน QR, รายงานจำนวน, แจ้งปัญหา, ข้อมูลกะ, ผลงาน Operator Panel คือหน้าจอดำเนินงานแบบละเอียดที่แสดงขั้นตอนปัจจุบัน รายละเอียดงาน และปุ่มดำเนินการ (เริ่ม/เสร็จ/พัก/แจ้งปัญหา)

---

## 11. [Activity Feed (ฟีดกิจกรรม)](./11-activity-feed.md)

ศูนย์กลางการสื่อสารและบันทึกการตรวจสอบที่แนบกับ Work Order ทุกรายการ รวมเหตุการณ์ระบบที่สร้างอัตโนมัติ (การเปลี่ยนสถานะ, การย้ายตาราง, เหตุการณ์ NCR, การอัปเดต OPO) กับความคิดเห็นที่ผู้ใช้ป้อนพร้อมรูปภาพ ไฟล์แนบ และแท็กหมวดหมู่

เข้าถึงได้จาก 6 ตำแหน่ง: หน้ารายละเอียด WO, Scheduling Board, Production Kanban, Operator Panel, Order Traceability, และรายละเอียด Outside Processing เปิดเป็นแผงเลื่อนด้านขวาบนเดสก์ท็อป หรือแผ่นล่างเต็มหน้าจอบนมือถือ

คุณสมบัติหลัก: ไทม์ไลน์เหตุการณ์ระบบที่ไม่สามารถแก้ไข, เธรดความคิดเห็นพร้อมรูปภาพ/ไฟล์, การกรองตามแท็กหมวดหมู่ (QC, โลจิสติกส์, วิศวกรรม, ลูกค้า, งานด่วน), หน้าต่างลบ 15 นาทีสำหรับความคิดเห็น, ป้ายจำนวนบนจุดเข้าถึง, อินพุตที่ปรับให้เหมาะกับมือถือพร้อมกล้องเป็นหลัก
