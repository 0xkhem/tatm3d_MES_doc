# 3. Process Definitions (คำจำกัดความกระบวนการ)

## การเข้าถึง Process Definitions

ไปที่ **Knowledge → Basic Data → Process Definition** หรือเข้าที่ `/knowledge/basic/process`

[image: process-def-list — ภาพหน้าจอเต็มของหน้ารายการ Process Definition]

---

## Process Definition คืออะไร?

เทมเพลตการดำเนินงานที่นำกลับมาใช้ได้ ซึ่งกำหนดขั้นตอนการผลิต (เช่น "CNC Milling", "Deburring", "Anodizing") Process Definition เป็นส่วนประกอบพื้นฐานของ Process Route (เส้นทางกระบวนการ)

---

## รายการ Process Definition

### แถบเครื่องมือ (Toolbar)
- ปุ่ม **+ Process Definition**
- **Search box** (ช่องค้นหา) — กรองตามรหัสหรือชื่อ

### คอลัมน์ในตาราง

| คอลัมน์ | คำอธิบาย |
|---------|----------|
| Process Code (รหัสกระบวนการ) | สร้างอัตโนมัติ (เช่น `PROC-CNC-001`) |
| Name (ชื่อ) | ชื่อกระบวนการ (เช่น "CNC 5-Axis Milling") |
| Category (หมวดหมู่) | หมวดกระบวนการ (เช่น "Machining", "Finishing") |
| Equipment Type (ประเภทเครื่องจักร) | ประเภทเครื่องจักรที่ต้องการ (เช่น "5-Axis CNC Mill") |
| Standard Time (เวลามาตรฐาน) | เวลาประมาณการเป็นนาที |
| Outside Service (บริการภายนอก) | ไอคอนรถบรรทุก 🚚 หากเป็นขั้นตอนส่งผลิตภายนอก |
| Status (สถานะ) | ป้าย Active / Inactive |
| Actions (การดำเนินการ) | แก้ไข (✏), ลบ (🗑) |

## การสร้าง Process Definition

1. คลิก **+ Process Definition**
2. กรอกแบบฟอร์ม:

| ฟิลด์ | จำเป็น | คำอธิบาย |
|-------|--------|----------|
| Process Code (รหัสกระบวนการ) | ใช่ | สร้างอัตโนมัติหรือกรอกเอง |
| Name (ชื่อ) | ใช่ | ชื่อกระบวนการ |
| Category (หมวดหมู่) | ไม่ | หมวดหมู่สำหรับจัดกลุ่ม |
| Equipment Type (ประเภทเครื่องจักร) | ไม่ | ดรอปดาวน์ของประเภทเครื่องจักร — เครื่องจักรประเภทใดที่รันกระบวนการนี้ |
| Standard Time (min) (เวลามาตรฐาน/นาที) | ไม่ | เวลา Cycle Time โดยประมาณเป็นนาที |
| Status (สถานะ) | ใช่ | Active หรือ Inactive |
| Is Outside Service (เป็นบริการภายนอก) | Toggle | เมื่อเปิด กระบวนการนี้ดำเนินการโดยผู้ขายภายนอก |
| Setup Group (กลุ่มการตั้งค่า) | ไม่ | ชื่อกลุ่มสำหรับการจัดตารางแบบ Setup-Aware (การดำเนินงานในกลุ่มเดียวกันจะข้ามเวลา Changeover) |

### ฟิลด์ Outside Service (แสดงเมื่อเปิด Toggle)

| ฟิลด์ | คำอธิบาย |
|-------|----------|
| Vendors (ผู้ขาย) | เลือกหลายรายจากรายชื่อผู้ขาย — ผู้ขายรายใดที่สามารถทำบริการนี้ได้ |
| Vendor Lead Time (days) (ระยะเวลานำส่งของผู้ขาย/วัน) | ระยะเวลานำส่งเริ่มต้นเป็นวันทำการ (ค่าเริ่มต้น: 5) |

3. คลิก **Save** (บันทึก)

## แนวคิดสำคัญ

### การเชื่อมโยง Equipment Type

การเชื่อมโยง Process Definition กับ Equipment Type จะบอกเครื่องมือจัดตารางเวลาว่าเครื่องจักรใดสามารถรันการดำเนินงานนี้ได้ หากไม่ได้ตั้งค่า การดำเนินงานจะไม่สามารถจัดตารางอัตโนมัติได้

### ธง Outside Service (บริการภายนอก)

เมื่อเปิดใช้งาน ขั้นตอนกระบวนการนี้จะ:
- สร้าง Outside Processing Order (OPO) อัตโนมัติเมื่อขั้นตอน WO ใช้กระบวนการนี้
- แสดงเป็นขั้นตอนที่ไม่สามารถลากได้ในแถบด้านข้างของตารางเวลา
- ทำหน้าที่เป็นช่วงเวลาว่างในเครื่องมือจัดตารางอัตโนมัติ

### Setup Group (กลุ่มการตั้งค่า)

การดำเนินงานที่มี Setup Group เดียวกันสามารถรันต่อเนื่องกันบนเครื่องจักรเดียวกันโดยไม่ต้องมีเวลา Changeover ใช้โดยกฎการจัดส่ง Setup-Aware Grouping

---

## การลบ Process Definition

Process Definition ที่ถูกอ้างอิงโดย Process Route จะไม่สามารถลบได้ ต้องนำออกจาก Route ทั้งหมดก่อน
