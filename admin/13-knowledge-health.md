# 13. Knowledge Health (สุขภาพข้อมูลหลัก)

## การเข้าถึงหน้า Knowledge Health

ไปที่ **Knowledge → Knowledge Health** หรือเปิด `/knowledge/health`

[image: knowledge-health-full — ภาพหน้าจอเต็มของหน้า Knowledge Health แสดงวงแหวนคะแนนและรายการตรวจสอบ]

---

## วัตถุประสงค์

แดชบอร์ด Knowledge Health แสดงความสมบูรณ์ของข้อมูลหลัก เป็นเครื่องมือแนะนำการเริ่มต้นใช้งานสำหรับการติดตั้งใหม่ และเป็นตัวตรวจสอบความถูกต้องของข้อมูลสำหรับโรงงานที่ใช้งานอยู่แล้ว

---

## คะแนนสุขภาพ (Health Score)

เปอร์เซ็นต์เดียว (0–100) ที่แสดงความพร้อมโดยรวม แสดงเป็นวงแหวนวงกลม:

- **เขียว** (≥80%) — อยู่ในสภาพดี พร้อมสำหรับการผลิต
- **เหลือง** (50–79%) — มีช่องว่างบางส่วน อาจส่งผลต่อการดำเนินงาน
- **แดง** (<50%) — ช่องว่างวิกฤต การผลิตจะถูกจำกัด

คะแนนมีน้ำหนัก: การตรวจสอบที่จำเป็นนับ 10×, แนะนำ 5×, เพิ่มเติม 2×

[image: knowledge-health-score — ภาพใกล้ของวงแหวนคะแนนแสดงเปอร์เซ็นต์]

## หมวดหมู่การตรวจสอบ

### จำเป็น (ต้องมีสำหรับการผลิต)

| การตรวจสอบ | เงื่อนไขผ่าน |
|-------|---------------|
| มีวัสดุ | กำหนดวัสดุอย่างน้อย 1 รายการ |
| มีคำจำกัดความกระบวนการ | กำหนดคำจำกัดความกระบวนการอย่างน้อย 1 รายการ |
| มีเส้นทางกระบวนการ | กำหนดเส้นทางกระบวนการอย่างน้อย 1 รายการ |
| มีประเภทเครื่องจักร | กำหนดประเภทเครื่องจักรอย่างน้อย 1 รายการ |
| มีเครื่องจักร | กำหนดเครื่องจักรอย่างน้อย 1 เครื่อง |
| มีช่างปฏิบัติงาน | กำหนดช่างปฏิบัติงานอย่างน้อย 1 คน |
| กำหนดกะ | กำหนดกะอย่างน้อย 1 กะ |

### แนะนำ (ช่วยปรับปรุงการดำเนินงาน)

| การตรวจสอบ | เงื่อนไขผ่าน |
|-------|---------------|
| มีลูกค้า | กำหนดลูกค้าอย่างน้อย 1 ราย |
| มีหมวดหมู่เครื่องมือ | กำหนดหมวดหมู่เครื่องมืออย่างน้อย 1 หมวด |
| มีคำจำกัดความเครื่องมือ | กำหนดคำจำกัดความเครื่องมืออย่างน้อย 1 รายการ |
| ตั้งค่าปฏิทินทำงาน | กำหนดปฏิทินทำงานอย่างน้อย 1 รายการ |
| มีหมวดหมู่สาเหตุผิดปกติ | กำหนดหมวดหมู่อย่างน้อย 1 หมวด |

### เพิ่มเติม (มีก็ดี)

| การตรวจสอบ | เงื่อนไขผ่าน |
|-------|---------------|
| มีประเภทการซ่อมบำรุง | กำหนดประเภทการซ่อมบำรุงอย่างน้อย 1 รายการ |
| มีกลยุทธ์การซ่อมบำรุง | กำหนดกลยุทธ์อย่างน้อย 1 รายการ |
| มี Tool Instance | กำหนด Tool Instance อย่างน้อย 1 รายการ |

## ตัวบ่งชี้สถานะการตรวจสอบ

| สถานะ | ไอคอน | สี | ความหมาย |
|--------|------|-------|---------|
| ผ่าน | ✅ | เขียว | การตรวจสอบผ่าน — มีข้อมูล |
| เตือน | ⚠ | เหลือง | แนะนำแต่ขาดหายไป |
| ไม่ผ่าน | ✖ | แดง | จำเป็นและขาดหายไป — บล็อกการผลิต |

[image: knowledge-health-checks — ภาพหน้าจอของรายการตรวจสอบแสดงรายการผ่าน, เตือน และไม่ผ่าน]

## ลิงก์แก้ไข (Fix Links)

การตรวจสอบที่ไม่ผ่านหรือเตือนแต่ละรายการมีลิงก์ **Fix →** ที่นำทางไปยังหน้า Knowledge ที่เกี่ยวข้องโดยตรงเพื่อเพิ่มข้อมูลที่ขาดหายไป

| การตรวจสอบที่ไม่ผ่าน | ลิงก์แก้ไข |
|-------------|----------|
| ไม่มีวัสดุ | → หน้า Materials |
| ไม่มีคำจำกัดความกระบวนการ | → หน้า Process Definitions |
| ไม่มีประเภทเครื่องจักร | → หน้า Equipment Types |
| ไม่มีช่างปฏิบัติงาน | → หน้า Operators |
| ไม่มีกะ | → หน้า Schedule Modeling |

[image: knowledge-health-fix-link — ภาพใกล้ของการตรวจสอบที่ไม่ผ่านพร้อมลิงก์ "Fix →"]
