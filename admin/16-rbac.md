# 16. การควบคุมการเข้าถึงตามบทบาท (Role-Based Access Control)

## ภาพรวม

TAM3D MES ใช้ระบบควบคุมการเข้าถึงตามบทบาท (RBAC) ผู้ใช้แต่ละคนได้รับบทบาทหนึ่ง และแต่ละบทบาทมีระดับการเข้าถึงที่กำหนดไว้ (เต็มที่, อ่านอย่างเดียว หรือไม่มี) สำหรับแต่ละโมดูล

---

## ระดับการเข้าถึง

| ระดับ | ความหมาย |
|-------|---------|
| Full (เต็มที่) | สามารถดู, สร้าง, แก้ไข และลบได้ |
| Read-Only (อ่านอย่างเดียว) | สามารถดูได้แต่ไม่สามารถสร้าง, แก้ไข หรือลบ UI จะแสดงตัวบ่งชี้ 👁 |
| None (ไม่มี) | โมดูลถูกซ่อนจากแถบด้านข้าง |

## คำอธิบายบทบาท

### Admin (ผู้ดูแลระบบ)
เข้าถึงได้ทุกอย่าง บทบาทเดียวที่สามารถเข้าถึงการตั้งค่าระบบ (การจัดการผู้ใช้, Running Numbers, เทมเพลตโรงงาน)

### Sales (ทีมขาย)
จัดการใบเสนอราคาและคำสั่งซื้อ สามารถดูโปรเจกต์การผลิต (อ่านอย่างเดียว) และข้อมูลหลัก (อ่านอย่างเดียว) เพื่อตรวจสอบเส้นทางกระบวนการและวัสดุเมื่อสร้างใบเสนอราคา

### Planner (ผู้วางแผน)
จัดการการจัดตารางการผลิต, Work Order และ Outside Processing สามารถดูคำสั่งซื้อ (อ่านอย่างเดียว) เพื่อเข้าใจความต้องการที่เข้ามา สามารถดูคลังสินค้า (อ่านอย่างเดียว) เพื่อตรวจสอบความพร้อมของวัสดุ

### Operator (ช่างปฏิบัติงาน)
ดำเนินงานในหน้างาน เข้าถึง Work Order (เริ่ม/เสร็จสิ้นขั้นตอน) และ Kanban ได้เต็มที่ เข้าถึง Outside Processing และ Quality ได้แบบอ่านอย่างเดียว ใช้หน้าจอ `/operator` แบบแยกต่างหาก

### QC Inspector (ผู้ตรวจสอบคุณภาพ)
จัดการการตรวจสอบคุณภาพและ NCR เข้าถึง Work Order ได้แบบอ่านอย่างเดียว (เพื่อดูสิ่งที่กำลังตรวจสอบ) เข้าถึง Knowledge ได้แบบอ่านอย่างเดียว (เพื่ออ้างอิงคำจำกัดความกระบวนการและวัสดุ)

### Warehouse Staff (พนักงานคลังสินค้า)
จัดการคลังสินค้า (รับเข้า, จ่ายออก, รายการเคลื่อนไหว, ยอดคงเหลือ) เข้าถึง Knowledge ได้แบบอ่านอย่างเดียว (เพื่ออ้างอิงวัสดุ)

### Maintenance Technician (ช่างซ่อมบำรุง)
จัดการสถานะเครื่องจักร, งานซ่อมบำรุง และคำร้องขอซ่อมบำรุง เข้าถึง Knowledge ได้แบบอ่านอย่างเดียว (เพื่ออ้างอิงประเภทเครื่องจักรและกลยุทธ์การซ่อมบำรุง)

### Production Manager (ผู้จัดการการผลิต)
เข้าถึงแบบอ่านอย่างเดียวในวงกว้างทั่วทั้งระบบ เข้าถึง Production และ Outside Processing ได้เต็มที่ เข้าถึง Sales, Inventory, Quality, Equipment และ Knowledge ได้แบบอ่านอย่างเดียว ไม่สามารถเข้าถึงการตั้งค่าระบบได้

## ตัวบ่งชี้อ่านอย่างเดียว

เมื่อผู้ใช้มีสิทธิ์อ่านอย่างเดียวในโมดูล UI จะแสดง:
- ไอคอน 👁 ในพื้นที่แถบเครื่องมือ
- ปุ่ม Create/Edit/Delete ถูกซ่อน
- ฟอร์มถูกปิดใช้งาน
- ผู้ใช้สามารถเรียกดูและค้นหาได้แต่ไม่สามารถแก้ไขข้อมูล

## กรณีพิเศษของ Operator

Operator ที่มีบทบาท "operator" จะถูกเปลี่ยนเส้นทางไปยัง `/operator` (หน้าจอมือถือแบบแยกต่างหาก) เมื่อเข้าสู่ระบบผ่านหน้าเข้าสู่ระบบมาตรฐาน พวกเขาจะไม่เห็นแถบด้านข้างเดสก์ท็อปแบบเต็ม

ถ้า Operator นำทางไปยัง `/home` โดยตรง จะถูกเปลี่ยนเส้นทางไปยัง `/operator`
